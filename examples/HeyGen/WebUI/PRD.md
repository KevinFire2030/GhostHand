# PRD - HeyGen WebUI 아바타 영상 생성기

## 1. 개요
본 문서는 Streamlit 기반의 HeyGen AI 아바타 영상 생성 WebUI 요구사항을 정의한다.

목표는 사용자가 웹 화면에서 HeyGen API Key, avatar_id, voice_id, input_text를 손쉽게 설정하고, 아바타 영상 생성 과정을 실시간으로 확인하며, 생성된 결과 파일(mp4/json/txt)을 다운로드할 수 있도록 하는 것이다.

---

## 2. 목적
- HeyGen API 기반 아바타 영상 생성을 웹 UI로 단순화한다.
- `.env` 기반 설정값을 화면에서 쉽게 관리할 수 있도록 한다.
- 사용 가능한 avatar / voice 목록을 API로 불러와 선택할 수 있도록 한다.
- 영상 생성 진행 상태를 실시간에 가깝게 사용자에게 보여준다.
- 생성 결과를 요약하고, 결과 파일을 다운로드할 수 있게 한다.

---

## 3. 사용자
본 WebUI의 주요 사용자는 다음과 같다.

- HeyGen API를 사용해 아바타 영상을 만들고 싶은 사용자
- 코드 직접 수정 없이 UI로 영상 생성 테스트를 하고 싶은 사용자
- avatar / voice 조합을 빠르게 바꿔가며 실험하고 싶은 사용자
- 생성 결과를 파일 형태로 관리하고 싶은 사용자

---

## 4. 화면 구성 요구사항
화면은 **좌/중/우 3등분 레이아웃**으로 구성한다.

### 4-1. 왼쪽 영역: 설정 / 입력
왼쪽 영역은 `.env` 관리 및 입력값 설정 영역이다.

#### (1) 설정 (.env)
다음 항목을 입력/수정할 수 있어야 한다.
- API Key
- avatar_id
- voice_id
- input_text

#### (2) 입력
다음 입력 항목을 제공해야 한다.
- API Key
- avatar_id
- voice_id
- input_text

#### (3) avatar / voice 목록 조회
- HeyGen API를 통해 사용 가능한 avatar 목록을 조회해야 한다.
- HeyGen API를 통해 사용 가능한 voice 목록을 조회해야 한다.
- 조회된 avatar와 voice는 사용자가 선택할 수 있는 리스트 형태로 제공해야 한다.

#### (4) 기본값 반영
- `.env`에 이미 저장된 값이 있으면 해당 값을 기본 선택값으로 반영해야 한다.
- 사용자는 리스트 선택 외에도 직접 avatar_id, voice_id를 수정할 수 있어야 한다.

#### (5) 저장 기능
- 현재 입력값을 `.env` 파일에 저장할 수 있어야 한다.

---

### 4-2. 중간 영역: 처리
중간 영역은 현재 처리 상태와 진행 로그를 보여주는 영역이다.

#### 표시 요구사항
아래와 같은 처리 상태를 사용자에게 보여줘야 한다.
- 생성 요청 중
- 생성중
- 생성완료
- 다운로드 완료
- 오류 발생 시 오류 메시지

#### 로그 요구사항
- 처리 과정을 순차 로그 형태로 출력해야 한다.
- 예시:
  - 생성 요청 시작
  - video_id 생성
  - 현재 상태: waiting
  - 현재 상태: processing
  - 생성 완료
  - 다운로드 완료

---

### 4-3. 오른쪽 영역: 출력
오른쪽 영역은 생성 결과 요약 및 다운로드 영역이다.

#### 출력 요구사항
아래 정보를 요약해서 보여줘야 한다.
- video_id
- status
- duration
- video_url
- thumbnail_url

#### 다운로드 요구사항
- 생성 완료 후 mp4 파일을 다운로드할 수 있어야 한다.
- 필요 시 json / txt 결과 파일도 확장 가능해야 한다.

---

## 5. 기능 요구사항

### 5-1. 환경설정 파일 처리
- `.env` 파일에서 아래 값을 읽어와야 한다.
  - `HEYGEN_API_KEY`
  - `HEYGEN_AVATAR_ID`
  - `HEYGEN_VOICE_ID`
  - `HEYGEN_INPUT_TEXT`
- UI에서 수정한 값을 `.env`에 다시 저장할 수 있어야 한다.

### 5-2. API 연동
- HeyGen API를 이용해 avatar 목록을 조회해야 한다.
- HeyGen API를 이용해 voice 목록을 조회해야 한다.
- 선택된 avatar_id, voice_id, input_text를 기반으로 영상 생성 API를 호출해야 한다.
- 응답으로 받은 `video_id`를 기반으로 영상 생성 상태를 polling 해야 한다.

### 5-3. 결과 저장
- 생성 완료 후 아래 파일을 저장해야 한다.
  - mp4 파일
  - json 결과 파일
  - txt 결과 요약 파일

### 5-4. 오류 처리
- API Key 누락 시 오류 표시
- avatar/voice 조회 실패 시 오류 표시
- 영상 생성 실패 시 오류 표시
- polling timeout 발생 시 오류 표시

---

## 6. 비기능 요구사항
- Streamlit 기반으로 구현한다.
- 화면은 직관적이고 단순해야 한다.
- 진행 상태가 즉시 눈에 보이도록 한다.
- `.env`에 저장된 기존 값을 최대한 재활용한다.
- 파일 저장 경로는 예제 프로젝트 구조와 일관되게 유지한다.

---

## 7. 파일/폴더 구조 요구사항
예상 구조는 아래와 같다.

```text
HeyGen/
├─ .env
├─ output/
├─ create_avatar_video.py
├─ list_voices.py
└─ WebUI/
   ├─ app.py
   └─ PRD.md
```

---

## 8. 성공 기준
아래 조건을 만족하면 본 WebUI는 요구사항을 충족한 것으로 본다.

1. 사용자가 화면에서 API Key, avatar_id, voice_id, input_text를 입력/저장할 수 있다.
2. 사용 가능한 avatar / voice 목록을 조회하고 선택할 수 있다.
3. `.env`에 저장된 값이 기본 선택값으로 반영된다.
4. 영상 생성 과정의 상태가 화면에 단계별로 표시된다.
5. 생성 완료 후 결과 요약이 화면에 출력된다.
6. mp4 파일을 다운로드할 수 있다.

---

## 9. 향후 확장 아이디어
- json / txt 파일 다운로드 버튼 추가
- input_text를 파일(txt/md) 기반으로 읽기
- 여러 문장을 일괄로 생성하는 batch 기능
- 이전 생성 이력 목록 표시
- 썸네일 미리보기 및 영상 미리보기
- 음성(mp3) 업로드 기반 생성 기능

---

## 10. 한 줄 요약
HeyGen WebUI는 **설정, 처리, 출력의 3단 구성**으로 HeyGen 아바타 영상 생성을 쉽게 실행하고 결과를 다운로드할 수 있게 하는 Streamlit 기반 도구이다.
