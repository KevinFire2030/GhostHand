# Chrome DevTools Recorder 사용법

이 문서는 Chrome에 들어 있는 DevTools Recorder로 사용자의 브라우징 행동을 기록하는 방법을 아주 쉽게 설명한다.

목표는 사용자가 한 번 직접 클릭하고 입력한 과정을 기록한 뒤, 그 기록을 Codex가 보고 브라우징 자동화 스크립트로 바꿀 수 있게 만드는 것이다.

Recorder는 다음 일을 할 수 있다.

- 내가 누른 버튼을 기록한다.
- 내가 입력한 글자를 기록한다.
- 내가 이동한 페이지를 기록한다.
- 기록한 내용을 Puppeteer 또는 Playwright 코드로 내보낼 수 있다.
- 별도 프로그램 설치 없이 Chrome만 있으면 시작할 수 있다.

단, 아주 안정적인 자동화까지 만들려면 나중에 Playwright codegen과 trace를 함께 쓰는 것이 더 좋다. Recorder는 처음 행동을 빠르게 기록하고 흐름을 확인하는 용도로 특히 좋다.

## 준비물

필요한 것은 세 가지다.

- Google Chrome
- 기록할 웹사이트 주소
- 사용자가 직접 따라 할 작업 순서

예를 들어 Gmail 메일 보내기를 기록한다면 다음처럼 준비한다.

- 웹사이트 주소: `https://mail.google.com`
- 받을 사람 이메일
- 메일 제목
- 메일 내용
- 첨부할 파일

## 가장 중요한 주의사항

비밀번호, 개인 정보, 결제 정보, 중요한 파일 경로는 그대로 공유하지 않는다.

Recorder는 사용자가 입력한 내용을 코드에 남길 수 있다. 그래서 기록을 다른 사람이나 Codex에게 보여주기 전에 비밀번호 같은 민감한 값이 들어 있는지 꼭 확인해야 한다.

안전한 방법:

- 비밀번호는 실제 값 대신 `MY_PASSWORD`처럼 가짜 값으로 바꾼다.
- 이메일 주소는 테스트용 주소를 사용한다.
- 첨부파일은 실제 문서 대신 샘플 파일을 사용한다.
- 로그인 세션 파일이나 쿠키는 공개 저장소에 올리지 않는다.

## Recorder 여는 방법

### 방법 1. 메뉴로 열기

1. Chrome을 연다.
2. 기록하고 싶은 웹사이트에 접속한다.
3. 키보드에서 `F12`를 누른다.
4. 화면 오른쪽 또는 아래쪽에 DevTools가 열린다.
5. DevTools 위쪽 탭에서 `Recorder`를 찾는다.
6. `Recorder`가 보이면 클릭한다.

만약 `Recorder` 탭이 안 보이면 다음 방법을 사용한다.

1. DevTools 오른쪽 위의 점 세 개 버튼을 누른다.
2. `More tools`를 누른다.
3. `Recorder`를 누른다.

### 방법 2. 명령어 검색으로 열기

1. Chrome에서 기록할 웹사이트를 연다.
2. `F12`를 눌러 DevTools를 연다.
3. `Ctrl + Shift + P`를 누른다.
4. 검색창에 `Recorder`라고 입력한다.
5. `Show Recorder` 또는 비슷한 이름의 항목을 누른다.

Chrome 버전에 따라 이름이 조금 다를 수 있다. 그래도 `Recorder`라는 단어를 찾으면 된다.

## 새 기록 만들기

Recorder 화면이 열리면 새 기록을 만든다.

1. `Create a new recording` 버튼을 누른다.
2. 기록 이름을 입력한다.
3. 예를 들어 `gmail-send-mail`처럼 적는다.
4. `Start recording` 버튼을 누른다.

이제부터 Chrome은 내가 웹페이지에서 하는 행동을 기록한다.

## 기록할 때 하는 일

기록을 시작한 뒤에는 평소처럼 웹사이트를 사용하면 된다.

예를 들어 Gmail 메일 보내기를 기록한다면 다음 순서로 한다.

1. Gmail에 접속한다.
2. 로그인한다.
3. `편지쓰기` 버튼을 누른다.
4. 받는 사람 칸을 누른다.
5. 받는 사람 이메일을 입력한다.
6. 제목 칸을 누른다.
7. 제목을 입력한다.
8. 본문 칸을 누른다.
9. 내용을 입력한다.
10. 첨부파일 버튼을 누른다.
11. 파일을 선택한다.
12. 보내기 버튼을 누른다.
13. 보냈다는 메시지가 보이는지 확인한다.

중요한 것은 너무 빠르게 하지 않는 것이다. 페이지가 바뀌거나 로딩되는 부분에서는 화면이 완전히 뜰 때까지 잠깐 기다린다.

## 기록 멈추기

작업이 끝나면 Recorder로 돌아간다.

1. DevTools의 `Recorder` 탭을 본다.
2. `End recording` 또는 중지 버튼을 누른다.
3. 기록된 단계들이 목록으로 보이는지 확인한다.

목록에는 보통 이런 행동이 들어 있다.

- Navigate
- Click
- Change
- Key down
- Submit

이 목록이 바로 내가 웹사이트에서 한 행동의 덤프다.

## 기록 다시 실행하기

Recorder는 기록한 행동을 다시 실행해볼 수 있다.

1. Recorder에서 방금 만든 기록을 클릭한다.
2. `Replay` 버튼을 누른다.
3. Chrome이 같은 행동을 다시 하는지 지켜본다.

만약 중간에 멈추면 그 부분이 자동화하기 어려운 부분일 수 있다.

자주 멈추는 이유:

- 페이지가 아직 로딩 중인데 너무 빨리 다음 버튼을 눌렀다.
- 버튼 이름이 바뀌었다.
- 로그인 또는 2단계 인증이 필요하다.
- 파일 업로드 창은 브라우저 밖 기능이라 기록이 완벽하지 않을 수 있다.
- 사이트가 자동화를 막는 보안 기능을 사용한다.

## 기록 내보내기

Codex가 사용하려면 기록을 코드 파일로 내보내는 것이 좋다.

1. Recorder에서 기록을 연다.
2. 오른쪽 위의 내보내기 버튼을 찾는다.
3. `Export`를 누른다.
4. 가능하면 `Playwright` 형식을 선택한다.
5. Playwright가 없으면 `Puppeteer` 형식을 선택한다.
6. 파일로 저장한다.

추천 파일 이름:

```text
recording.spec.ts
```

또는:

```text
recording.js
```

## 저장할 폴더 구조

기록 하나를 저장할 때는 아래처럼 정리하면 Codex가 이해하기 쉽다.

```text
my-recording/
  README.md
  recording.spec.ts
  steps.md
  screenshots/
  uploads/
```

각 파일의 의미는 다음과 같다.

- `README.md`: 어떤 작업을 기록했는지 설명
- `recording.spec.ts`: Recorder에서 내보낸 코드
- `steps.md`: 사람이 읽기 쉽게 적은 단계
- `screenshots/`: 중요한 화면 캡처
- `uploads/`: 테스트용 첨부 파일

## 사람이 읽는 단계도 꼭 적기

Recorder가 코드를 만들어주지만, 사람 말로 된 설명도 같이 있으면 훨씬 좋다.

예시:

```markdown
# Gmail 메일 보내기 기록

1. Chrome에서 Gmail 접속
2. 로그인
3. 편지쓰기 클릭
4. 받는 사람 입력
5. 제목 입력
6. 본문 입력
7. 첨부파일 선택
8. 보내기 클릭
9. "메일을 보냈습니다" 메시지 확인
```

Codex는 이 설명과 Recorder 코드를 함께 보고 더 안정적인 자동화 스크립트를 만들 수 있다.

## Gmail 예시

아래는 Gmail을 예로 든 기록 방법이다. Gmail은 예시일 뿐이고, 다른 웹사이트도 같은 방식으로 기록하면 된다.

### 1단계. Gmail 열기

1. Chrome을 연다.
2. 주소창에 `https://mail.google.com`을 입력한다.
3. Enter를 누른다.

### 2단계. Recorder 열기

1. `F12`를 누른다.
2. DevTools가 열리면 `Recorder` 탭을 찾는다.
3. 없으면 `Ctrl + Shift + P`를 누르고 `Recorder`를 검색한다.
4. `Show Recorder`를 누른다.

### 3단계. 기록 시작

1. `Create a new recording`을 누른다.
2. 이름을 `gmail-send-mail`로 적는다.
3. `Start recording`을 누른다.

### 4단계. 메일 보내기 행동하기

1. 로그인이 필요하면 로그인한다.
2. `편지쓰기` 버튼을 누른다.
3. 받는 사람 칸에 테스트 이메일을 입력한다.
4. 제목 칸에 테스트 제목을 입력한다.
5. 본문 칸에 테스트 내용을 입력한다.
6. 첨부파일 버튼을 누른다.
7. 테스트 파일을 선택한다.
8. `보내기` 버튼을 누른다.
9. 보냈다는 메시지가 보이는지 확인한다.

### 5단계. 기록 중지

1. Recorder 탭으로 돌아간다.
2. `End recording`을 누른다.
3. 기록된 단계가 보이는지 확인한다.

### 6단계. 다시 실행해보기

1. `Replay` 버튼을 누른다.
2. 같은 행동이 다시 실행되는지 본다.
3. 실패한 단계가 있으면 그 단계 이름을 메모한다.

### 7단계. 코드로 내보내기

1. `Export` 버튼을 누른다.
2. `Playwright` 형식을 고른다.
3. 파일 이름을 `recording.spec.ts`로 저장한다.

## Codex에게 넘길 때 같이 주면 좋은 정보

Recorder 파일만 주는 것보다 아래 정보를 같이 주면 자동화가 더 잘 된다.

```text
작업 이름:
시작 URL:
로그인이 필요한지:
입력해야 할 값:
업로드해야 할 파일:
마지막에 성공했다고 판단하는 화면:
자동화하면 안 되는 위험한 버튼:
중간에 사람이 직접 해야 하는 단계:
```

예시:

```text
작업 이름: Gmail 첨부 메일 보내기
시작 URL: https://mail.google.com
로그인이 필요한지: 필요함
입력해야 할 값: 받는 사람, 제목, 본문
업로드해야 할 파일: sample.pdf
마지막 성공 화면: "메일을 보냈습니다" 메시지
자동화하면 안 되는 위험한 버튼: 전체 삭제, 계정 삭제
중간에 사람이 직접 해야 하는 단계: 2단계 인증
```

## Recorder 결과를 자동화 스크립트로 바꾸는 흐름

Codex는 보통 다음 순서로 작업한다.

1. Recorder에서 내보낸 코드를 읽는다.
2. 사람이 적은 `steps.md`를 같이 읽는다.
3. 클릭, 입력, 업로드 순서를 확인한다.
4. 비밀번호 같은 민감한 값은 환경 변수로 바꾼다.
5. 불안정한 셀렉터를 더 안정적인 셀렉터로 바꾼다.
6. 필요한 대기 조건을 추가한다.
7. 마지막 성공 확인 코드를 추가한다.
8. Playwright 스크립트로 실행한다.
9. 실패하면 스크린샷과 trace를 보고 고친다.

## 좋은 기록을 만드는 팁

- 한 번에 너무 많은 일을 기록하지 않는다.
- 하나의 기록에는 하나의 목표만 담는다.
- 버튼을 누른 뒤 화면이 바뀌면 잠깐 기다린다.
- 입력값은 테스트용 값을 사용한다.
- 첨부파일은 샘플 파일을 사용한다.
- 기록 이름은 알아보기 쉽게 짓는다.
- 실패한 단계가 있으면 꼭 메모한다.
- 마지막 성공 화면을 캡처해둔다.

좋은 기록 이름:

```text
gmail-send-mail
invoice-upload
admin-create-user
dashboard-download-report
```

나쁜 기록 이름:

```text
test
aaa
recording1
```

## 한계

Chrome DevTools Recorder는 편하지만 완벽하지는 않다.

어려울 수 있는 것:

- 2단계 인증
- CAPTCHA
- 파일 선택 창
- 팝업 창
- 브라우저 밖에서 일어나는 행동
- 화면이 자주 바뀌는 사이트
- 보안이 강한 사이트

이럴 때는 Recorder 결과를 그대로 쓰기보다 Playwright 코드로 정리해야 한다.

## 최종 체크리스트

기록을 끝낸 뒤 아래를 확인한다.

- Recorder 기록을 저장했는가?
- 코드로 Export 했는가?
- 사람이 읽는 단계 설명을 적었는가?
- 비밀번호와 개인정보를 지웠는가?
- 샘플 첨부파일을 준비했는가?
- 마지막 성공 조건을 적었는가?
- Replay로 한 번 다시 실행해봤는가?
- 실패한 단계가 있다면 메모했는가?

## 결론

Chrome DevTools Recorder는 사용자가 직접 한 브라우징 행동을 빠르게 기록하는 도구다. Chrome만 있으면 시작할 수 있고, 클릭과 입력 흐름을 코드로 내보낼 수 있다.

Recorder로 먼저 행동을 기록하고, 그 결과를 Codex가 Playwright 스크립트로 다듬으면 반복적인 브라우징 작업을 자동화하기 쉬워진다.
