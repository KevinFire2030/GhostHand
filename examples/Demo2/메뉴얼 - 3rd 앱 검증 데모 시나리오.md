# 메뉴얼 - 3rd 앱 검증 데모 시나리오

## 메시지 입력 시나리오

### 1. 작업 폴더 생성
`Demo2` 폴더에 `3rdApp` 폴더를 만들고 GitHub에 푸시한다.

### 2. 검증 시나리오 문서 작성 및 GitHub 푸시
`3rdApp` 폴더에 `3rd 앱 검증 시나리오.md` 파일을 작성한다.  
TC는 10개 작성한다.  
TC는 일반 사용자가 일상에서 자주 사용하는 기능 위주로 검증할 수 있는 케이스로 작성한다.  
작성한 파일을 GitHub에 푸시한다.

참고 시나리오:
- 필요 시 기존 `examples` 하위 검증 시나리오 문서를 참고해 형식을 맞춘다.

### 3. GitHub 업로드 결과 확인
GitHub에 올라간 검증 시나리오 파일을 확인한다.

### 4. 구글 플레이 앱 실행
구글 플레이 앱을 실행한다.

### 5. 인스타그램 앱을 다운로드 및 설치
구글 플레이에서 인스타그램 앱을 검색하고 설치한다.

### 6. 검증 시나리오 실행
작성한 시나리오 기준으로 3rd 앱 검증을 실행한다.

각 TC의 실행 결과는 화면 캡처한다.  
캡처 파일은 `3rdApp/screencapture` 폴더에 저장한다.  
예: `TestCase1.png`

캡처한 화면을 검증하여 결과를 `테스트 결과.md`에 정리한다.  
결과 파일을 GitHub에 푸시한다.

### 7. 검증 결과 알림
검증이 끝나면 결과를 텔레그램 메신저로 알린다.

포함 내용:
- 결과 요약
- `테스트 결과.md` 파일 첨부

### 8. 스킬 추가
이 데모 워크플로우를 자동화할 수 있도록 스킬 형태로 정리한다.  
방금 실행한 데모를 `테마앱 검증 스킬.md`로 저장하고 GitHub에 푸시한다.
