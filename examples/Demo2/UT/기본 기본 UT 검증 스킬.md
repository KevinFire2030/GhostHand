# 기본 기본 UT 검증 스킬

## 1. 목적
메시지 입력만으로 기본 UT 검증 시나리오 작성, 기본 검증 실행, 화면 캡처 및 녹화 저장, 테스트 결과 정리, FAIL 항목 GitHub Issues 등록, 메신저 결과 알림까지 이어지는 데모 워크플로우를 반복 가능하게 수행한다.

## 2. 입력 조건
- 작업 경로: `examples/Demo2`
- 검증 폴더: `examples/Demo2/UT`
- 시나리오 문서: `examples/Demo2/UT/기본 UT 검증 시나리오.md`
- 결과 문서: `examples/Demo2/UT/테스트 결과.md`
- 캡처 경로: `examples/Demo2/UT/screencapture`
- 녹화 경로: `examples/Demo2/UT/screenrecord`

## 3. 사전 준비
1. `examples/Demo2/UT` 폴더가 없으면 생성한다.
2. 아래 하위 폴더가 없으면 함께 생성한다.
   - `UT/screencapture`
   - `UT/screenrecord`
3. ADB 연결 상태를 확인한다.
4. 현재 단말이 심카드 미삽입 상태임을 확인한다.
5. 전화 걸기와 문자 보내기는 기본 FAIL 데모 케이스로 처리한다.

## 4. 실행 절차

### Step 1. 검증 시나리오 문서 작성
- `기본 UT 검증 시나리오.md` 파일을 작성한다.
- 테스트 케이스는 아래 5개를 사용한다.
  1. 전화 걸기 - 홈으로 이동
  2. 문자 보내기 - 홈으로 이동
  3. 크롬 앱 실행 - 네이버 이동 - 수원 날씨 검색 - 홈으로 이동
  4. 유튜브 앱 실행 - 오픈클로 검색 - 재생 - 홈으로 이동
  5. 시계 앱 실행 - 07:00 알람 설정 - 알람 해제 - 홈으로 이동
- 시나리오 문서를 GitHub에 푸시한다.

### Step 2. GitHub 업로드 결과 확인
- GitHub 저장소에서 `기본 UT 검증 시나리오.md` 업로드 여부를 확인한다.

### Step 3. 기본 검증 실행
각 테스트 케이스에 대해 아래 절차를 반복한다.
1. 화면 녹화를 시작한다.
2. 테스트 시나리오를 실행한다.
3. 결과 화면을 캡처한다.
4. 녹화를 종료하고 저장한다.
5. 홈 화면으로 복귀한다.

### Step 4. 증빙 저장 규칙
- 캡처 파일 경로: `UT/screencapture/TestCase1.png` 형식 사용
- 녹화 파일 경로: `UT/screenrecord/TestCase1.mp4` 형식 사용
- 테스트 케이스 번호와 파일명을 일치시킨다.

### Step 5. 테스트 결과 정리
- `테스트 결과.md` 파일을 작성한다.
- 각 TC별로 아래 항목을 기록한다.
  - 테스트 항목명
  - PASS/FAIL
  - 사유
  - 캡처 파일 경로
  - 녹화 파일 경로
- 전화 걸기와 문자 보내기는 심카드 미삽입 상태로 인해 FAIL 처리한다.
- 결과 문서를 GitHub에 푸시한다.

### Step 6. FAIL 항목 이슈 등록
FAIL 항목이 발생하면 GitHub Issues에 등록한다.

기본 등록 대상:
- 전화 걸기 실패
- 문자 보내기 실패

이슈 본문 필수 항목:
- 재현 경로
- FAIL 사유
- 첨부파일 경로 또는 시험 영상 URL 링크

### Step 7. 검증 결과 알림
검증 종료 후 메신저로 아래 내용을 요약 전달한다.
- PASS/FAIL 요약
- FAIL 항목 목록
- GitHub Issue 링크
- 결과 문서 경로

## 5. 판정 기준
- FAIL:
  - 심카드 미삽입으로 인한 전화 실패
  - 심카드 미삽입으로 인한 문자 실패
  - 앱 실행 실패
  - 검색 실패
  - 영상 재생 실패
  - 알람 설정 실패
- PASS:
  - 앱 실행 및 핵심 동작 성공
  - 캡처/녹화 저장 완료
  - 결과 문서 정리 완료

## 6. 이번 데모 반영 기준
- 전화 걸기 실패 이슈 등록 완료
- 문자 보내기 실패 이슈 등록 완료
- 캡처 5건, 녹화 5건 저장 완료
- 결과 문서 작성 및 GitHub 푸시 완료
