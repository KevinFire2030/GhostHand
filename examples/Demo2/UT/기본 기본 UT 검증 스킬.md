# 기본 기본 UT 검증 스킬

## 1. 목적
메시지 입력만으로 기본 UT 검증 시나리오 작성, 검증 실행, 캡처/녹화 저장, 결과 문서 작성, FAIL 항목 이슈 등록, 메신저 알림까지 이어지는 데모 워크플로우를 반복 가능하게 수행한다.

## 2. 입력 조건
- 작업 경로: `examples/Demo2`
- 기준 시나리오 문서: `examples/Demo2/UT/기본 UT 검증 시나리오.md`
- 결과 문서: `examples/Demo2/UT/테스트 결과.md`
- 캡처 경로: `examples/Demo2/UT/screencapture`
- 녹화 경로: `examples/Demo2/UT/screenrecord`
- 참고 문서: `examples/plm/scrrec.md`

## 3. 사전 확인
1. `examples/Demo2/UT` 폴더가 존재하는지 확인한다.
2. 아래 하위 폴더가 없으면 생성한다.
   - `UT/screencapture`
   - `UT/screenrecord`
3. ADB 연결 상태를 확인한다.
4. 검증 대상 앱(전화, 메시지, 크롬, 유튜브, 시계) 실행 가능 여부를 확인한다.
5. 현재 단말이 **심카드 미삽입 상태**임을 전제로 전화/문자 항목은 FAIL 예상 케이스로 처리한다.

## 4. 실행 절차

### Step 1. 시나리오 문서 준비
- `기본 UT 검증 시나리오.md`가 없으면 생성한다.
- 테스트 케이스는 아래 5개를 사용한다.
  1. 전화 걸기 - 홈으로 이동
  2. 문자 보내기 - 홈으로 이동
  3. 크롬 앱 실행 - 네이버 이동 - 수원 날씨 검색 - 홈으로 이동
  4. 유튜브 앱 실행 - 오픈클로 검색 - 재생 - 홈으로 이동
  5. 시계 앱 실행 - 07:00 알람 설정 - 알람 해제 - 홈으로 이동

### Step 2. 기본 검증 실행
각 TC에 대해 아래를 반복 수행한다.
1. 화면 녹화를 시작한다.
2. 테스트 시나리오를 실행한다.
3. 결과 화면을 캡처한다.
4. 녹화를 종료하고 파일을 저장한다.
5. 홈 화면으로 복귀한다.

### Step 3. 산출물 저장 규칙
- 캡처 파일명 예시:
  - `TestCase1.png`
  - `TestCase2.png`
- 녹화 파일명 예시:
  - `TestCase1.mp4`
  - `TestCase2.mp4`

### Step 4. 결과 정리
- 각 TC의 PASS/FAIL 결과를 `테스트 결과.md`에 기록한다.
- 필수 항목:
  - 테스트 항목명
  - 결과(PASS/FAIL)
  - 실패 또는 성공 근거
  - 첨부 캡처/녹화 파일 경로
- 전화/문자 항목은 아래 기준으로 기록한다.
  - 결과: FAIL
  - 사유: 심카드 미삽입 환경
  - 후속 조치: GitHub Issues 등록 대상

### Step 5. GitHub 반영
아래 항목을 GitHub에 푸시한다.
- `기본 UT 검증 시나리오.md`
- `테스트 결과.md`
- `screencapture/*`
- `screenrecord/*`
- 본 스킬 문서

### Step 6. FAIL 항목 이슈 등록
FAIL 항목이 존재하면 GitHub Issues에 등록한다.

#### 기본 등록 대상
- 전화 걸기 실패
- 문자 보내기 실패

#### 이슈 작성 형식
- 제목: 실패 항목명
- 본문:
  - 재현 경로
  - FAIL 사유
  - 첨부파일 경로 또는 시험 영상 URL 링크

#### 예시
- 제목: `전화 걸기 실패`
- 내용:
  - 재현 경로
  - FAIL 사유: 심카드 미삽입으로 통화 불가
  - 첨부파일: `UT/screenrecord/TestCase1.mp4`

### Step 7. 메신저 알림
검증 종료 후 메신저로 아래 내용을 요약 전달한다.
- 전체 결과 요약(PASS/FAIL 개수)
- FAIL 항목 목록
- GitHub Issue 링크
- 결과 문서 경로

## 5. 결과 판정 기준
- **FAIL**
  - 전화 걸기 실패
  - 문자 보내기 실패
  - 기타 앱 실행 불가, 네트워크 실패, 검색 실패, 재생 실패, 알람 설정 실패
- **PASS**
  - 앱 실행 성공
  - 시나리오 핵심 동작 완료
  - 증빙 파일 저장 완료

## 6. 권장 자동화 흐름
1. 메시지 입력 수신
2. UT 폴더 구조 확인
3. 시나리오 문서 확인 또는 생성
4. ADB로 각 TC 실행
5. 캡처/녹화 저장
6. 결과 문서 작성
7. GitHub 커밋/푸시
8. FAIL 이슈 등록
9. 메신저 결과 알림

## 7. 이번 데모 실행 기준 반영 내용
- 전화 걸기 FAIL 이슈 등록 완료
- 문자 보내기 FAIL 이슈 등록 완료
- 테스트 결과 문서 작성 완료
- 캡처 5건 및 녹화 5건 저장 완료
