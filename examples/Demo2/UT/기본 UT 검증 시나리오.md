# 기본 UT 검증 시나리오

## 1. 개요
본 문서는 `examples/Demo2/UT` 기준의 기본 UT 검증 시나리오를 정의한다.  
총 테스트 케이스는 5개이며, 각 TC는 실행 후 화면 캡처 및 화면 녹화를 남긴다.

> 현재 단말은 **심카드 미삽입 상태**이므로, **전화 걸기** 및 **문자 보내기** 항목은 환경 제약에 의해 **FAIL 예상 케이스**로 처리한다.  
> 해당 항목은 검증 결과 정리 후 **GitHub Issues 등록 대상**에 포함한다.

## 2. 테스트 케이스 목록

### TC1. 전화 걸기 - 홈으로 이동
- 목적: 전화 앱 실행 및 전화 걸기 동작 확인
- 사전 조건: 심카드 미삽입 상태
- 절차:
  1. 전화 앱을 실행한다.
  2. 지정된 번호 또는 테스트 연락처로 전화 걸기를 시도한다.
  3. 실패 메시지 또는 통화 불가 상태를 확인한다.
  4. 홈 화면으로 이동한다.
- 예상 결과:
  - 전화 앱은 정상 실행된다.
  - 심카드 미삽입으로 인해 통화 연결은 실패한다.
  - FAIL 케이스로 기록하고 이슈 등록 대상에 포함한다.
- 예상 판정: FAIL

### TC2. 문자 보내기 - 홈으로 이동
- 목적: 메시지 앱 실행 및 문자 발송 동작 확인
- 사전 조건: 심카드 미삽입 상태
- 절차:
  1. 메시지 앱을 실행한다.
  2. 지정된 연락처 또는 테스트 번호를 입력한다.
  3. 테스트 메시지를 작성한다.
  4. 문자 전송을 시도한다.
  5. 실패 메시지 또는 전송 불가 상태를 확인한다.
  6. 홈 화면으로 이동한다.
- 예상 결과:
  - 메시지 앱은 정상 실행된다.
  - 심카드 미삽입으로 인해 문자 전송은 실패한다.
  - FAIL 케이스로 기록하고 이슈 등록 대상에 포함한다.
- 예상 판정: FAIL

### TC3. 크롬 앱 실행 - 네이버 이동 - 수원 날씨 검색 - 홈으로 이동
- 목적: 브라우저 실행 및 검색 기능 확인
- 절차:
  1. 크롬 앱을 실행한다.
  2. 네이버로 이동한다.
  3. 검색창에 `수원 날씨`를 입력하고 검색한다.
  4. 검색 결과 화면이 정상 표시되는지 확인한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 크롬 앱이 정상 실행된다.
  - 네이버 접속이 정상 동작한다.
  - `수원 날씨` 검색 결과가 정상 표시된다.
- 예상 판정: PASS

### TC4. 유튜브 앱 실행 - 오픈클로 검색 - 재생 - 홈으로 이동
- 목적: 유튜브 검색 및 동영상 재생 기능 확인
- 절차:
  1. 유튜브 앱을 실행한다.
  2. 검색창에서 `오픈클로`를 검색한다.
  3. 검색 결과 중 임의의 영상을 선택한다.
  4. 영상 재생이 정상 동작하는지 확인한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 유튜브 앱이 정상 실행된다.
  - 검색 결과가 정상 표시된다.
  - 선택한 영상이 정상 재생된다.
- 예상 판정: PASS

### TC5. 시계 앱 실행 - 07:00 알람 설정 - 알람 해제 - 홈으로 이동
- 목적: 시계 앱의 알람 설정 및 해제 기능 확인
- 절차:
  1. 시계 앱을 실행한다.
  2. 07:00 알람을 신규 설정한다.
  3. 저장된 알람이 목록에 표시되는지 확인한다.
  4. 설정한 알람을 해제한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 시계 앱이 정상 실행된다.
  - 07:00 알람이 정상 등록된다.
  - 알람 해제가 정상 동작한다.
- 예상 판정: PASS

## 3. 증빙 산출물 규칙
- 화면 캡처 저장 경로: `UT/screencapture`
- 화면 녹화 저장 경로: `UT/screenrecord`
- 파일명 예시:
  - `TestCase1.png`
  - `TestCase1.mp4`
  - `TestCase2.png`
  - `TestCase2.mp4`

## 4. 결과 정리 및 후속 조치
- 각 TC 수행 후 `테스트 결과.md`에 PASS/FAIL 및 근거를 기록한다.
- FAIL 항목은 재현 경로, 사유, 첨부파일 정보를 포함하여 GitHub Issues에 등록한다.
- 특히 아래 2개 항목은 기본 FAIL 데모 대상으로 관리한다.
  - 전화 걸기 실패
  - 문자 보내기 실패
