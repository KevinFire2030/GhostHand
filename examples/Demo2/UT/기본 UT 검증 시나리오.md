# 기본 UT 검증 시나리오

## 1. 개요
본 문서는 `examples/Demo2/UT` 경로 기준의 기본 UT 검증 시나리오를 정의한다.
테스트 케이스(TC)는 총 5개로 구성한다.
현재는 심카드가 삽입되지 않은 상태이므로, 전화 걸기와 문자 보내기는 항상 FAIL 케이스로 처리하고 문제점 등록 대상에 포함한다.

## 2. 테스트 케이스

### TC1. 전화 걸기 - 홈으로 이동
- 목적: 전화 앱 실행 및 전화 걸기 동작 확인
- 절차:
  1. 전화 앱을 실행한다.
  2. 테스트 번호를 입력하거나 선택한다.
  3. 전화 걸기를 시도한다.
  4. 홈 화면으로 이동한다.
- 예상 결과:
  - 심카드 미삽입 상태로 인해 통화 연결은 실패한다.
  - FAIL 케이스로 기록하고 문제점 등록 대상에 포함한다.
- 예상 판정: FAIL

### TC2. 문자 보내기 - 홈으로 이동
- 목적: 메시지 앱 실행 및 문자 보내기 동작 확인
- 절차:
  1. 메시지 앱을 실행한다.
  2. 테스트 번호를 입력한다.
  3. 테스트 메시지를 작성한다.
  4. 전송을 시도한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 심카드 미삽입 상태로 인해 문자 전송은 실패한다.
  - FAIL 케이스로 기록하고 문제점 등록 대상에 포함한다.
- 예상 판정: FAIL

### TC3. 크롬 앱 실행 - 네이버 이동 - 수원 날씨 검색 - 홈으로 이동
- 목적: 브라우저 실행 및 검색 기능 확인
- 절차:
  1. 크롬 앱을 실행한다.
  2. 네이버로 이동한다.
  3. `수원 날씨`를 검색한다.
  4. 검색 결과를 확인한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 크롬 앱 실행 및 검색이 정상 동작한다.
- 예상 판정: PASS

### TC4. 유튜브 앱 실행 - 오픈클로 검색 - 재생 - 홈으로 이동
- 목적: 유튜브 검색 및 재생 기능 확인
- 절차:
  1. 유튜브 앱을 실행한다.
  2. `오픈클로`를 검색한다.
  3. 검색 결과 중 하나를 선택한다.
  4. 영상 재생을 확인한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 유튜브 앱 실행, 검색, 재생이 정상 동작한다.
- 예상 판정: PASS

### TC5. 시계 앱 실행 - 07:00 알람 설정 - 알람 해제 - 홈으로 이동
- 목적: 알람 설정 및 해제 기능 확인
- 절차:
  1. 시계 앱을 실행한다.
  2. 07:00 알람을 설정한다.
  3. 알람이 등록되었는지 확인한다.
  4. 알람을 해제한다.
  5. 홈 화면으로 이동한다.
- 예상 결과:
  - 알람 설정 및 해제가 정상 동작한다.
- 예상 판정: PASS
