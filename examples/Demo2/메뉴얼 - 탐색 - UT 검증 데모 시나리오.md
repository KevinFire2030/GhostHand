# 메뉴얼 - 탐색 - UT 검증 데모 시나리오

## 1. 목적
메시지 입력 기반으로 메뉴얼 - 탐색 - UT 검증 시나리오를 작성하고, 검증 실행 결과를 캡처/녹화/정리한 뒤, FAIL 항목은 GitHub Issues에 등록하고 최종 결과를 메신저로 알리는 데모를 수행한다.

## 2. 메시지 입력 시나리오

### 1. 작업 폴더 생성
`examples/Demo2` 폴더 아래에 `UT` 폴더를 생성한다.  
필요 시 아래 하위 폴더도 함께 생성한다.

- `UT/screencapture`
- `UT/screenrecord`

### 2. 검증 시나리오 문서 작성 및 GitHub 푸시
`기본 UT 검증 시나리오.md` 파일을 작성한다.  
테스트 케이스(TC)는 5개 작성한다.  
작성한 파일을 GitHub에 푸시한다.  
현재는 심카드가 삽입되지 않은 상태이므로, 전화 걸기와 문자 보내기는 항상 FAIL 케이스로 처리하고 문제점 등록 대상에 포함되도록 한다.

#### 기본 UT 검증 시나리오 예시
1. 전화 걸기 - 홈으로 이동
2. 문자 보내기 - 홈으로 이동
3. 크롬 앱 실행 - 네이버 이동 - 수원 날씨 검색 - 홈으로 이동
4. 유튜브 앱 실행 - 오픈클로 검색 - 재생 - 홈으로 이동
5. 시계 앱 실행 - 07:00 알람 설정 - 알람 해제 - 홈으로 이동

### 3. GitHub 업로드 결과 확인
GitHub 저장소에 업로드된 검증 시나리오 파일을 확인한다.

### 4. 기본 검증 실행
작성한 시나리오 기준으로 기본 UT 검증을 실행한다.  
각 TC의 실행 결과는 화면 캡처한다.  
캡처 파일은 `UT/screencapture` 폴더에 저장한다.  
예: `TestCase1.png`  
각 TC의 실행 시 녹화 시작하고 종료 시 녹화를 종료하고 저장한다.  
녹화 파일은 `UT/screenrecord` 폴더에 저장한다.  
예: `TestCase1.mp4`  
캡처한 화면을 검증하여 결과를 `테스트 결과.md`에 정리한다.  
결과 파일을 GitHub에 푸시한다.

### 5. FAIL 항목 이슈 등록
FAIL 항목이 발생한 경우 GitHub Issues에 문제점으로 등록한다.  
특히 심카드 미삽입 상태로 인해 실패가 예상되는 전화 걸기, 문자 보내기 항목은 문제점 등록 데모의 기본 FAIL 케이스로 활용한다.

#### 등록 예시
- 제목: 전화 걸기 실패
- 내용:
  - 재현 경로
  - FAIL 사유
  - 첨부파일: 시험 영상 URL 링크

### 6. 검증 결과 알림
검증이 끝나면 결과를 텔레그램 메신저로 알린다.

### 7. 스킬 추가
이 데모 워크플로우를 자동화할 수 있도록 스킬 형태로 정리한다.  
방금 실행한 데모를 `기본 기본 UT 검증 스킬.md`로 저장하고 GitHub에 푸시한다.

### 8. Cron Job 등록
저장한 기본 UT 검증 스킬이 매시간 정시에 실행될 수 있게 Cron Job으로 등록한다.

## 3. 기대 효과
- 메시지 입력만으로 기본 검증 시나리오 작성부터 실행, 결과 정리, 이슈 등록까지 이어지는 흐름을 시연할 수 있다.
- 증빙 자료를 캡처 및 녹화 형태로 함께 남길 수 있다.
- FAIL 항목을 GitHub Issues와 연계하여 추적 가능한 형태로 관리할 수 있다.
- 심카드 미삽입과 같은 환경 제약도 의도적인 FAIL 케이스로 활용하여 문제점 등록 시연이 가능하다.
- 향후 스킬 형태로 자동화 가능한 워크플로우의 기반 시나리오로 활용할 수 있다.
