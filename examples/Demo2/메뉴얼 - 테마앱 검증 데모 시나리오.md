# 메뉴얼 - 테마앱 검증 데모 시나리오

## 데모 시나리오

## 메시지 입력 시나리오

### 1. 작업 폴더 생성
`Demo2` 폴더에 `ThemeApp` 폴더를 만든다.

### 2. 검증 시나리오 문서 작성 및 GitHub 푸시
`ThemeApp` 폴더에 `테마앱 검증 시나리오.md` 파일을 작성한다.
TC는 10개 작성한다.
작성한 파일을 GitHub에 푸시한다.
참고 시나리오:

- <https://github.com/KevinFire2030/GhostHand/blob/main/examples/Theme/%ED%85%8C%EC%8A%A4%ED%8A%B8%20%EC%BC%80%EC%9D%B4%EC%8A%A4.md>

### 3. GitHub 업로드 결과 확인
GitHub에 올라간 검증 시나리오 파일을 확인한다.

### 4. 테마 앱 실행
테마 앱을 실행한다.

### 5. 무료 테마 다운로드 및 설치
무료 테마를 검색하고 설치한다.

### 6. 검증 시나리오 실행
작성한 시나리오 기준으로 테마 앱 검증을 실행한다.

각 TC의 실행 결과는 화면 캡처한다.
캡처 파일은 `ThemeApp/screencapture` 폴더에 저장한다.
예: `TestCase1.png`

캡처한 화면을 검증하여 결과를 `테스트 결과.md`에 정리한다.
결과 파일을 GitHub에 푸시한다.
참고 테스트 결과 리포트:

- <https://github.com/KevinFire2030/GhostHand/blob/main/examples/Theme/%EC%BA%A1%EC%B2%98%20%ED%8F%AC%ED%95%A8%20%ED%85%8C%EC%8A%A4%ED%8A%B8%20%EA%B2%B0%EA%B3%BC.md>

### 7. FAIL 항목 이슈 등록
FAIL 항목이 발생한 경우 GitHub Issues에 문제점으로 등록한다.

#### 등록 예시
- 제목: 테마앱 - 글자 깨짐
- 내용:
  - 재현 경로
  - FAIL 사유
  - 첨부파일: 시험 결과 캡처 이미지 URL 링크

### 8. 검증 결과 알림
검증이 끝나면 결과를 텔레그램 메신저로 알린다.
포함 내용:
- 결과 요약
- `테스트 결과.md` 파일 첨부

### 9. 스킬 추가
이 데모 워크플로우를 자동화할 수 있도록 스킬 형태로 정리한다.
방금 실행한 데모를 `테마앱 검증 스킬.md`로 저장하고 GitHub에 푸시한다.

## 3. 기대 효과
- 메시지 입력만으로 테마앱 검증 시나리오 작성부터 실행, 결과 정리, 이슈 등록까지 이어지는 흐름을 시연할 수 있다.
- 증빙 자료를 캡처 및 녹화 형태로 함께 남길 수 있다.
- FAIL 항목을 GitHub Issues와 연계하여 추적 가능한 형태로 관리할 수 있다.
- 향후 스킬 형태로 자동화 가능한 워크플로우의 기반 시나리오로 활용할 수 있다.
