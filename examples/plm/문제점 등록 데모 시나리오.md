# 문제점 등록 데모 시나리오

## 1. 목적
사용자가 모바일 단말에서 발견한 문제 상황을 녹화하고, 해당 영상을 근거로 GitHub Issues에 문제점을 등록하는 데모를 수행한다.

## 2. 데모 흐름
1. 단말에서 테스트 또는 시연 중 문제 상황을 재현한다.
2. ADB `screenrecord` 기능으로 짧은 영상을 녹화한다.
3. 녹화 영상을 로컬 저장소의 `examples/plm/videos` 폴더에 저장한다.
4. 영상을 GitHub 저장소에 푸시한다.
5. GitHub PAT를 이용해 Issues에 문제점을 등록한다.
6. 이슈 본문에 녹화 영상 링크를 첨부한다.

## 3. 준비물
- 무선 또는 USB ADB 연결된 Android 단말
- GitHub 저장소: `KevinFire2030/GhostHand`
- 로컬 저장소 경로: `~/PRJs/GhostHand`
- GitHub PAT (`GITHUB_TOKEN`)
- 녹화 저장 폴더: `examples/plm/videos`

## 4. 상세 시나리오
### Step 1. 폴더 준비
- `examples/plm`
- `examples/plm/videos`

### Step 2. 화면 녹화
예시 명령:
```bash
adb shell screenrecord --time-limit 10 /sdcard/plm_10s.mp4
adb pull /sdcard/plm_10s.mp4 ~/PRJs/GhostHand/examples/plm/videos/plm_10s.mp4
adb shell rm /sdcard/plm_10s.mp4
```

### Step 3. GitHub 업로드
예시:
```bash
cd ~/PRJs/GhostHand
git add examples/plm/videos/plm_10s.mp4
git commit -m "Add 10-second PLM demo screen recording"
git push origin main
```

### Step 4. Issue 등록
제목 예시:
- 문제점 등록

본문 예시:
```md
문제점 등록 테스트

첨부 파일:
- https://github.com/KevinFire2030/GhostHand/blob/main/examples/plm/videos/plm_10s.mp4
```

### Step 5. 결과 확인
- GitHub Issues에 신규 이슈가 생성되었는지 확인한다.
- 이슈 본문에서 영상 링크가 정상적으로 열리는지 확인한다.

## 5. 기대 효과
- 문제 상황을 텍스트만이 아니라 영상으로 함께 전달할 수 있다.
- 재현 및 공유가 쉬워진다.
- 단말 녹화 → 저장소 반영 → 이슈 등록까지 하나의 흐름으로 시연할 수 있다.
