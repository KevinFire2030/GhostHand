# 기본 검증 시나리오

## 개요
본 문서는 PLM 데모를 위한 기본 검증 시나리오를 정의한다.
총 테스트 케이스(TC)는 5개이며, 현재 단말은 심카드가 삽입되지 않은 상태이므로 `전화 걸기`, `문자 보내기` 항목은 항상 FAIL 케이스로 처리하고 문제점 등록 대상에 포함한다.

## 테스트 케이스

### TC1. 전화 걸기
- 목적: 전화 앱 실행 및 발신 동작 확인
- 예상 결과: 심카드 미삽입 상태로 인해 정상 발신 불가
- 판정 기준: FAIL
- 비고: 문제점 등록 대상

### TC2. 문자 보내기
- 목적: 메시지 앱 실행 및 문자 발신 동작 확인
- 예상 결과: 심카드 미삽입 상태로 인해 정상 발신 불가
- 판정 기준: FAIL
- 비고: 문제점 등록 대상

### TC3. 홈화면 이동
- 목적: 현재 실행 중인 앱에서 홈화면으로 정상 이동하는지 확인
- 예상 결과: 홈화면으로 즉시 이동
- 판정 기준: PASS

### TC4. 유튜브 앱 실행
- 목적: 유튜브 앱이 정상 실행되는지 확인
- 예상 결과: 유튜브 앱 실행
- 판정 기준: PASS

### TC5. 알람설정
- 목적: 시계 앱에서 알람 설정이 가능한지 확인
- 예상 결과: 지정한 시간의 알람 설정 가능
- 판정 기준: PASS

## 참고
- FAIL 항목(`전화 걸기`, `문자 보내기`)은 검증 후 GitHub Issues에 문제점으로 등록한다.
- 실행 결과는 화면 캡처 및 화면 녹화로 증빙한다.
