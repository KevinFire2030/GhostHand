# 기본 검증 스킬

## 목적
PLM 데모 워크플로우를 자동화하기 위한 기본 검증 절차를 정의한다.
이 스킬은 검증 시나리오 작성, 실행, 증빙 수집, 결과 정리, FAIL 이슈 등록, 메신저 알림까지의 흐름을 하나의 작업 단위로 정리한 것이다.

## 입력
- 검증 대상 단말
- 기본 검증 시나리오 문서
- GitHub 저장소 경로
- GitHub PAT (`GITHUB_TOKEN`)
- 무선 또는 USB ADB 연결 상태

## 기본 검증 항목
1. 전화 걸기
2. 문자 보내기
3. 홈화면 이동
4. 유튜브 앱 실행
5. 알람설정

## 사전 조건
- `examples/plm/TC` 폴더가 생성되어 있어야 한다.
- 아래 하위 폴더가 준비되어 있어야 한다.
  - `TC/screencapture`
  - `TC/screenrecord`
- 현재 단말은 심카드 미삽입 상태이므로 `전화 걸기`, `문자 보내기`는 기본 FAIL 케이스로 처리한다.

## 수행 절차
### 1. 검증 시나리오 문서 작성
- `기본 검증 시나리오.md`를 작성한다.
- 테스트 케이스 5개를 정의한다.
- 시나리오 파일을 GitHub에 푸시한다.

### 2. 기본 검증 실행
- 각 TC를 순서대로 실행한다.
- 각 TC 결과를 화면 캡처한다.
  - 저장 위치: `TC/screencapture/TestCaseN.png`
- 각 TC 시작/종료 시 화면을 녹화한다.
  - 저장 위치: `TC/screenrecord/TestCaseN_start.mp4`
  - 저장 위치: `TC/screenrecord/TestCaseN_end.mp4`

### 3. 테스트 결과 정리
- `테스트 결과.md` 파일을 작성한다.
- 각 TC의 예상 결과, 실제 결과, 판정(PASS/FAIL), 비고를 기록한다.
- 결과 파일과 증빙 파일을 GitHub에 푸시한다.

### 4. FAIL 항목 이슈 등록
- FAIL 항목을 GitHub Issues에 등록한다.
- 기본 FAIL 케이스:
  - 전화 걸기 실패
  - 문자 보내기 실패
- 이슈 본문에는 아래를 포함한다.
  - 재현 경로
  - FAIL 사유
  - 시험 영상 URL 링크

### 5. 검증 결과 알림
- 검증 종료 후 텔레그램 메신저로 결과를 알린다.
- 알림에는 PASS/FAIL 요약과 등록된 이슈 번호를 포함한다.

## 산출물
- `TC/기본 검증 시나리오.md`
- `TC/screencapture/TestCase1.png` ~ `TestCase5.png`
- `TC/screenrecord/TestCase1_start.mp4` ~ `TestCase5_end.mp4`
- `TC/테스트 결과.md`
- GitHub Issues (FAIL 항목)
- 텔레그램 결과 알림

## 기대 효과
- 반복적인 기본 검증 절차를 정형화할 수 있다.
- PASS/FAIL 증빙을 자동으로 축적할 수 있다.
- FAIL 발생 시 즉시 GitHub Issues로 연계 가능하다.
- 메신저 알림까지 포함한 end-to-end 검증 자동화 기반을 마련할 수 있다.
