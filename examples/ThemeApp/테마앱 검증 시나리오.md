# 테마앱 검증 시나리오

대상: Galaxy Themes를 통해 다운로드 및 적용한 테마 앱
목적: 테마 설치 후 시인성, UI 적용 상태, 기본 사용성을 빠르게 검증한다.

---

## TC-1 홈 화면 적용 여부 확인
### 목적
- 테마가 홈 화면에 정상 적용되었는지 확인

### 절차
1. 홈 화면으로 이동한다.
2. 배경화면, 아이콘, 위젯 스타일을 확인한다.
3. 기본 테마와 다른 시각 요소가 적용되었는지 확인한다.

### 기대 결과
- 배경화면 또는 아이콘 스타일이 변경되어야 한다.
- 홈 화면이 깨지지 않고 정상 표시되어야 한다.

---

## TC-2 알림창 / 빠른 설정 UI 확인
### 목적
- 상단 알림창과 빠른 설정 영역에 테마가 정상 반영되는지 확인

### 절차
1. 상단 알림창을 내린다.
2. 빠른 설정 패널을 확장한다.
3. 시계, 날짜, 토글 버튼, 배경색 및 대비를 확인한다.

### 기대 결과
- 알림창과 빠른 설정 UI가 정상 표시되어야 한다.
- 글자와 아이콘이 식별 가능해야 한다.
- 색상 대비 문제로 가독성이 크게 떨어지지 않아야 한다.

---

## TC-3 설정 앱 가독성 확인
### 목적
- 리스트형 화면에서 테마 적용 후 텍스트 가독성을 확인

### 절차
1. 설정 앱을 실행한다.
2. 메뉴 목록과 아이콘을 확인한다.
3. 2~3회 스크롤하여 화면 깨짐 여부를 본다.

### 기대 결과
- 텍스트와 아이콘이 잘 보인다.
- 메뉴 간 구분이 가능하다.
- 스크롤 중 레이아웃 이상이 없다.

---

## TC-4 메시지/채팅 화면 가독성 확인
### 목적
- 실사용 화면에서 말풍선, 시간, 입력창이 잘 보이는지 확인

### 절차
1. 메시지 앱 또는 채팅 앱을 실행한다.
2. 받은 메시지, 보낸 메시지, 입력창, 시간 표시를 본다.

### 기대 결과
- 말풍선과 글자색 대비가 충분하다.
- 입력창과 보조 정보가 읽기 쉬워야 한다.

---

## TC-5 다크 모드 / 밝은 화면 호환성 확인
### 목적
- 테마가 다른 표시 환경에서도 안정적인지 확인

### 절차
1. 밝은 화면 기반 앱을 실행한다.
2. 가능하면 다크 모드 상태도 함께 확인한다.
3. 버튼, 제목, 본문 텍스트의 식별성을 확인한다.

### 기대 결과
- 밝은 화면과 어두운 화면 모두에서 주요 정보가 식별 가능해야 한다.
- 버튼, 탭, 선택 상태가 명확해야 한다.

---

## 결과 기록 방식
- 각 TC는 화면 캡처를 남긴다.
- 파일명 예시: `TestCase1.png`, `TestCase2.png`
- 실행 후 `테스트 결과.md`에 PASS / FAIL / 메모를 기록한다.
