# 테마앱 검증 시나리오

대상: Galaxy Themes를 통해 다운로드 및 적용한 테마 앱  
목적: 테마 설치 후 시인성, UI 적용 상태, 기본 사용성을 빠르게 검증한다.

---

## TC-1 홈 화면 적용 여부 확인
### 목적
- 테마가 홈 화면에 정상 적용되었는지 확인

### 절차
1. 홈 화면으로 이동한다.
2. 배경화면, 아이콘, 위젯 스타일을 확인한다.
3. 기본 테마와 다른 시각 요소가 적용되었는지 확인한다.

### 기대 결과
- 배경화면 또는 아이콘 스타일이 변경되어야 한다.
- 홈 화면이 깨지지 않고 정상 표시되어야 한다.

---

## TC-2 홈 화면 가독성 확인
### 목적
- 홈 화면에서 텍스트와 아이콘이 잘 보이는지 확인

### 절차
1. 홈 화면의 시계, 위젯 제목, 앱 이름을 본다.
2. 10초 이상 화면을 보며 가독성을 확인한다.

### 기대 결과
- 글자와 아이콘이 배경에 묻히지 않아야 한다.
- 장시간 봐도 눈의 피로가 과하지 않아야 한다.

---

## TC-3 앱 아이콘 식별성 확인
### 목적
- 아이콘팩 적용 시 앱 간 구분이 쉬운지 확인

### 절차
1. 홈 화면 또는 앱 화면에서 주요 앱 아이콘을 확인한다.
2. 전화, 메시지, 카메라, Chrome, Play 스토어 등 자주 쓰는 앱을 구분한다.

### 기대 결과
- 아이콘이 서로 지나치게 비슷하지 않아야 한다.
- 작은 크기에서도 주요 앱을 식별할 수 있어야 한다.

---

## TC-4 알림창 UI 확인
### 목적
- 상단 알림창에서 시계, 날짜, 알림 정보가 잘 보이는지 확인

### 절차
1. 상단 알림창을 내린다.
2. 알림 제목, 날짜, 상태 아이콘을 확인한다.

### 기대 결과
- 알림창의 정보가 정상 표시되어야 한다.
- 텍스트와 배경의 대비가 충분해야 한다.

---

## TC-5 빠른 설정 패널 UI 확인
### 목적
- 빠른 설정 패널의 버튼과 토글 상태가 잘 보이는지 확인

### 절차
1. 빠른 설정 패널을 확장한다.
2. 토글 버튼, 시계, 날짜, 배경색을 확인한다.

### 기대 결과
- 버튼 ON/OFF 상태가 명확해야 한다.
- 가독성과 식별성이 유지되어야 한다.

---

## TC-6 설정 앱 가독성 확인
### 목적
- 설정 앱 리스트형 화면에서 텍스트와 아이콘이 잘 보이는지 확인

### 절차
1. 설정 앱을 실행한다.
2. 메뉴 목록과 아이콘을 확인한다.
3. 2~3회 스크롤하여 화면 깨짐 여부를 본다.

### 기대 결과
- 텍스트와 아이콘이 잘 보여야 한다.
- 메뉴 간 구분이 명확해야 한다.
- 스크롤 중 레이아웃 이상이 없어야 한다.

---

## TC-7 메시지/채팅 화면 가독성 확인
### 목적
- 실사용 화면에서 말풍선, 시간, 입력창이 잘 보이는지 확인

### 절차
1. 메시지 앱 또는 채팅 앱을 실행한다.
2. 받은 메시지, 보낸 메시지, 입력창, 시간 표시를 확인한다.

### 기대 결과
- 말풍선과 글자색 대비가 충분해야 한다.
- 입력창과 보조 정보가 읽기 쉬워야 한다.

---

## TC-8 밝은 배경 앱 호환성 확인
### 목적
- 밝은 화면 기반 앱에서 제목, 버튼, 본문 텍스트가 잘 보이는지 확인

### 절차
1. Play 스토어 또는 브라우저를 실행한다.
2. 제목, 버튼, 탭, 본문 텍스트를 확인한다.

### 기대 결과
- 버튼이 배경에 묻히지 않아야 한다.
- 보조 텍스트가 지나치게 흐리지 않아야 한다.

---

## TC-9 다크 모드 호환성 확인
### 목적
- 다크 모드 환경에서 핵심 정보가 잘 보이는지 확인

### 절차
1. 다크 모드 상태를 확인한다.
2. 홈 화면, 설정, 알림창 또는 앱 화면을 본다.

### 기대 결과
- 어두운 배경에서도 텍스트와 아이콘이 식별 가능해야 한다.
- 강조색이 과하게 튀거나 사라지지 않아야 한다.

---

## TC-10 오조작 유발 여부 확인
### 목적
- 버튼, 탭, 선택 상태가 직관적으로 보이는지 확인

### 절차
1. 설정, 메시지 앱, 테마 앱 등에서 버튼과 탭을 확인한다.
2. 눌러야 할 요소와 장식 요소를 구분한다.

### 기대 결과
- 탭 가능한 요소가 명확하게 보인다.
- 선택된 상태와 선택되지 않은 상태가 구분된다.
- 오조작을 유발하지 않아야 한다.

---

## 결과 기록 방식
- 각 TC는 화면 캡처를 남긴다.
- 파일명 예시: `TestCase1.png`, `TestCase2.png`
- 실행 후 `테스트 결과.md`에 PASS / FAIL / 메모를 기록한다.
