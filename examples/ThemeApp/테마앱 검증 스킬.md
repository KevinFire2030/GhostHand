# 테마앱 검증 스킬

목적: Galaxy Themes 기반 무료 테마 검색, 설치, 적용, 검증, 결과 기록까지의 데모 워크플로우를 반복 가능하게 정리한다.

---

## 사용 목적
- 테마 앱 검증 데모 자동화
- 무료 테마 검색 및 설치 절차 표준화
- 화면 캡처 기반 결과 기록 자동화
- GitHub 반영까지 포함한 검증 흐름 재사용

---

## 입력 예시
- "PRJs 폴더에 ThemeApp 폴더를 만들어줘"
- "테마앱 검증 시나리오를 작성해줘"
- "무료 테마 검색해서 설치해줘"
- "검증 시나리오 실행해줘"
- "테스트 결과를 GitHub에 푸시해줘"

---

## 전체 워크플로우
### 1. 작업 폴더 준비
- `PRJs/ThemeApp` 폴더를 생성한다.

### 2. 검증 시나리오 문서 작성
- `테마앱 검증 시나리오.md` 파일을 작성한다.
- TC는 10개로 구성한다.
- 시나리오 문서를 GitHub `examples/ThemeApp`에 푸시한다.

### 3. GitHub 업로드 결과 확인
- GitHub에 업로드된 시나리오 파일을 확인한다.
- 시나리오의 핵심 검증 항목을 요약한다.

### 4. 테마 앱 실행
- Galaxy Themes 앱을 실행한다.

### 5. 무료 테마 검색 및 설치
- 테마 앱 검색 또는 무료 인기 태그를 사용해 무료 테마 목록으로 진입한다.
- 무료 테마 1개를 선택한다.
- 다운로드 후 적용까지 완료한다.

### 6. 검증 시나리오 실행
- 작성한 시나리오 기준으로 TC 10개를 수행한다.
- 각 TC마다 화면을 캡처한다.
- 캡처 파일은 `PRJs/ThemeApp/screencapture`에 `TestCase1.png` 형식으로 저장한다.

### 7. 테스트 결과 기록
- `테스트 결과.md` 파일에 각 TC 결과를 기록한다.
- 판정은 `PASS / FAIL / OBSERVE` 기준으로 정리한다.
- 요약 결론, 종합 평가, 최종 결론을 포함한다.

### 8. GitHub 반영
- 시나리오 문서, 테스트 결과 문서, 캡처 파일을 GitHub `examples/ThemeApp`에 푸시한다.

### 9. 결과 알림
- 검증이 끝나면 텔레그램 메신저로 결과 요약을 알린다.
- 필요 시 `테스트 결과.md` 경로를 함께 안내한다.

---

## 권장 검증 항목
1. 홈 화면 적용 여부
2. 홈 화면 가독성
3. 앱 아이콘 식별성
4. 알림창 UI
5. 빠른 설정 패널 UI
6. 설정 앱 가독성
7. 메시지/채팅 화면 가독성
8. 밝은 배경 앱 호환성
9. 다크 모드 호환성
10. 오조작 유발 여부

---

## 출력물
- `테마앱 검증 시나리오.md`
- `테스트 결과.md`
- `screencapture/TestCase1.png` ~ `TestCase10.png`
- GitHub 반영 결과

## 한 줄 요약
