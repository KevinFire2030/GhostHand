# 담당자.md

UT 검증 실패 시 1차 로그 분석 결과를 바탕으로 담당자 또는 담당 조직을 지정하기 위한 매핑 문서입니다.

## 목적

- FAIL 항목 발생 시 담당자 지정을 빠르게 하기 위함
- GitHub Issue assignee 및 PLM `inChargeUser` 후보를 정하기 위함
- 문제 유형별 담당 조직/담당자를 일관되게 적용하기 위함

## 담당자 매핑표

| 문제 유형 | 세부 키워드/조건 | 담당 조직 | 담당자 | 비고 |
|---|---|---|---|---|
| CRASH | `FATAL EXCEPTION`, `AndroidRuntime`, `SIGSEGV` | 앱 개발 | 홍길동 | 앱 비정상 종료 |
| ANR | `ANR in`, `Input dispatching timed out` | 플랫폼/프레임워크 | 홍길동 | 응답 없음/멈춤 |
| NETWORK | `SocketTimeoutException`, `UnknownHostException`, `HTTP 500` | 서버/API 연동 | 홍길동 | 네트워크/API 오류 |
| PERMISSION | `Permission Denial`, `SecurityException` | 플랫폼/권한 | 홍길동 | 권한/보안 정책 |
| CAMERA | `CameraService`, preview 실패, camera provider 오류 | 카메라 개발 | 홍길동 | 카메라 기능 문제 |
| AUDIO | 오디오 재생/녹음 관련 오류 | 오디오 개발 | 홍길동 | 스피커/마이크/미디어 |
| DISPLAY/UI | UI mismatch, 검은 화면, 흰 화면, 렌더링 이상 | UI/UX 또는 앱 개발 | 홍길동 | 화면 표시 문제 |
| TEST_SCRIPT | `selector mismatch`, `uiautomator timeout`, `adb input` 실패 | QA 자동화 | 홍길동 | 자동화 스크립트 문제 |
| DEVICE_ENV | SIM 미삽입, 네트워크 미연결, 계정 미로그인, 설정 미비 | 시험 환경/검증 운영 | 홍길동 | 환경 이슈 |
| UNKNOWN | 명확한 분류 불가 | Triage 담당 | 홍길동 | 1차 확인 필요 |

## 담당자 지정 규칙

1. 1차 로그 분석 결과를 먼저 확인한다.
2. 문제 유형을 위 매핑표 기준으로 분류한다.
3. 세부 키워드/조건이 일치하는 담당 조직과 담당자를 지정한다.
4. 명확하지 않으면 `UNKNOWN`으로 분류하고 Triage 담당에게 전달한다.
5. 복합 이슈인 경우 주 원인 기준으로 1명 지정하고, 필요하면 CC로 관련 담당자를 추가한다.

## 활용 예시

### 예시 1
- 현상: 전화 걸기 실패
- 조건: SIM 미삽입 상태
- 분류: `DEVICE_ENV`
- 담당자: 홍길동

### 예시 2
- 현상: 카메라 실행 후 preview 미노출
- 로그: `CameraService timeout`
- 분류: `CAMERA`
- 담당자: 홍길동

### 예시 3
- 현상: 앱 실행 중 강제 종료
- 로그: `FATAL EXCEPTION`
- 분류: `CRASH`
- 담당자: 홍길동

## 향후 확장

- 실제 조직/담당자명으로 업데이트
- GitHub assignee / PLM `inChargeUser` 계정과 연동
- 모듈별 서브 담당자까지 세분화
- 우선순위 및 CC 규칙 추가
