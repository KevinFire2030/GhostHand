# 직무발명(고안) 명세서(Invention Disclosure) 초안

## 1. 발명의 명칭 (Title of Invention)

### 국문
**동일 모바일 단말 내 인공지능 에이전트를 이용한 자연어 기반 자기 단말 제어 장치 및 방법**

### 영문
**Apparatus and Method for Natural Language-Based Self-Device Control Using an Artificial Intelligence Agent on the Same Mobile Terminal**

## 3. 발명의 배경

### 가. 본 발명의 기술 분야
본 발명은 모바일 단말 내부에서 실행되는 인공지능 에이전트가 사용자의 자연어 명령을 해석하고, 그 해석 결과에 따라 동일 모바일 단말의 애플리케이션, 운영체제 기능 또는 시스템 설정을 제어하는 기술 분야에 관한 것이다.  
보다 구체적으로는, 본 발명은 자연어 처리 기술, 온디바이스 인공지능 기술, 단말 제어 기술, 무선 디버깅 또는 로컬 브리지 기반 명령 전달 기술, 실행 결과 검증 기술 및 실패 시 대체 경로 선택 기술을 결합하여, 외부 PC 또는 별도 제어 장치에 의존하지 않고 자기 단말을 직접 제어할 수 있는 장치 및 방법에 관한 것이다.

### 나. 종래기술의 설명
종래의 모바일 단말 제어 기술은 주로 외부 PC, 서버 또는 별도 제어 장치를 통해 대상 단말을 제어하거나, 매크로 또는 사전 정의된 스크립트를 통해 반복 작업을 자동화하는 구조를 가진다. 예를 들어, ADB와 같은 디버깅 도구를 활용하여 앱 실행, 설정 변경 및 파일 전송을 수행하거나, 접근성 기반 자동화 도구를 이용하여 화면 조작을 수행하는 방식이 사용되어 왔다.

### 다. 종래기술의 문제점
종래기술은 다음과 같은 문제를 가진다.  
첫째, 외부 호스트 또는 별도 장치 의존성이 높아 동일 단말 단독으로 즉시 활용하기 어렵다.  
둘째, 자연어 명령을 직접 해석하여 제어 시퀀스를 생성하는 기능이 제한적이다.  
셋째, 명령 실행 후 목표 상태 도달 여부를 확인하는 검증 기능이 부족하다.  
넷째, 실행 실패 시 자동으로 다른 제어 수단을 선택하는 대체 경로 구조가 미흡하다.  
다섯째, 민감 작업에 대한 확인 절차, 권한 점검 및 보안 정책이 충분히 체계화되어 있지 않다.

### 라. 종래기술과 본 발명의 차이점
본 발명은 동일 모바일 단말 내에서 실행되는 인공지능 에이전트가 자연어 명령을 직접 해석하고, 자기 단말 식별, 정책 판단, 실행 경로 선택, 단말 제어, 실행 결과 검증, 실패 시 대체 경로 재선택을 수행하는 폐루프 제어 구조를 제공한다는 점에서 종래기술과 차별된다.  
특히, 본 발명은 외부 PC 없이도 자기 단말을 직접 제어할 수 있고, 민감 작업에 대해서는 추가 사용자 확인 절차를 적용할 수 있으며, 다양한 실행 경로를 조합하여 제어 신뢰성을 향상시킬 수 있다.

## 4. 발명의 구성 및 상세 설명

### 가. 발명의 핵심 구성 요소 및 동작 설명

#### 1) 입력부 및 AI 에이전트 실행부
입력부는 사용자의 텍스트 또는 음성 명령을 수신한다. AI 에이전트 실행부는 전체 제어 흐름을 관리하며, 자연어 해석부, 자기 단말 인식부, 정책 판단부, 실행 경로 선택부, 제어 실행부, 실행 결과 검증부 및 대체 경로 선택부를 연계하여 동작시킨다.

#### 2) 자연어 해석부
자연어 해석부는 사용자의 명령으로부터 의도를 분류하고, 앱명, 연락처명, 시간 정보, 설정 항목, 메시지 내용 등 실행 파라미터를 추출한다. 또한 추출 결과의 신뢰도를 산정하고, 필요 시 사용자 재확인을 요청할 수 있다.

#### 3) 자기 단말 인식부
자기 단말 인식부는 현재 제어 대상이 인공지능 에이전트가 실행 중인 동일 모바일 단말인지 여부를 확인하고, 단말 모델, 운영체제 버전, 설치 앱 목록, 권한 상태 및 단말 식별 토큰 등 제어에 필요한 상태 정보를 활용한다.

#### 4) 정책 판단부
정책 판단부는 해당 명령이 일반 작업인지 또는 민감 작업인지를 판정한다. 민감 작업은 전화 발신, 메시지 전송, 파일 삭제, 금융 애플리케이션 실행, 위치 정보 전송, 카메라 또는 마이크 활성화, 앱 설치 또는 삭제, 시스템 권한 변경 등을 포함할 수 있다. 민감도가 기준치 이상인 경우 추가 사용자 확인 절차를 수행할 수 있다.

#### 5) 실행 경로 선택부
실행 경로 선택부는 자연어 해석 결과 및 단말 상태를 바탕으로 적절한 실행 경로를 선택한다. 실행 경로는 운영체제 API 호출, 시스템 인텐트, 딥링크, 접근성 기반 UI 제어, 무선 디버깅 브리지 또는 로컬 브리지를 통한 명령 실행, 사전 정의된 UI 시나리오 등을 포함할 수 있다.

#### 6) 무선 디버깅 브리지부 또는 로컬 브리지부
무선 디버깅 브리지부 또는 로컬 브리지부는 AI 에이전트와 제어 실행부 사이에서 제어 명령 및 상태 정보를 중계하는 역할을 수행한다. 상기 브리지는 ADB 유사 디버깅 인터페이스, 로컬 소켓, IPC, 바인더, WebSocket 또는 로컬 HTTP를 포함하는 다양한 방식으로 구현될 수 있다.

#### 7) 제어 실행부
제어 실행부는 선택된 실행 경로에 따라 실제 앱 실행, 시스템 설정 변경, 메시지 전송, 전화 발신, 알람 설정, 파일 조작 또는 앱 설치/실행 등을 수행한다.

#### 8) 실행 결과 검증부
실행 결과 검증부는 제어 후 UI 상태 변화, 시스템 설정값, 포그라운드 앱 상태, 명령 출력값, 접근성 이벤트 또는 화면 인식 결과 등을 이용하여 목표 상태 도달 여부를 판단한다.

#### 9) 대체 경로 선택부
대체 경로 선택부는 목표 상태 미도달이 판정되면 다른 실행 경로를 선택하여 재시도를 수행한다. 이때 우선순위 테이블, 과거 성공 이력 또는 정책 조건을 반영할 수 있다.

#### 10) 권한 및 보안 관리부
권한 및 보안 관리부는 명령 수행에 필요한 권한을 사전 점검하고, 부족한 권한이 존재하는 경우 사용자에게 승인 또는 권한 부여 절차를 안내한다. 또한 민감 정보 로그에 대한 마스킹, 로컬 암호화 저장 및 오프라인 동작 지원을 포함할 수 있다.

#### 11) 개발자 모드 기반 통합 제어
일 실시예에 따르면, 개발자 모드가 활성화된 경우 동일 모바일 단말 내에서 앱의 빌드, 설치, 실행 및 테스트 제어를 통합적으로 수행할 수 있다.

### 3) 복합 명령 처리 예시
사용자가 하나의 자연어 명령으로 복수 작업을 요청하는 경우, 자연어 해석부는 이를 복수의 하위 작업으로 분해할 수 있다. 예를 들어, “메시지 보내고 7시에 알람 맞춰줘”라는 명령은 메시지 전송과 알람 설정으로 분해될 수 있으며, 각 하위 작업에 대해 개별적인 실행 경로 선택, 결과 검증 및 재시도가 수행될 수 있다.

## 6. 도면의 간단한 설명

- **도 1. 자연어 기반 자기 단말 제어 시스템 구성도**  
  동일 모바일 단말 내에서 실행되는 인공지능 에이전트가 사용자의 자연어 명령을 해석하고, 단말 제어 및 결과 검증, 대체 경로 선택을 수행하는 전체 시스템 구성을 도시한 도면.

- **도 2. 자연어 기반 자기 단말 제어 방법의 기본 동작 순서도**  
  사용자 명령 수신 단계부터 자연어 해석, 정책 판단, 실행 경로 선택, 단말 제어, 실행 결과 검증, 재시도 및 결과 피드백에 이르는 기본 동작 절차를 도시한 도면.

- **도 3. 민감 작업에 대한 사용자 확인 절차 흐름도**  
  자연어 명령이 민감 작업에 해당하는지 여부를 판정하고, 추가 사용자 확인 또는 인증 절차를 거쳐 실행 여부를 결정하는 과정을 도시한 도면.

- **도 4. 복합 명령 처리 절차 흐름도**  
  하나의 자연어 명령을 복수의 하위 작업으로 분해하고, 각 하위 작업에 대해 실행 및 검증을 수행하는 절차를 도시한 도면.

## 7. 회피 방안 관점

### 1) 입력 방식 변형에 대한 포섭
타사가 문자 기반 자연어 입력 대신 음성, 제스처, 화면 객체 선택 또는 이미지 기반 입력을 사용하는 경우에도, 사용자 의도를 추출하여 자기 단말 제어 명령으로 변환하는 구조를 포섭할 수 있다.

### 2) 브리지 구조 변형에 대한 포섭
타사가 ADB 대신 Bluetooth, 로컬 IPC, 전용 실행 인터페이스 또는 접근성 기반 실행기를 사용하는 경우에도, 에이전트와 제어 실행부 사이에서 명령과 상태를 중계하는 브리지 구조 전반을 포섭할 수 있다.

### 3) 검증 구조 변형에 대한 포섭
타사가 UI 검증 대신 상태 코드, 로그, 접근성 이벤트 또는 화면 인식 기반 검증 방식을 사용하더라도, 실행 결과 확인 및 실패 시 재실행을 포함하는 폐루프 구조를 포섭할 수 있다.

## 8. 침해 적발 방법

### 1) 기능 테스트 기반 적발
자연어 명령 입력 후 앱 실행, 설정 변경, 전화 발신, 메시지 전송, 실패 시 재시도 여부 등을 테스트하여 자연어 해석-실행-검증-재시도 구조의 존재를 확인할 수 있다.

### 2) 로그 및 동작 분석 기반 적발
실행 로그, 접근성 이벤트, 시스템 호출 흔적, 포그라운드 앱 전환 및 UI 반응을 분석하여 실행 결과 검증 및 대체 경로 선택 구조의 존재를 판단할 수 있다.

### 3) 구조 분석 기반 적발
애플리케이션 패키지, 권한 사용 패턴, 로컬 통신 구조 또는 디버깅 인터페이스 사용 흔적을 분석하여 자연어 해석부, 정책 판단부, 실행부 및 검증부의 결합 구조를 확인할 수 있다.

## 9. 발명의 효과

### 1) 사용자 편의성 향상
사용자는 복잡한 메뉴 탐색이나 스크립트 작성 없이 자연어 명령만으로 동일 모바일 단말의 다양한 기능을 직관적으로 제어할 수 있다.

### 2) 외부 장치 의존성 감소
외부 PC 또는 별도 제어 장치 없이 동일 모바일 단말 내에서 직접 제어가 가능하므로 독립성과 이동성이 향상된다.

### 3) 제어 신뢰성 향상
실행 결과 검증 및 대체 경로 재선택을 포함하는 폐루프 제어 구조를 통해 제어 성공률을 향상시킬 수 있다.

### 4) 보안성 및 안전성 향상
민감 작업 판정, 사용자 확인 절차 및 권한 점검 구조를 통해 오동작과 보안 위험을 줄일 수 있다.

### 5) 확장성 확보
다양한 앱, 운영체제 기능, 제어 경로 및 향후 단말 환경에 유연하게 적용 가능하다.

### 6) 개발 및 운영 자동화 가능
개발자 모드에서는 앱의 빌드, 설치, 실행 및 테스트 제어를 동일 모바일 단말 내에서 통합적으로 수행할 수 있다.

사용자로부터 자연어 명령을 수신하는 입력부;  
상기 자연어 명령을 해석하여 의도 및 실행 파라미터를 추출하는 자연어 해석부;  
상기 장치가 제어 대상인 자기 단말인지 여부 및 단말 상태를 확인하는 자기 단말 인식부;  
상기 자연어 명령의 민감도 또는 보안 필요성을 판정하는 정책 판단부;  
상기 자연어 명령의 해석 결과 및 단말 상태에 기초하여 제어 실행 경로를 선택하는 실행 경로 선택부;  
선택된 실행 경로에 따라 상기 모바일 단말의 앱 또는 시스템 기능을 제어하는 제어 실행부;  
상기 제어 결과를 검증하여 목표 상태 도달 여부를 판단하는 실행 결과 검증부; 및  
상기 목표 상태 미도달 시 대체 경로를 선택하여 재실행하는 대체 경로 선택부를 포함하는,
