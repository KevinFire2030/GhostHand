# 변리사 제출 패키지 v3 (최종 단일본)

## 과제명
**동일 모바일 단말 내 인공지능 에이전트를 이용한 자연어 기반 자기 단말 제어 장치 및 방법**

---

## 1) 발명의 명칭
**동일 모바일 단말 내 인공지능 에이전트를 이용한 자연어 기반 자기 단말 제어 장치 및 방법**

---

## 2) 요약(Abstract)
본 발명은 모바일 단말 내부에서 실행되는 인공지능 에이전트가 사용자의 자연어 명령을 해석하고, 로컬 실행 환경 및 무선 디버깅 브리지를 이용하여 그 동일 모바일 단말을 제어하는 장치 및 방법에 관한 것이다. 상기 장치 및 방법은 사용자 명령 수신, 의도 분석, 제어 대상 결정, 실행 경로 선택, 앱 실행 또는 시스템 제어 수행, UI 상태·단말 상태·명령 출력 기반 결과 검증, 실패 시 대체 경로 선택 및 재시도, 그리고 결과 피드백 제공을 포함한다. 또한 동일 단말 내에서 애플리케이션 빌드, 설치, 실행 및 후속 단말 제어까지 통합 수행할 수 있어, 외부 PC 없이도 온디바이스 AI 기반 자기 단말 제어 및 운영 자동화를 구현할 수 있다.

---

## 3) 기술적 효과
- 동일 단말 내부 AI에 의한 **자기 단말 제어 구현**
- 외부 PC 없이 **온디바이스 실행 및 운영 자동화 가능**
- 자연어 명령을 실제 단말 제어 시퀀스로 변환
- 앱 실행, 설정 이동, 빠른설정 토글 등 **실제 단말 동작 자동화**
- 실행 결과 검증 및 fallback/retry를 통한 **신뢰성 향상**
- 앱 빌드·설치·실행·제어를 하나의 단말 내에서 통합
- 사용자 친화적 대화형 인터페이스 제공
- 개인 단말 기반 AI 오퍼레이터 레이어 구현 가능

---

## 4) 도면의 간단한 설명
- **도 1:** 시스템 구성도  
- **도 2:** 사용자 명령 수신 → 해석 → 제어 실행 → 검증 → 반환 순서도  
- **도 3:** 동일 단말 무선 디버깅 연결 및 제어 흐름도  
- **도 4:** 실행 실패 시 대체 제어 경로 선택 흐름도  
- **도 5:** 앱 빌드·설치·실행·제어 통합 루프도  
- **도 6:** 메시지/전화/설치 등 민감 작업에 대한 사용자 확인 경계 구조도  
- **도 7:** 명령 실행 결과 피드백 및 로그 구조도  

---

## 5) 도면 상세 설명

### 도 1. 시스템 구성도(상세)
도 1은 자기 단말 제어 시스템(100)이 사용자 명령 입력부(110), 자연어 해석 모듈(120), 실행 계획 생성 모듈(130), 로컬 실행 환경 모듈(140), 무선 디버깅 제어 모듈(150), 상태 검증 모듈(160), 대체 경로 선택 모듈(170), 결과 피드백 모듈(180)을 포함하고, 동일 모바일 단말(200)의 애플리케이션 계층 및 시스템 계층과 연결되는 구조를 나타낸다.

### 도 2. 전체 처리 순서도(상세)
(1) 사용자 명령 수신 → (2) 자연어 의도 분석 → (3) 제어 대상 및 제어 방식 결정 → (4) 로컬 명령 또는 무선 디버깅 명령 생성 → (5) 앱 실행/시스템 제어 수행 → (6) UI 상태, 단말 상태, 명령 출력값 검증 → (7) 성공 시 결과 반환 → (8) 실패 시 대체 경로 선택 및 재시도 → (9) 최종 결과 보고 순으로 동작한다.

### 도 3. 동일 단말 무선 디버깅 연결 흐름도(상세)
모바일 단말 내부 실행 환경에서 동일 단말에 대한 무선 디버깅 연결을 수립하고, 이를 통해 앱 실행, 설정 이동, 빠른설정 토글, 입력 수행 등의 제어 명령을 전달하는 과정을 도시한다. 이에 따라 외부 PC 없이 동일 단말 제어가 가능해진다.

### 도 4. 대체 제어 경로 선택 흐름도(상세)
제1 제어 방식이 실패하면, 제2 제어 방식으로 전환한다. 예를 들면, 패키지 기반 앱 실행이 실패하면 인텐트 호출로 전환하고, 인텐트 호출이 실패하면 UI 탐색 또는 좌표 기반 조작으로 전환할 수 있다. 이를 통해 제어 성공률을 향상시킨다.

### 도 5. 앱 빌드·설치·실행·제어 통합 루프도(상세)
동일 단말 내에서 애플리케이션 소스 작성, APK 빌드, APK 설치, 앱 실행 및 이후의 UI/기능 제어까지 하나의 루프로 수행하는 구조를 나타낸다. 이는 단말이 제어 대상일 뿐 아니라 개발·운영 플랫폼으로도 기능함을 보여준다.

### 도 6. 민감 작업 사용자 확인 경계 구조도(상세)
전화 발신, 문자 전송, 메신저 메시지 전송, 애플리케이션 설치 등의 민감 작업에 대해서는 준비 단계와 최종 실행 단계를 분리한다. 준비 단계는 자동 수행될 수 있으나, 외부 상대방에게 영향을 주는 최종 실행 단계는 사용자 확인 이후 수행되도록 구성된다.

### 도 7. 결과 피드백 및 로그 구조도(상세)
결과 피드백 모듈은 실행한 명령, 대상 앱 또는 시스템 기능, 실행 성공 여부, 실패 사유, 대체 경로 적용 여부 등을 사용자에게 제공한다. 또한 선택적으로 실행 로그를 저장하여 반복 가능한 자동화 흐름 구축에 활용할 수 있다.

## 6) 청구항(초안 1~10)

### 청구항 1 (독립항: 방법)
모바일 단말 내부의 실행 환경에서 인공지능 에이전트를 실행하는 단계;  
사용자의 자연어 명령을 수신하는 단계;  
상기 자연어 명령에 기초하여 동일 모바일 단말에 대한 제어 동작을 결정하는 단계;  
무선 디버깅 인터페이스 또는 로컬 실행 경로를 통해 상기 제어 동작을 수행하는 단계;  
상기 제어 동작의 실행 결과를 단말 상태, UI 상태 또는 명령 출력값을 이용하여 검증하는 단계; 및  
상기 검증 결과에 따라 실행 성공 결과를 반환하거나 대체 제어 경로를 선택하여 재시도하는 단계를 포함하는, 동일 모바일 단말 자기제어 방법.

### 청구항 2
청구항 1에 있어서, 상기 제어 동작은 앱 실행, URL 열기, 시스템 설정 이동, 빠른설정 기능 제어, 전화 화면 호출, 문자 작성 화면 호출 중 적어도 하나를 포함하는 방법.

### 청구항 3
청구항 1에 있어서, 상기 자연어 명령에 대한 해석은 사용자 의도 분류, 제어 대상 식별, 실행 경로 선택을 포함하는 방법.

### 청구항 4
청구항 1에 있어서, 상기 검증 단계는 UI 덤프 정보, 포그라운드 앱 정보, 패키지 상태 정보, 명령 출력 정보 중 적어도 하나를 이용하는 방법.

### 청구항 5
청구항 1에 있어서, 제1 제어 방식이 실패할 경우 제2 제어 방식으로 전환하는 fallback 절차를 포함하는 방법.

### 청구항 6
청구항 5에 있어서, 상기 fallback 절차는 패키지 실행, 인텐트 호출, UI 탐색, 좌표 기반 제어 중 적어도 둘 이상을 순차적으로 적용하는 방법.

### 청구항 7
청구항 1에 있어서, 상기 모바일 단말 내부 실행 환경은 애플리케이션 빌드, 설치, 실행 및 후속 단말 제어를 통합적으로 수행하는 방법.

### 청구항 8
청구항 1에 있어서, 전화 발신, 메시지 전송, 메신저 전송, 애플리케이션 설치와 같은 민감 작업에 대해 준비 단계와 최종 실행 단계를 분리하고, 최종 실행은 사용자 확인 후 수행하는 방법.

### 청구항 9
청구항 1에 있어서, 상기 자연어 명령은 채팅 인터페이스를 통해 입력되는 방법.

### 청구항 10 (독립항: 장치)
사용자 명령 입력부, 자연어 해석 모듈, 실행 계획 생성 모듈, 로컬 실행 환경 모듈, 무선 디버깅 제어 모듈, 상태 검증 모듈, 대체 경로 선택 모듈 및 결과 피드백 모듈을 포함하고, 청구항 1 내지 9 중 어느 한 항의 방법을 수행하는 동일 모바일 단말 자기제어 장치.

## 8) 거절 대응 예상 Q&A (5)

### Q1. AI와 단말 제어의 단순 결합 아닌가?
**A1.** 본 발명은 단순한 AI 명령 실행이 아니라, 동일 단말 내부 실행, 무선 디버깅 브리지, 결과 검증, fallback/retry, 앱 빌드·설치·실행·제어 통합이라는 기능적 상호작용 구조를 가진다.

### Q2. 모바일 자동화는 이미 알려진 기술 아닌가?
**A2.** 기존 모바일 자동화는 외부 PC 제어, 단순 매크로, 또는 서버 기반 제어가 중심이며, 동일 단말 내부 AI 에이전트가 자기 단말을 자연어 기반으로 제어하는 구조와는 차이가 있다.

### Q3. ADB 기반 제어는 공지기술 아닌가?
**A3.** 무선 디버깅 자체는 공지일 수 있으나, 이를 동일 단말 내부 AI 에이전트, 자연어 해석, 결과 검증 및 fallback 구조와 결합한 자기제어 아키텍처는 차별적이다.

### Q4. 앱 실행이나 설정 이동은 자명하지 않은가?
**A4.** 개별 동작 자체보다, 자연어 해석 → 실행 경로 선택 → 결과 검증 → 대체 경로 재시도까지 이어지는 폐루프 구조가 핵심이다.

### Q5. 산업상 이용 가능성이 약한가?
**A5.** 본 발명은 개인 AI 에이전트, 모바일 자동화, 온디바이스 개발/운영, 접근성 보조, 현장용 단말 운영 자동화 등에 적용 가능하여 산업상 이용 가능성이 충분하다.

## 10) 변리사 검토 요청 포인트
- 독립항 범위를 방법/장치/기록매체까지 확장할지 여부  
- “동일 단말 자기제어”를 핵심 청구항 문구로 얼마나 강하게 넣을지 여부  
- 무선 디버깅 브리지 표현을 일반화할지 구체화할지 여부  
- 결과 검증 및 fallback/retry를 독립항에 포함할지 종속항으로 둘지 여부  
- 앱 빌드·설치·실행·제어 통합 루프를 핵심 청구항으로 올릴지 여부  
- 민감 작업의 사용자 확인 경계를 안전성 요소로 포함할지 여부  
- 해외 출원(PCT) 시 same-device self-control 표현의 정합화 여부
