# 온디바이스 AI Agent 단말 검증 장점 5가지

온디바이스 AI Agent를 활용한 단말 검증 방식의 장점을 기존 방식(As-Is)과 비교해 정리합니다.

## 5가지 장점

| No. | 장점 | As-Is (기존 방법) | To-Be (이 방법) | 비고 |
|---|---|---|---|---|
| 1 | 실패 원인 1차 분류 자동화 | 테스트 실패 후 사람이 영상/로그를 직접 확인하며 원인을 추정해야 함 | 단말 내 AI Agent가 로그, 화면, 시나리오 맥락을 바탕으로 실패 유형을 1차 자동 분류 | Crash, ANR, Network, Permission, UI, Test Script 등으로 초기 triage 가능 |
| 2 | 담당자 지정 속도 향상 | 이슈를 누가 봐야 할지 사람이 판단해야 해서 전달 지연 또는 오배정 발생 | 분석 결과를 바탕으로 기능 영역/모듈/키워드 기준 담당자 후보를 즉시 추천 | GitHub assignee, PLM `inChargeUser` 후보 자동화에 유리 |
| 3 | 검증 증거 수집 품질 향상 | 영상, 로그, 스크린샷, 단말 정보가 흩어져 저장되거나 누락될 수 있음 | 시나리오별로 영상, 로그, 메타정보, 핵심 에러 요약을 구조적으로 묶어서 저장 | 재현, 리뷰, 감사, 보고 시 증거 일관성 향상 |
| 4 | 문제 등록 품질 향상 | GitHub Issue / PLM 등록 내용을 사람이 매번 수동 작성해야 해서 품질 편차가 큼 | AI Agent가 실패 요약, 재현 경로, 추정 원인, 핵심 로그를 바탕으로 초안을 자동 생성 | 등록 속도와 문서 품질을 동시에 개선 |
| 5 | triage 및 의사결정 효율 향상 | 모든 실패를 비슷한 수준으로 다루기 쉬워 노이즈가 많고 우선순위 판단이 늦어짐 | 실패 유형, 재현성, 영향 범위, 로그 signature를 기준으로 등록 필요 여부와 우선순위 판단을 보조 | 중복 이슈 감소, 실제 중요한 건 우선 대응 가능 |

## 요약

온디바이스 AI Agent 기반 단말 검증은 단순히 "AI가 분석해준다" 수준이 아니라, 아래 흐름 전체를 개선합니다.

- 기존 방법:
  - 테스트 수행
  - 실패 확인
  - 사람이 로그/영상 확인
  - 담당자 판단
  - 이슈/PLM 수동 등록

- 개선 방법:
  - 테스트 수행
  - 실패 감지
  - 온디바이스 AI Agent가 1차 분석
  - 담당자 후보 추천
  - GitHub/PLM 초안 자동 생성

즉, 기존의 수동 확인 중심 방식에서, 분석/분류/전달/등록이 연결된 자동화 방식으로 전환된다는 점이 핵심입니다.

## 한 줄 정리

온디바이스 AI Agent 단말 검증의 핵심 장점은 **실패 분석, 담당자 추천, 증거 수집, 문제 등록, triage 판단**을 하나의 흐름으로 묶어 기존 방식보다 더 빠르고 정확하게 운영할 수 있다는 점입니다.
