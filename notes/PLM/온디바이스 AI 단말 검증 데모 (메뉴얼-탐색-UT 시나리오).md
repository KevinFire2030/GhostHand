# 온디바이스 AI 단말 검증 데모 (메뉴얼-탐색-UT 시나리오)

## 1. 목적

메시지 입력 기반으로 메뉴얼 - 탐색 - UT 검증 시나리오를 작성하고, 검증 실행 결과를 캡처/녹화/정리한 뒤, FAIL 항목은 GitHub Issues에 등록하고 최종 결과를 메신저로 알리는 데모를 수행한다.

---

## 2. 메시지 입력 시나리오

### 2.1 시험 준비

#### 2.1.1 시험 폴더 및 결과 저장 폴더 생성

- `examples/Demo3` 폴더 아래 다음 구조를 생성한다.
  - `UT` 폴더 생성 → 시나리오 저장
  - `runs` 폴더 생성 → 시험 산출물(로그, 스크린샷, 녹화, 시험 결과 등) 저장

예시:

```text
examples/Demo3/
  UT/
    UT 검증 시나리오.md

  UT/runs/
    2026-04-18/
      UT-001_login/
        screen.mp4
        logcat.txt
        meta.json
        result.md
        screenshots/
        issue.md
      UT-002_camera_launch/
        screen.mp4
        logcat.txt
        meta.json
        result.json
```

##### 2.1.2.2 테스트 케이스(TC) 5개 작성

아래 참고 시나리오와 기본 시나리오 예시를 참조하여 작성한다.

## 3. 시험 실행

### 3.1 로그 초기화

- 시험 시작 전 로그를 초기화하거나, 최소한 시나리오 시작 시각을 기록한다.

### 3.2 작성한 시나리오 기준으로 기본 UT 검증 실행

#### 3.2.1 각 TC의 실행 결과 화면 캡처

- 캡처 파일은 `UT/runs/screenshots/` 폴더에 저장한다.
- 예: `TestCase1.png`

#### 3.2.2 각 TC의 실행 화면 녹화

- 시작할 때 녹화를 시작한다.
- 종료 시 녹화를 종료한다.
- 녹화 파일은 `UT/runs/` 폴더에 저장한다.
- 예: `TestCase1.mp4`

#### 3.2.3 각 TC의 로그 파일 저장

- 시작할 때 로깅을 시작한다.
- 종료 시 로깅을 종료한다.
- 로그 파일은 `UT/runs/` 폴더에 저장한다.
- 예: `logcat.txt`

#### 3.2.4 테스트 결과 저장

- 캡처 화면 / 영상 / 로그를 바탕으로 시험 결과를 판단(판정)한다.
- 결과를 `UT/runs/` 폴더에 저장한다.
- `PASS / FAIL` 판정을 기록한다.
- 사유는 최대한 자세히 기록한다.
  - 화면/영상/로그 분석 결과 바탕으로 최대한 자세히 작성
  - 특히 FAIL일 경우 아래 항목을 반드시 포함
    - 1차 로그 분석 결과 기록
    - 담당자 지정 (`담당자.md` 파일 참조하여 해당 문제의 담당자 지정)

예:
- `result.md`

#### 3.2.5 결과 파일 GitHub에 푸시

## 4. FAIL 항목 이슈 등록

### 4.1 FAIL 항목이 발생한 경우 GitHub Issues에 문제점 등록

특히 심카드 미삽입 상태로 인해 실패가 예상되는 **전화 걸기**, **문자 보내기** 항목은 문제점 등록 데모의 기본 FAIL 케이스로 활용한다.

등록 예시:

- 제목: `전화 걸기 실패`

- 내용:
  - 재현 경로
  - FAIL 사유
  - 1차 로그 분석 결과
  - 담당자: 홍길동
  - 첨부파일: 파일 URL 링크

### 4.2 검증 결과 알림

- 검증이 끝나면 결과를 담당자에게 텔레그램 메신저로 통보한다.

---

## 5. 스킬 추가

이 데모 워크플로우를 자동화할 수 있도록 스킬 형태로 정리한다.

- 방금 실행한 데모를 `기본 UT 검증 스킬.md`로 저장한다.
- 해당 파일을 GitHub에 푸시한다.

---

## 6. Cron Job 등록

저장한 UT 검증 스킬이 매시간 정시에 실행될 수 있게 Cron Job으로 등록한다.

---

## 한 줄 정리

이 데모는 **메시지 입력 → UT 시나리오 생성 → 실행 결과 수집 → FAIL 분석 및 담당자 지정 → GitHub Issues 등록 → 메신저 알림 → 스킬화 → Cron 자동 실행**까지의 전체 흐름을 보여주는 온디바이스 AI 단말 검증 시나리오다.
