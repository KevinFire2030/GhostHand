# 로그 저장 방법

스마트폰 기능 검증 테스트(예: UT 시나리오) 수행 시, 시나리오별로 화면 녹화와 단말 로그를 함께 저장하고, 검증 실패 시 문제 로그를 GitHub Issue로 올리기 위한 방법을 정리합니다.

## 결론

- ADB로 단말 로그 추출 가능
- 시나리오별 영상 + 로그 저장 가능
- 검증 실패 시 핵심 로그를 추려 GitHub Issue 자동 등록 가능
- 이후 PLM defect 초안 자동 생성 흐름으로도 확장 가능

## ADB로 추출 가능한 항목

대표적으로 아래 항목을 수집할 수 있습니다.

- `logcat`
  - 앱 로그
  - 시스템 로그
  - crash / ANR / exception
- `bugreport`
  - 종합 진단 덤프
- `dumpsys`
  - 현재 앱 상태
  - activity/window/package 상태
- `screenrecord`
  - 화면 녹화
- `screencap`
  - 스크린샷
- `getprop`
  - 단말 정보
  - 빌드/모델/OS 버전
- 이벤트 로그
  - 필요 시 `event`, `radio`, `main`, `system`, `crash` 버퍼 분리

## 기본 명령 예시

```bash
adb logcat
adb logcat -b all
adb shell screenrecord /sdcard/test.mp4
adb bugreport
adb shell dumpsys activity
adb shell getprop
```

## 추천 운영 흐름

시나리오 하나 실행할 때마다 아래 순서로 처리합니다.

1. 시나리오 시작 전
   - 로그 초기화 또는 기준 시각 기록
   - 녹화 시작
   - `logcat` 저장 시작
2. 시나리오 실행
3. 시나리오 종료 후
   - 녹화 종료
   - `logcat` 종료
   - 결과 판정
4. 실패 시
   - 관련 로그 추출
   - 주요 에러 grep
   - 영상/스크린샷/로그 경로 정리
   - GitHub Issue 생성

이 방식의 장점은 테스트 증거가 시나리오 단위로 남는다는 점입니다.

## 디렉터리 구조 예시

```text
runs/
  2026-04-18/
    UT-001_login/
      screen.mp4
      logcat.txt
      meta.json
      result.json
      screenshots/
      issue.md
    UT-002_camera_launch/
      screen.mp4
      logcat.txt
      meta.json
      result.json
```

### 파일 역할

- `screen.mp4` : 테스트 영상
- `logcat.txt` : 전체 로그
- `meta.json` : 단말/빌드/시나리오 정보
- `result.json` : pass/fail, 실패 시점
- `issue.md` : GitHub 이슈 본문 초안

## 시나리오별 로그 저장 방법

핵심은 시나리오 시작 시점부터 로그를 분리 저장하는 것입니다.

### 방법 1. 가장 쉬운 방식: 로그 버퍼 초기화 후 저장

```bash
adb logcat -c
adb logcat > logcat.txt
```

- `-c`: 기존 로그 버퍼 비우기
- 그 뒤 시나리오 실행
- 종료 후 프로세스 종료

장점:
- 시나리오 로그만 따로 남기기 쉬움

주의:
- 다른 중요한 로그까지 비워질 수 있으므로 운영 환경에서는 신중히 사용해야 함

### 방법 2. 더 안전한 방식: timestamp 기준 저장

```bash
adb logcat -v time > full_logcat.txt
```

그리고 시나리오 메타에 아래를 기록합니다.

- `startTime`
- `endTime`

나중에 해당 시간 구간만 잘라 사용합니다.

장점:
- 로그 유실 위험이 적음
- 동시에 여러 분석이 가능함
- 실무 환경에서 더 안정적임

## 실패 시 문제 로그 추리기

실패한 경우 전체 로그 원본과 핵심 에러 요약본을 분리하는 것이 좋습니다.

### 예시 키워드

- `FATAL EXCEPTION`
- `ANR`
- `CRASH`
- `AndroidRuntime`
- `System.err`
- 앱 패키지명 태그
- 특정 네트워크 오류
- permission 오류

### 예시 명령

```bash
grep -E "FATAL EXCEPTION|ANR|AndroidRuntime|System.err|Crash" logcat.txt
```

또는 패키지 기준 필터:

```bash
grep "com.example.app" logcat.txt
```

추천 구성:

- `logcat.txt` : 원본 전체 로그
- `error-summary.txt` : 실패 핵심만 추린 요약 로그

## GitHub Issue 자동 등록

실패 시 아래 정보를 모아 GitHub Issue를 자동 생성할 수 있습니다.

- 제목 자동 생성
- 재현 경로 자동 작성
- 로그 요약 첨부
- 영상 파일 경로/링크 첨부
- 스크린샷 첨부

### 예시 이슈 본문

```md
Title: [UT][FAIL] Camera launch scenario - preview not shown

## Summary
카메라 앱 실행 후 5초 내 preview가 표시되지 않음

## Scenario
UT-002_camera_launch

## Device
- Model: SM-...
- OS: Android 14
- Build: ...

## Reproduction
1. 카메라 앱 실행
2. 권한 허용
3. preview 확인
4. 실제로는 검은 화면 유지

## Evidence
- Video: runs/2026-04-18/UT-002_camera_launch/screen.mp4
- Log: runs/2026-04-18/UT-002_camera_launch/logcat.txt

## Error Summary
- FATAL EXCEPTION ...
- Camera service timeout ...
```

## 중요한 주의사항

GitHub Issue에 전체 로그를 그대로 전부 올리는 것은 비추천입니다.

이유:
- 개인정보/민감정보 포함 가능
- 토큰/URL/단말 식별자 포함 가능
- 로그가 너무 길어짐

권장 방식:
- Issue에는
  - 요약
  - 핵심 로그 일부
  - 첨부 경로/아티팩트 링크
- 전체 로그는
  - 로컬 보관
  - 사내 저장소 보관
  - 필요 시만 별도 첨부

즉,

- `Issue = 요약/인덱스`
- `원본 로그 = 별도 아티팩트`

## 영상 + 로그 + 판정 자동화 구조

### 1. 시나리오 실행기
- 시나리오 시작
- ADB `screenrecord` 시작
- ADB `logcat` 저장 시작
- 테스트 자동 수행

### 2. 판정기
- 예상 화면 도달 여부
- OCR/로그/결과 코드 기반 pass/fail 판정

### 3. 실패 수집기
- 실패 시점 전후 영상 구간 저장
- 핵심 로그 추출
- 스크린샷 저장
- 단말 정보 저장

### 4. 리포터
- GitHub issue 초안 생성
- 필요 시 자동 등록
- 나중에 PLM defect 초안으로도 변환 가능

## 실무 추천 명령 조합

### 1) 단말 정보 저장

```bash
adb shell getprop > device-props.txt
```

### 2) 로그 저장

```bash
adb logcat -v threadtime > logcat.txt
```

### 3) 화면 녹화

```bash
adb shell screenrecord /sdcard/test.mp4
```

종료 후:

```bash
adb pull /sdcard/test.mp4 ./screen.mp4
```

### 4) crash 중심 로그

```bash
adb logcat -b crash -v threadtime > crash.txt
```

`crash` 버퍼 저장은 crash/ANR 중심 확인에 유용합니다.

## 시나리오 메타 정보 추천

시나리오별로 아래 메타 정보를 함께 저장하면 이후 자동화가 쉬워집니다.

```json
{
  "scenarioId": "UT-002",
  "scenarioName": "camera_launch",
  "startTime": "2026-04-18T06:10:00+09:00",
  "endTime": "2026-04-18T06:11:15+09:00",
  "deviceSerial": "ABC123",
  "model": "SM-XXXX",
  "androidVersion": "14",
  "appPackage": "com.example.app",
  "result": "FAIL"
}
```

이 메타가 있으면 아래 작업이 쉬워집니다.

- 로그 자동 정리
- 이슈 자동 작성
- 나중에 PLM 연동

## 전하 목적 기준 추천안

### 빠름
- 시나리오별 `screen.mp4` + `logcat.txt` 저장
- 실패 시 수동으로 issue 작성

### 실무
- 실패 시 `issue.md` 자동 생성
- `gh issue create`로 등록

### 완성형
- 실패 시
  - 영상
  - 에러 로그
  - 스크린샷
  - 메타정보
  - 요약문
  자동 생성 후
  - GitHub Issue
  - PLM defect 초안
  둘 다 생성

## 한 줄 정리

- ADB로 단말 로그 추출 가능
- 시나리오별 영상 + 로그 저장 가능
- 검증 실패 시 핵심 로그를 추려 GitHub Issue 자동 등록 가능
- 이 구조는 UT 데모 흐름과 잘 맞고, 이후 PLM defect 자동화까지 확장 가능
