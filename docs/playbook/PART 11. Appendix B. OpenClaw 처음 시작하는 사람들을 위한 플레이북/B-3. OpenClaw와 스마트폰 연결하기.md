# B-3. OpenClaw와 스마트폰 연결하기

OpenClaw를 설치하고 WebUI에서 첫 대화까지 성공했다면, 그다음으로 체감이 커지는 단계가 있다.  
바로 스마트폰과의 실제 연결이다. 여기서 말하는 연결은 단순히 같은 기기에서 앱을 여는 수준이 아니라, OpenClaw가 스마트폰을 더 깊이 이해하고 제어할 수 있는 기반을 만드는 과정에 가깝다.

이 장의 핵심은 **Wireless ADB**다.  
처음 듣는 사람에게는 조금 낯설게 느껴질 수 있지만, 개념 자체는 단순하다. 스마트폰을 PC나 다른 실행 환경과 무선으로 연결해, 명령을 주고받을 수 있게 만드는 방법이라고 생각하면 된다. OpenClaw나 Ghost Hand 같은 실행형 구조를 생각한다면, 이 연결은 매우 중요하다. 왜냐하면 여기서부터 AI가 단순히 말만 잘하는 존재가 아니라, 실제 스마트폰 환경과 연결되는 존재가 되기 때문이다.

이 장에서는 초심자도 따라갈 수 있도록 다음 순서로 설명한다.

1. Wireless ADB란 무엇인가  
2. 스마트폰에서 개발자 옵션과 무선 디버깅 켜기  
3. 페어링 준비  
4. Termux에서 `adb pair` 실행  
5. Termux에서 `adb connect` 실행  
6. 연결 확인

---

## 1. Wireless ADB란 무엇인가

ADB는 Android Debug Bridge의 줄임말이다.  
쉽게 말하면 안드로이드 기기와 외부 환경이 명령을 주고받을 수 있게 해주는 연결 도구다. 원래는 개발자나 테스트 환경에서 많이 쓰이지만, 모바일 AI 에이전트 관점에서는 훨씬 흥미로운 의미를 가진다. 스마트폰을 단순 소비 기기가 아니라, 실제로 제어 가능한 실행 환경으로 바꾸는 연결 다리 역할을 하기 때문이다.

여기서 Wireless ADB는 말 그대로 케이블 없이 무선으로 ADB 연결을 하는 방식이다.  
USB 케이블을 꽂지 않아도 같은 네트워크 안에서 스마트폰과 연결할 수 있다는 뜻이다. 초심자 입장에서는 이게 왜 중요한지 처음엔 잘 안 와닿을 수 있다. 하지만 한 번 연결이 되면, 스마트폰 화면을 켜고 끄거나, 앱을 열고, 특정 동작을 자동화하는 구조로 확장할 수 있다. 즉 Wireless ADB는 OpenClaw와 스마트폰 실행 계층 사이를 이어주는 핵심 기반 중 하나다.

---

## 2. 스마트폰에서 개발자 옵션과 무선 디버깅 켜기

Wireless ADB를 쓰려면 먼저 스마트폰 쪽 준비가 되어 있어야 한다.  
핵심은 개발자 옵션과 무선 디버깅을 활성화하는 것이다.

### 2.1 개발자 옵션 켜기
보통 안드로이드 스마트폰에서는 다음 흐름으로 들어간다.
- **설정** 열기
- **휴대전화 정보** 또는 **디바이스 정보**로 이동
- **빌드 번호**를 여러 번 눌러 개발자 옵션 활성화

제조사와 버전에 따라 메뉴 이름이 조금 다를 수 있지만, 핵심은 “개발자 옵션” 메뉴를 활성화하는 것이다.

### 2.2 무선 디버깅 켜기
개발자 옵션이 열리면, 그 안에서 **무선 디버깅** 항목을 찾는다.  
이 기능을 켜야 Wireless ADB 연결이 가능해진다.

보통 흐름은 다음과 비슷하다.
- **설정**
- **개발자 옵션**
- **무선 디버깅** 켜기

여기까지 되면 스마트폰 쪽 기본 준비는 된 것이다.

---

## 3. 페어링 준비

무선 디버깅을 켠 뒤에는 보통 페어링(pairing) 단계가 필요하다.  
이 과정은 쉽게 말해 “내가 지금 연결하려는 환경이 신뢰할 수 있는 대상인지”를 스마트폰이 확인하는 절차다. 초심자는 이 단계에서 IP 주소와 포트 번호, 페어링 코드 같은 것이 갑자기 나와 당황할 수 있다. 하지만 구조는 생각보다 단순하다.

스마트폰의 무선 디버깅 화면 안에는 보통 다음 같은 정보가 보인다.
- 페어링용 IP 주소
- 페어링용 포트 번호
- 페어링 코드
- 연결용 IP 주소/포트 정보

중요한 것은 이 정보를 잘 보고, Termux에서 입력할 준비를 하는 것이다.

---

## 4. Termux에서 adb 설치 준비

`adb pair`나 `adb connect`를 쓰려면 Termux 안에 adb 도구가 있어야 한다.  
환경에 따라 이미 준비되어 있을 수도 있지만, 없다면 먼저 설치해야 한다.

Termux에서 아래처럼 설치할 수 있다.

```bash
pkg install -y android-tools
```

설치가 잘 되었는지 확인하려면 아래처럼 버전을 확인할 수 있다.

```bash
adb version
```

출력이 나오면 준비가 된 것이다.

---

## 5. Termux에서 adb pair 실행

이제 실제 페어링 단계다.  
스마트폰에서 확인한 **페어링용 IP:포트번호**를 사용해 Termux에서 아래 명령을 입력한다.

```bash
adb pair IP:포트번호
```

예를 들면 이런 형태다.

```bash
adb pair 192.168.0.10:37123
```

그러면 Termux가 **pairing code** 입력을 요구할 수 있다.  
이때 스마트폰 화면에 보이는 코드를 그대로 입력하면 된다.

페어링이 성공하면, 이제 이 Termux 환경은 해당 스마트폰과 신뢰 관계를 맺게 된다. 즉 다음 단계인 실제 연결로 넘어갈 수 있다.

---

## 6. Termux에서 adb connect 실행

페어링이 끝났다고 해서 곧바로 모든 연결이 끝난 것은 아니다.  
이제는 실제 연결용 주소를 사용해 접속해야 한다. 스마트폰의 무선 디버깅 화면에서 **연결용 IP:포트번호**를 확인한 뒤, 아래 명령을 실행한다.

```bash
adb connect IP:포트번호
```

예를 들면 이런 식이다.

```bash
adb connect 192.168.0.10:37655
```

이 단계가 성공하면, Termux 환경에서 스마트폰과 실제 ADB 연결이 이루어진다.

---

## 7. 연결 확인

연결이 잘 되었는지 확인하려면 아래 명령을 사용할 수 있다.

```bash
adb devices
```

정상적으로 연결되었다면, 연결된 기기 목록에 스마트폰 정보가 보이게 된다.  
이 출력이 확인되면, Wireless ADB 연결은 성공한 것이다.

이제부터는 단순 연결 확인을 넘어, 실제로 다양한 실행형 흐름으로 확장할 수 있다. 예를 들어 앱 실행, 입력 자동화, 화면 제어, 테스트 흐름, Ghost Hand 같은 구조와의 연결 가능성이 열리기 시작한다.

---

## 연결이 안 될 때 체크할 것

초보자가 가장 많이 막히는 구간도 바로 여기다.  
따라서 아래 항목을 차례로 확인해보면 좋다.

- 스마트폰과 Termux 환경이 같은 네트워크에 있는가
- 무선 디버깅이 켜져 있는가
- 페어링용 주소와 연결용 주소를 헷갈리지 않았는가
- pairing code를 정확히 입력했는가
- `adb`가 Termux에 정상 설치되어 있는가

특히 **pair용 포트**와 **connect용 포트**가 다를 수 있다는 점은 초심자가 자주 헷갈리는 부분이다.  
이 둘을 섞어 쓰면 연결이 안 될 수 있다.

---

## 핵심 요약

Wireless ADB 연결의 핵심 흐름은 아래와 같다.

1. 스마트폰에서 개발자 옵션 켜기  
2. 무선 디버깅 켜기  
3. 페어링 정보 확인  
4. Termux에서 adb 설치  
5. `adb pair IP:포트번호` 실행  
6. `adb connect IP:포트번호` 실행  
7. `adb devices`로 연결 확인

실전에서 최소 흐름만 보면 이렇게 정리할 수 있다.

```bash
pkg install -y android-tools
adb pair IP:포트번호
adb connect IP:포트번호
adb devices
```

결국 이 연결은 단순한 기술 설정이 아니다.  
이 단계가 열리면 OpenClaw는 더 이상 단순히 텍스트로만 대화하는 플랫폼이 아니라, 스마트폰 실행 환경과 맞닿을 수 있는 기반을 갖추게 된다. 그리고 바로 그 지점에서, 모바일 AI 에이전트는 진짜로 ‘움직이는 존재’가 되기 시작한다.
