# B-2. 스마트폰에 OpenClaw 설치하기

OpenClaw를 처음 시작하려는 사람에게 가장 큰 장벽은 보통 기능이 아니라 설치다.  
“이거 좋아 보이는데, 내 스마트폰에선 어떻게 시작하지?”

바로 이 질문에서 많은 초심자가 멈춘다. 그래서 설치 과정은 단순한 준비 단계가 아니라, OpenClaw와의 첫 관계를 만드는 시작점이라고 볼 수 있다. 이 장에서는 안드로이드 스마트폰 기준으로 OpenClaw를 실제로 시작하기 위해 어떤 순서로 환경을 준비하면 되는지, 그리고 어떤 명령을 입력해야 하는지까지 따라 하기 쉽게 정리한다.

OpenClaw를 스마트폰에서 쓰기 위해서는 보통 다음 흐름을 거치게 된다.

1. Termux 설치  
2. Termux 초기 준비  
3. OpenClaw 설치  
4. OpenClaw 온보드와 기본 설정  
5. LLM 연결  
6. Search/Skill 같은 확장 요소 이해  
7. Boot 구조 이해  
8. WebUI에서 첫 대화하기

이 순서를 따라가면, OpenClaw는 더 이상 낯선 개발자 도구가 아니라 실제로 켜보고 만져볼 수 있는 나만의 에이전트 플랫폼으로 느껴지기 시작한다.

---

## 1. Termux 설치

안드로이드 스마트폰에서 OpenClaw를 시작할 때 가장 먼저 필요한 것은 Termux다.  
Termux는 쉽게 말해 스마트폰 안에 작은 리눅스형 작업 환경을 열어주는 앱이다. 일반 앱이 아니라, 명령어 기반으로 도구를 설치하고 실행할 수 있는 작업 공간이라고 이해하면 된다. OpenClaw를 스마트폰에서 다루기 위해서는 이 환경이 사실상 출발점이 된다.

### 어디서 받는가
가장 많이 권장되는 방법은 **F-Droid**를 통해 Termux를 설치하는 것이다.  
Play 스토어의 Termux는 최신 상태가 아닐 수 있어서, 입문자는 보통 F-Droid 쪽을 더 많이 쓴다.

기본 흐름은 이렇다.
- 스마트폰 브라우저에서 **F-Droid** 공식 사이트 접속
- F-Droid 앱 설치
- F-Droid 안에서 **Termux** 검색
- 설치

### 어떻게 실행하는가
설치가 끝나면 일반 앱처럼 Termux를 실행하면 된다.  
처음 열면 검은 화면과 함께 프롬프트가 보일 수 있는데, 그게 정상이다. 놀랄 필요 없다. 여기서부터 명령어를 입력해 환경을 준비하면 된다.

---

## 2. Termux 초기 준비

Termux를 처음 실행했다면, 바로 OpenClaw를 깔기 전에 먼저 기본 패키지를 최신 상태로 맞춰주는 것이 좋다.  
초보자는 이 단계에서 “뭘 벌써 업데이트하지?” 싶을 수 있지만, 이는 스마트폰 안 작업 환경을 준비하는 아주 기본적인 과정이다.

아래 명령을 순서대로 입력한다.

```bash
pkg update && pkg upgrade -y
```

이 명령은 패키지 목록을 새로 받고, 기본 도구들을 최신 상태로 맞춰준다.

그다음 OpenClaw 설치에 필요한 기본 도구를 준비한다.

```bash
pkg install -y nodejs git
```

환경에 따라 추가 도구가 필요할 수 있지만, 처음 시작할 때는 보통 이 정도가 가장 기본이다.

설치가 잘 되었는지 확인하고 싶다면 아래처럼 버전을 확인해볼 수 있다.

```bash
node -v
git --version
```

숫자나 버전 문자열이 보이면 정상이다.

---

## 3. OpenClaw 설치

Termux가 준비되면 그다음은 OpenClaw 설치다.  
이 단계는 말 그대로 플랫폼 본체를 가져오는 과정이다. 초심자 입장에서는 이때부터 비로소 “이제 진짜 시작하는구나”라는 감각이 생긴다.

OpenClaw는 보통 npm을 통해 설치한다. Termux에서 아래 명령을 입력한다.

```bash
npm install -g openclaw
```

설치가 끝난 뒤에는 아래 명령으로 정상 설치 여부를 확인할 수 있다.

```bash
openclaw --version
```

버전이 출력되면 설치가 성공한 것이다.

만약 설치 중 권한이나 네트워크 이슈가 생기면, 대부분은 네트워크 상태나 패키지 갱신 문제인 경우가 많다. 이럴 때는 무리하게 여러 명령을 섞기보다, 먼저 `pkg update`와 `pkg upgrade`가 제대로 끝났는지부터 다시 확인하는 편이 좋다.

---

## 4. OpenClaw 설정 (온보드)

설치가 끝났다고 해서 바로 다 준비되는 것은 아니다.  
OpenClaw를 실제로 쓰려면 자신의 환경에 맞게 최소한의 설정을 해야 한다. 이때 중요한 절차가 바로 온보드(onboard)다. 온보드는 OpenClaw가 처음 시작될 때 필요한 기본 환경을 잡아주는 과정으로 이해하면 된다.

Termux에서 아래 명령으로 온보드를 시작한다.

```bash
openclaw onboard
```

이 과정에서는 보통 다음 같은 항목들을 차례로 맞추게 된다.
- 기본 프로필 또는 사용자 환경
- 사용할 모델 관련 설정
- 기본 연결 정보
- 이후 WebUI나 채널 연결을 위한 기초 정보

초심자에게 온보드는 조금 추상적으로 느껴질 수 있다.  
하지만 쉽게 말하면 “내 OpenClaw를 처음 살아 움직이게 만드는 설정 단계”라고 보면 된다. 여기서 기본 프로필, 연결 정보, 초기 환경 구성이 잡히고, 이후 사용 경험의 기초가 마련된다.

즉 온보드는 설치와 활용 사이를 이어주는 다리다.  
초심자일수록 이 단계를 “귀찮은 설정”으로 보기보다, 내 에이전트 환경의 첫 초기화라고 이해하는 것이 좋다.

---

## 5. LLM 연결

OpenClaw가 실제로 대화하고, 요약하고, 초안을 만들고, 판단을 돕기 위해서는 결국 LLM이 필요하다.  
즉 OpenClaw는 에이전트 플랫폼이고, LLM은 그 안에서 사고와 언어 생성을 담당하는 두뇌라고 볼 수 있다. 그래서 설치 후 가장 중요한 설정 중 하나가 바로 어떤 LLM을 연결할 것인가다.

초보자는 이 지점에서 종종 헷갈린다.  
OpenClaw 자체가 AI인 줄 알았는데, 왜 또 모델을 연결해야 하는지 의문이 들 수 있다. 하지만 구조를 이해하면 쉽다. OpenClaw는 AI를 운영하는 플랫폼이고, 실제 답변과 생성은 연결된 모델이 맡는다.

실전에서는 보통 API 키를 환경 변수나 설정 파일에 넣는 방식으로 연결한다.  
예를 들어 OpenAI 계열 모델을 쓴다면 보통 `OPENAI_API_KEY` 같은 값이 필요하다. 이 부분은 온보드 과정이나 설정 파일에서 이어질 수 있다.

입문자 관점에서 중요한 것은 다음이다.
- 플랫폼과 모델은 다르다.
- 먼저 하나의 모델만 안정적으로 연결해보는 것이 좋다.
- “최고의 모델 찾기”보다 “내 환경에서 실제로 응답이 나오게 만들기”가 먼저다.

---

## 6. Search 설정

LLM만 연결하면 대화는 가능하지만, 더 실제적인 도움을 받으려면 검색 기능도 중요하다.  
Search는 OpenClaw가 외부 정보를 참고하거나 웹 자료를 찾아볼 수 있게 해주는 역할을 한다. 즉 모델이 가진 일반 지식만으로 답하는 것이 아니라, 더 최신 정보나 외부 맥락을 활용할 수 있게 하는 확장축이다.

초심자 입장에서는 Search를 처음부터 복잡하게 이해할 필요는 없다.  
지금 단계에서 중요한 것은 “OpenClaw는 단순히 머릿속 지식만으로 답하는 것이 아니라, 필요하면 외부 정보를 참고하도록 확장될 수 있다”는 감각이다.

---

## 7. Skill 설정

OpenClaw의 강력한 특징 중 하나는 Skill 구조다.  
Skill은 쉽게 말해 특정 작업을 더 잘 수행하도록 도와주는 전문 능력 묶음이다. 예를 들어 날씨, 음성, 자동화, 특정 API 활용, 문서 처리처럼 어떤 목적에 맞게 에이전트 행동을 더 정교하게 만들어준다.

초보자에게 중요한 것은 스킬을 당장 많이 붙이는 것이 아니다.  
오히려 “OpenClaw는 나중에 내가 원하는 작업을 더 잘하게 확장할 수 있는 구조를 가지고 있다”는 점을 이해하는 것이 먼저다. 처음에는 기본 상태로 시작하고, 익숙해진 뒤 필요한 스킬을 하나씩 붙여가면 된다.

---

## 8. Boot 이해

Boot는 초심자에게 가장 낯선 단어 중 하나일 수 있다.  
쉽게 말하면 OpenClaw가 켜질 때 어떤 맥락과 초기 상태로 시작하는가와 관련된 개념이라고 이해하면 된다. 즉 단순히 프로그램이 실행되는 것을 넘어서, 어떤 환경과 기억, 어떤 기본 태도로 시작할 것인가를 다루는 층이 있는 셈이다.

이 개념은 처음부터 완벽히 이해할 필요는 없다.  
다만 초심자 입장에서는 “OpenClaw는 매번 빈 상태로 뜨는 앱이라기보다, 일정한 맥락과 설정을 가지고 동작할 수 있는 구조다” 정도로 이해하면 충분하다.

---

## 9. WebUI에서 첫 대화하기

설치와 설정의 가장 중요한 마무리는 결국 첫 대화다.  
아무리 구조를 잘 이해해도, 실제로 말을 걸어보고 응답을 받아보지 않으면 체감은 생기지 않는다. 그래서 WebUI에서 첫 대화를 성공시키는 순간이 매우 중요하다.

보통 OpenClaw를 실행하면 로컬 WebUI 주소가 함께 안내된다. 환경에 따라 다르지만, 시작은 보통 아래처럼 한다.

```bash
openclaw
```

혹은 설치된 환경에 따라 gateway 또는 web 관련 실행 흐름을 따를 수 있다. 중요한 것은 실행 후 출력되는 안내 메시지를 잘 보는 것이다.  
거기에서 접속 주소나 포트 정보가 보일 수 있다.

브라우저에서 WebUI에 접속한 뒤, 첫 대화는 거창할 필요가 없다.  
오히려 아주 단순한 요청이 좋다.

- 자기소개 시키기
- 오늘 할 일 정리 요청하기
- 짧은 문서 요약 시키기
- 간단한 질문 던져보기

이런 첫 대화의 목적은 성능 평가가 아니다.  
내 환경에서 OpenClaw가 실제로 동작하고 있다는 감각, 그리고 앞으로 여기에 무엇을 붙여갈 수 있을지에 대한 감각을 얻는 것이다. 첫 대화가 성공하면, 설치는 끝난 것이 아니라 비로소 시작된 것이다.

---

## 빠르게 따라 하는 최소 설치 요약

아래 흐름만 따라가도 최소 시작은 가능하다.

```bash
pkg update && pkg upgrade -y
pkg install -y nodejs git
npm install -g openclaw
openclaw onboard
openclaw
```

이후에는
- 온보드 진행
- 모델 연결 확인
- WebUI 접속
- 첫 대화 실행

까지 가면 된다.

---

## 설치의 핵심은 완벽함이 아니라 첫 성공 경험이다

이 장에서 설명한 흐름은 언뜻 보면 단계가 많아 보일 수 있다.  
하지만 실제 핵심은 단순하다.

- Termux로 기반을 만들고
- OpenClaw를 설치하고
- 온보드와 기본 설정을 하고
- LLM, Search, Skill 같은 확장 가능성을 이해하고
- WebUI에서 첫 대화를 성공시키는 것

즉 스마트폰에 OpenClaw를 설치한다는 것은 단순히 프로그램 하나를 올리는 일이 아니다.  
그것은 내 스마트폰 위에, 앞으로 점점 더 나에게 맞춰 성장할 수 있는 에이전트 환경의 첫 바닥을 까는 일이다. 그리고 초심자에게 가장 중요한 것은 처음부터 모든 걸 이해하는 것이 아니라, **한 번 실제로 켜보고, 한 번 실제로 대화해보는 것**이다. 거기서부터 OpenClaw는 낯선 플랫폼이 아니라 현실의 도구가 된다.
