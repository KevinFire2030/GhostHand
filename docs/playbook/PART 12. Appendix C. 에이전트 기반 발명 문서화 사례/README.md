# PART 12. Appendix C. 에이전트 기반 발명 문서화 사례

이 부록은 에이전트 프로젝트가 실제 발명 문서와 제도적 산출물로 어떻게 이어질 수 있는지를 보여주는 실전 사례 파트다.  
앞선 본문과 부록들이 개념, 구축, 활용, 전환 플레이북, OpenClaw 입문을 다뤘다면, Appendix C는 그 결과물이 어떻게 발명 문서화와 직무발명 신고서 같은 형식으로 연결될 수 있는지를 다룬다.

핵심 질문은 다음과 같다.  
**에이전트 프로젝트와 AX 실험은 어떻게 발명 문서로 정리되고, 조직 안의 제도적 자산으로 전환될 수 있는가?**

---

## 구성
- C-1. 왜 발명 문서화가 중요한가
- C-2. 직무발명 신고서 초안 사례
- C-3. 이 문서의 핵심 포인트
- C-4. 에이전트 프로젝트를 발명 문서로 바꾸는 법

---

## 이 부록의 핵심 메시지

프로젝트는 구현으로 끝나지 않는다.  
잘 정리된 문서는 곧 자산이 되고, 발명 문서는 그 자산을 조직과 제도 안에서 보존하고 확장하는 방식이 된다. 즉 Appendix C는 실험이 어떻게 기록되고, 기록이 어떻게 발명으로 구조화되는지를 보여주는 부록이다.
