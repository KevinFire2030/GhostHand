# PART 10. Appendix A. 에이젠틱 전환 플레이북

이 부록은 책 본문에서 설명한 개념과 구조를 실제 실행 단계로 옮기기 위한 **실천형 플레이북**이다.  
앞선 파트들이 모바일 AI 에이전트의 개념, 플랫폼, 자동화, 업무 활용, 미래를 설명했다면, Appendix A는 독자가 실제로 자신의 삶과 조직 안에서 에이전틱 전환을 시작할 때 무엇부터 해야 하는지를 안내한다.

핵심 질문은 다음과 같다.  
**모바일 AI 에이전트를 이해하는 것을 넘어, 실제로 도입하고 정착시키려면 무엇부터 시작해야 하는가?**

이 부록은 기술 설명서라기보다 실행 지침서에 가깝다.  
즉 “무엇이 가능한가”보다 “어떻게 시작하고, 어디까지 확장하며, 무엇을 조심해야 하는가”에 초점을 둔다.

---

## 권장 구성

### A-1. 에이젠틱 전환이란 무엇인가
에이젠틱 전환을 단순한 AI 도입이 아니라, 삶과 업무의 구조를 에이전트 중심으로 다시 설계하는 변화로 정의한다.

### A-2. 현재 수준 진단표
개인 또는 조직이 현재 어느 단계에 와 있는지 점검할 수 있는 레벨별 진단표를 제시한다.

예시 단계:
- Level 0. AI 미활용
- Level 1. 단발성 생성형 AI 사용
- Level 2. 반복 업무 보조 자동화
- Level 3. 문서/지식 연결형 에이전트
- Level 4. 실행형 모바일 에이전트
- Level 5. 삶/업무 통합형 에이전트 운영

### A-3. 개인용 30일 전환 로드맵
한 달 안에 작은 자동화부터 시작해 실제 에이전트 활용 루틴을 만드는 단계별 계획을 제안한다.

### A-4. 팀/조직용 전환 플레이북
개인 수준을 넘어, 팀 또는 조직이 어떻게 에이전트 기반 업무 방식을 정착시킬 수 있는지 다룬다.

### A-5. 무엇부터 자동화할 것인가
초기 도입 시 가장 효과가 큰 반복 업무부터 선정하는 기준과 우선순위를 설명한다.

예:
- 메일 요약
- 회의록 자동화
- 일정 정리
- 아침 브리핑
- 정보 수집
- 문서 초안 생성

### A-6. 무엇은 자동화하지 말아야 하는가
고위험 자동화 영역과 반드시 인간 승인 또는 검토가 필요한 영역을 정리한다.

예:
- 금전 거래 전면 자동화
- 민감한 외부 커뮤니케이션 자동 발송
- 승인 없는 고위험 의사결정 실행
- 검증 없는 의료/법률/인사 판단

### A-7. 에이전트 설계 체크리스트
좋은 에이전트를 만들기 위해 반드시 확인해야 할 설계 질문들을 정리한다.

예:
- 누구를 돕는가
- 어떤 반복을 줄이는가
- 어떤 데이터를 참고하는가
- 어디까지 자동 실행하는가
- 어디서 인간 승인을 받는가
- 실패 시 어떻게 중단하는가

### A-8. 성공 패턴과 실패 패턴
도입이 잘 되는 경우와 실패하는 경우를 비교해, 전환 과정에서 흔히 빠지는 함정을 보여준다.

### A-9. 바로 시작하는 최소 실행안
복잡한 구축 없이도 지금 당장 시작할 수 있는 최소한의 실행 시나리오를 제안한다.

예:
- 메신저 기반 요약 비서 만들기
- 회의록 자동 생성 흐름 붙이기
- RAG 기반 개인 문서 검색 만들기
- 하루 1개 루틴 자동화부터 시작하기

## 이 부록의 핵심 메시지

Appendix A의 핵심은 단순하다.  
에이전틱 전환은 거대한 프로젝트가 아니라, **작은 반복을 줄이는 것에서 시작해 점진적으로 삶과 업무 구조를 바꾸어가는 과정**이라는 점이다.

즉 이 부록은 독자에게 이렇게 묻는다.  
“당신은 무엇을 가장 먼저 에이전트에게 맡기고 싶은가?”

그 질문에 답하는 순간, 에이젠틱 전환은 더 이상 추상적인 미래 이야기가 아니라 현실의 실행 계획이 된다.
