# PART 3. AX 프레임워크 설계

이 파트는 모바일 AI 에이전트를 단순히 구현하는 수준을 넘어, **어떤 방법론과 운영 구조로 설계할 것인가**를 다룬다.  
PART 1이 왜 모바일 AI 에이전트가 필요한지를 설명하고, PART 2가 그것을 어떻게 구축하는지를 설명했다면, PART 3는 그것을 **어떤 프레임워크로 기획하고, 어떤 스택으로 개발하고, 어떤 방식으로 배포·운영하며, 어떻게 자동화 전략으로 살아 움직이게 할 것인가**를 설명하는 파트다.

핵심 질문은 다음과 같다.  
**AI 에이전트를 기능 모음이 아니라 지속적으로 운영 가능한 AX 플랫폼으로 만들려면, 어떤 설계 프레임과 실행 전략이 필요한가?**

---

## 10장 요약. AX 기획 방법론

10장은 AX를 단순한 기술 조합이 아니라, 사용자 문제를 중심으로 설계해야 하는 경험 체계로 바라본다.  
여기서는 디자인 씽킹, 5D 프로세스, AX 서비스 기획이라는 세 가지 관점을 통해 에이전트 서비스 기획의 출발점을 설명한다.

핵심 내용은 다음과 같다.
- AX 설계의 출발점은 기술이 아니라 사용자 문제다.
- 디자인 씽킹은 자동화 가능성보다 먼저 사용자의 맥락과 위임 욕구를 이해하게 해준다.
- 5D 프로세스(Define–Discover–Design–Develop–Deploy)는 AX 서비스를 단계적으로 검증하는 실전 프레임이다.
- AX 서비스 기획은 화면 중심 기획이 아니라 목표와 위임 흐름 중심 기획이어야 한다.

즉 이 장은 “무엇을 만들까”보다 “왜 이 경험이 필요한가”와 “어떻게 설계할까”를 다루는 전략적 장이다.

---

## 11장 요약. 개발 스택

11장은 모바일 AI 에이전트를 실제로 구현할 때 어떤 개발 스택이 적합한지를 설명한다.  
여기서는 Python, Node.js, 그리고 APK/WebView 전략을 중심으로 각각의 역할을 분리해서 본다.

핵심 내용은 다음과 같다.
- Python은 AI 연동, 데이터 처리, 빠른 프로토타이핑, Web UI/API 구현에 강하다.
- Streamlit은 운영 화면과 검증용 Web UI에 적합하고, FastAPI는 서비스 레이어와 API 노출에 적합하다.
- Node.js는 이벤트 기반 처리, CLI 생태계, 플랫폼 연결, 타입스크립트 기반 구조화에 강하다.
- APK/WebView 전략은 에이전트를 실제 모바일 제품 형태로 확장하는 실용적 방법이다.

즉 이 장은 개발 스택을 취향의 문제가 아니라, 역할 분담의 문제로 바라봐야 한다는 점을 강조한다.

---

## 12장 요약. 배포와 운영

12장은 에이전트를 실제 가치로 바꾸는 단계인 배포와 운영을 다룬다.  
좋은 시스템을 만드는 것과, 그것을 안정적으로 사용자의 일상 속에서 돌리는 것은 전혀 다른 문제라는 점을 보여준다.

핵심 내용은 다음과 같다.
- Web 배포는 가장 빠른 진입성과 유연한 운영성을 제공한다.
- APK 배포는 제품성과 접근성을 높이고, 에이전트를 사용자 일상에 더 가깝게 붙인다.
- AaaS(Agent as a Service)는 에이전트를 하나의 서비스 단위로 재정의하는 관점이다.
- n8n 자동화는 에이전트 실행을 시간/이벤트 흐름과 연결해 살아 있는 운영 구조로 만든다.

즉 이 장은 “에이전트를 만들었다”를 넘어, “에이전트를 실제로 굴린다”는 것이 무엇인지 설명한다.

---

## 13장 요약. 자동화 전략 (Vibe Coding)

13장은 모바일 AI 에이전트를 능동적 시스템으로 만드는 자동화 전략을 다룬다.  
여기서는 자동화 설계 패턴, n8n 활용, 이벤트 기반 시스템 관점을 중심으로 설명한다.

핵심 내용은 다음과 같다.
- 자동화는 많이 붙이는 것이 아니라, 적절한 패턴으로 설계하는 것이 중요하다.
- 스케줄 기반, 이벤트 기반, 조건 기반, 승인 기반, 파이프라인 패턴이 핵심 자동화 구조다.
- n8n은 에이전트의 실행을 상황과 타이밍에 맞게 연결해주는 자동화 허브다.
- 이벤트 기반 시스템은 에이전트를 단순 반응형 봇이 아니라, 맥락 변화에 맞춰 움직이는 플랫폼으로 만든다.

즉 이 장은 AX 플랫폼의 움직임, 즉 “언제 어떻게 먼저 행동할 것인가”를 설계하는 장이다.

---

## PART 3의 핵심 정리

PART 3 전체를 관통하는 메시지는 분명하다.

1. AX 플랫폼은 기술 조합만으로 완성되지 않는다.  
2. 사용자 문제를 중심으로 한 기획 방법론이 필요하다.  
3. 역할에 맞는 개발 스택과 배포 전략이 필요하다.  
4. 자동화와 이벤트 기반 설계를 통해 살아 있는 운영 시스템이 되어야 한다.

즉 PART 3은 모바일 AI 에이전트를 단순 실험이 아니라 **지속 가능한 플랫폼**으로 만들기 위한 설계 철학과 운영 전략을 제공한다.  
이 파트를 통해 독자는 “만드는 법” 다음 단계인 “굴리는 법”과 “확장하는 법”을 이해하게 된다.
