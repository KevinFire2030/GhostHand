# 01. 새 채팅 실습

## 실습 목표

이 실습은 ChatGPT의 **가장 기본이면서도 가장 자주 쓰는 기능인 새 채팅**을 업무 관점에서 익히는 것을 목표로 합니다.

교육생은 아래를 직접 경험합니다.

- 새 채팅에서 업무 요청을 시작하는 방법
- 목적에 맞게 프롬프트를 구체화하는 방법
- 결과물이 마음에 들지 않을 때 다시 요청하는 방법
- 단순 질문이 아니라 **실무형 요청**으로 바꾸는 방법

## 실습 시나리오

해외지원팀에서 일하다 보면 아래와 같은 상황이 자주 발생할 수 있습니다.

- 메일 초안을 빠르게 작성하고 싶다.
- 회의 내용을 정리하고 싶다.
- 보고서 문장을 더 자연스럽게 다듬고 싶다.
- 영어 표현을 업무 톤에 맞게 바꾸고 싶다.
- 여러 정보를 표나 목록 형태로 빠르게 정리하고 싶다.

이럴 때 가장 먼저 활용할 수 있는 기능이 **새 채팅**입니다.

새 채팅은 복잡한 설정 없이 바로 시작할 수 있어서, AI를 업무에 붙이는 첫 단계로 가장 적합합니다.

## 진행 방법

1. ChatGPT를 엽니다.
2. **새 채팅**을 시작합니다.
3. 아래 예시 프롬프트 중 하나를 그대로 복사해서 붙여 넣습니다.
4. 결과를 확인합니다.
5. 결과가 아쉬우면 추가 요청을 통해 더 다듬습니다.

예를 들면 아래처럼 이어서 요청할 수 있습니다.

```text
조금 더 간단하게 줄여줘.
```

```text
해외 고객에게 보내는 메일 톤으로 바꿔줘.
```

```text
표 형식으로 다시 정리해줘.
```

```text
핵심만 3줄로 요약해줘.
```

## 예시 프롬프트 1. 영어 메일 초안 작성

상황: 해외 파트너에게 일정 지연 안내 메일을 보내야 하는 경우

```text
아래 상황을 바탕으로 해외 파트너에게 보내는 정중한 영어 메일 초안을 작성해줘.

상황:
- 원래 전달 예정일은 이번 주 금요일
- 내부 검토가 지연되어 다음 주 수요일로 일정 변경 필요
- 최대한 협조에 감사하는 톤
- 너무 길지 않게 작성
```

## 예시 프롬프트 2. 회의 내용 정리

상황: 회의 메모가 흩어져 있어서 핵심만 정리하고 싶은 경우

```text
아래 회의 메모를 바탕으로 핵심 내용만 정리해줘.

형식:
1. 결정사항
2. 담당자
3. 후속 액션
4. 리스크

회의 메모:
- 고객사 요청 사항은 이번 주 내 1차 회신 필요
- 시스템 로그는 김대리가 추가 확인
- 견적안은 다음 주 월요일까지 초안 작성
- 일정 지연 가능성이 있어 고객 커뮤니케이션 필요
- 기술팀과 지원팀 간 역할 구분을 문서로 남기기로 함
```

## 예시 프롬프트 3. 보고서 문장 다듬기

상황: 보고서 문장이 너무 딱딱하거나 어색한 경우

```text
아래 문장을 보고서에 들어갈 수 있게 더 자연스럽고 전문적인 표현으로 다듬어줘.

문장:
현재 고객사 요청사항이 많아서 대응이 늦어지고 있고, 관련 부서랑 같이 보고 있는데 아직 정확한 일정은 확인이 안 된 상태입니다.

조건:
- 너무 과장되지 않게
- 회사 보고서 문체로
- 2가지 버전 제시
```

## 예시 프롬프트 4. 영어 표현 검토

상황: 내가 작성한 영어 문장이 자연스러운지 확인하고 싶은 경우

```text
아래 영어 문장을 해외 비즈니스 커뮤니케이션 관점에서 검토해줘.

1. 어색한 부분이 있으면 수정해주고
2. 왜 수정했는지도 간단히 설명해줘
3. 더 자연스러운 대안 표현이 있으면 함께 제시해줘

문장:
We will share the result after internal checking is finished.
```

## 예시 프롬프트 5. 정보 정리 및 비교

상황: 여러 항목을 빠르게 비교 정리하고 싶은 경우

```text
아래 3가지 업무 요청 방식의 장단점을 비교해서 표로 정리해줘.

항목:
1. 이메일 요청
2. 메신저 요청
3. 회의에서 구두 요청

조건:
- 비교 항목은 속도, 정확성, 추적 용이성, 협업 편의성
- 마지막에 어떤 상황에서 어떤 방식이 적합한지도 정리해줘
```

## 실습 후 추가 요청 예시

처음 결과를 받은 뒤, 아래와 같은 방식으로 한 번 더 요청해보면 품질 차이를 바로 체감할 수 있습니다.

```text
좀 더 실무적으로 바꿔줘.
```

```text
해외지원팀에서 바로 쓸 수 있게 예시를 추가해줘.
```

```text
조금 더 간결하게 다시 작성해줘.
```

```text
영어와 한국어 버전을 같이 보여줘.
```

```text
상대방이 임원이라고 가정하고 더 정중하게 바꿔줘.
```

## 실습 포인트

이 실습에서 중요한 포인트는 “한 번에 완벽한 프롬프트를 쓰는 것”이 아닙니다.

중요한 것은 아래 3가지입니다.

- 먼저 초안을 빠르게 받아본다.
- 부족한 부분을 추가 요청으로 보완한다.
- 내가 원하는 결과 형태를 점점 구체화한다.

즉, 새 채팅은 **질문 1번으로 끝내는 도구**가 아니라, **업무 결과물을 함께 다듬는 출발점**으로 보는 것이 좋습니다.

## 강사 진행 멘트 예시

- 먼저 새 채팅 하나 열어보겠습니다.
- 여기서는 복잡한 기능보다, 그냥 바로 업무 요청을 넣어보는 경험이 중요합니다.
- 아래 예시 프롬프트를 그대로 복사해서 붙여 넣어보셔도 됩니다.
- 결과가 마음에 안 들면 다시 지시해서 점점 원하는 방향으로 다듬는 것이 핵심입니다.

## 기대 효과

이 실습이 끝나면 교육생은 아래를 할 수 있어야 합니다.

- 새 채팅으로 업무형 질문을 바로 시작할 수 있다.
- 메일, 보고서, 회의 정리 등 반복 업무에 AI를 붙일 수 있다.
- 결과가 아쉬울 때 어떤 식으로 다시 요청하면 되는지 감을 잡을 수 있다.
- 이후 이미지, 리서치, 프로젝트, GPTs, Codex 실습으로 자연스럽게 넘어갈 수 있다.
