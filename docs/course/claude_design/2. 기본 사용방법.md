# 2. 기본 사용방법

이번에는 Claude Design을 실제로 어떻게 사용하는지 가장 기본적인 흐름 중심으로 보겠습니다.
이 파트에서는 복잡한 기능을 다 설명하기보다,
처음 접속한 사람이 무엇부터 하면 되는지 순서대로 이해하는 것이 중요합니다.

Claude Design의 기본 구조는 아주 단순합니다.
왼쪽에는 Claude와 대화하는 채팅 영역이 있고,
오른쪽에는 결과물이 보이는 캔버스 영역이 있습니다.
즉, 우리는 왼쪽에서 원하는 것을 설명하고,
오른쪽에서 만들어진 결과를 확인하면서 계속 수정해나가는 방식으로 사용합니다.

## 1. 새 프로젝트 시작하기
처음 시작할 때는 새 프로젝트를 만드는 것부터 시작합니다.
공식 가이드 기준으로 새 프로젝트를 만들면,
조직에서 설정한 디자인 시스템이 있다면 그 설정을 자동으로 상속받게 됩니다.
즉, 회사나 팀의 색상, 폰트, 컴포넌트 패턴이 이미 반영된 상태에서 작업을 시작할 수 있습니다.

강의에서는 이 부분을 이렇게 설명하시면 됩니다.
"우리는 빈 화면에서 시작하지만, 완전히 맨땅에서 시작하는 것은 아닙니다. 조직 환경이 설정되어 있다면 기본 스타일이 자동으로 적용됩니다."

## 2. 처음 프롬프트 입력하기
프로젝트를 만들었으면 이제 Claude에게 무엇을 만들고 싶은지 설명해야 합니다.
이때 가장 중요한 것은 너무 짧고 모호하게 쓰지 않는 것입니다.
Claude Design 공식 가이드에서도 좋은 결과를 얻으려면 목표, 배치, 내용, 대상 사용자를 구체적으로 설명하는 것이 좋다고 안내합니다.

예를 들어 이렇게 입력할 수 있습니다.
- "신규 서비스 소개용 랜딩페이지 초안을 만들어줘. 첫 화면에는 핵심 메시지와 CTA 버튼이 보이게 해줘. 대상은 처음 방문하는 일반 사용자야."
- "모바일 앱 홈 화면 와이어프레임을 만들어줘. 상단에는 검색창, 중간에는 추천 카드, 하단에는 탭바가 들어가게 해줘."
- "팀장 보고용 5장 발표자료 초안을 만들어줘. 첫 장은 배경, 두 번째는 문제점, 세 번째는 개선안, 네 번째는 기대효과, 마지막은 결론으로 구성해줘."

즉, 그냥 "화면 만들어줘"라고 하는 것보다,
무엇을 만들지, 누가 볼지, 어떤 구성이 필요한지까지 함께 말해주는 것이 훨씬 좋습니다.

## 3. 결과 확인하기
프롬프트를 입력하면 Claude가 오른쪽 캔버스에 결과를 생성합니다.
이때 중요한 점은 첫 결과물을 정답으로 생각하지 않는 것입니다.
공식 문서에서도 첫 생성 결과는 시작점이라고 설명합니다.
진짜 가치는 첫 결과를 보고 계속 발전시키는 과정에 있습니다.

강의에서는 이 부분을 이렇게 말씀하시면 좋습니다.
"처음 나온 결과가 완벽할 필요는 없습니다. 오히려 초안을 빨리 보고, 그다음에 수정하는 것이 이 도구의 핵심입니다."

## 4. 수정 요청하기
결과를 본 뒤에는 원하는 방향으로 계속 수정 요청을 할 수 있습니다.
수정 방법은 크게 두 가지입니다.

### 채팅으로 수정하기
채팅은 구조를 바꾸거나 전체 방향을 바꾸는 데 적합합니다.
예를 들면,
- "좀 더 깔끔하고 미니멀한 스타일로 바꿔줘"
- "상단 섹션을 더 강조해줘"
- "전체적으로 B2B 서비스 느낌이 나게 수정해줘"
- "이 내용을 3장짜리 발표자료 구조로 다시 정리해줘"

이런 식으로 큰 방향을 바꾸는 요청은 채팅으로 하는 것이 좋습니다.

### 인라인 코멘트로 수정하기
반면 인라인 코멘트는 특정 부분만 정확하게 수정하고 싶을 때 유리합니다.
예를 들어 버튼 하나, 특정 문구, 특정 카드 영역처럼
캔버스 안의 구체적인 위치를 바로 짚어서 수정하는 방식입니다.

공식 가이드에서도 인라인 코멘트는
"이 버튼 문구 바꿔줘", "이 간격 줄여줘", "이 섹션만 더 강조해줘"처럼
부분 수정에 적합하다고 안내합니다.

즉 정리하면,
- 전체 구조나 방향 수정은 채팅
- 특정 요소 수정은 인라인 코멘트
이렇게 기억하시면 됩니다.

## 5. 참고 자료 추가하기
Claude Design은 프로젝트에 참고 자료를 붙여서 더 나은 결과를 만들 수 있습니다.
공식 가이드에서도 문서, 이미지 같은 자료를 프로젝트에 첨부해 맥락을 추가할 수 있다고 안내합니다.

이 기능은 실무에서 꽤 중요합니다.
왜냐하면 말로만 설명하는 것보다,
기존 기획 문서나 레퍼런스 이미지를 같이 보여주면 훨씬 원하는 방향에 가깝게 결과가 나오기 때문입니다.

강의에서는 이렇게 설명하시면 자연스럽습니다.
"내가 원하는 걸 다 문장으로 잘 설명하지 못하더라도, 기존 문서나 참고 이미지를 같이 넣어주면 Claude가 훨씬 더 정확하게 이해합니다."

## 6. 버전 바꾸기와 다른 방향 시도하기
작업을 하다 보면 지금까지 만든 안을 유지한 채 다른 방향을 보고 싶을 때가 있습니다.
이럴 때는 완전히 다시 시작하기보다,
현재 결과를 저장하고 다른 방향을 시도하도록 요청할 수 있습니다.

공식 문서 예시처럼,
"지금까지 만든 내용을 저장하고 완전히 다른 방향으로 다시 시도해줘"
라고 말하면 현재 작업을 유지하면서 다른 안을 탐색할 수 있습니다.

이 기능은 특히 회의 준비나 시안 비교 때 유용합니다.
한 가지 안만 보는 것이 아니라,
여러 방향을 빠르게 비교하면서 의사결정을 할 수 있기 때문입니다.

## 7. 공유와 내보내기
결과물이 어느 정도 정리되면 공유하거나 내보낼 수 있습니다.
공식 가이드 기준으로 우측 상단 Export 기능을 통해 다양한 형식으로 내보낼 수 있고,
조직 내부에서는 링크 공유도 가능합니다.

공식 소개 문서에서는 PDF, PPTX, standalone HTML, Canva 연동 같은 방향도 안내하고 있습니다.
그래서 발표자료, 시안 공유, 내부 검토 등 여러 상황에 맞춰 결과물을 전달할 수 있습니다.

## 8. 기본 사용방법 핵심 정리
이 파트는 마지막에 이렇게 정리하시면 됩니다.

Claude Design의 기본 사용 흐름은 어렵지 않습니다.
- 새 프로젝트를 시작하고
- 만들고 싶은 결과를 구체적으로 설명하고
- 생성된 초안을 확인하고
- 채팅과 인라인 코멘트로 계속 수정하고
- 필요하면 참고 자료를 추가하고
- 마지막에 공유하거나 내보내면 됩니다.

즉, Claude Design은 한 번에 완성품을 받는 도구가 아니라,
대화를 통해 초안을 만들고 점점 원하는 결과로 발전시키는 도구입니다.
그래서 잘 쓰는 핵심은 처음부터 완벽하게 입력하는 것이 아니라,
작게 시작하고 빠르게 수정하는 것입니다.
