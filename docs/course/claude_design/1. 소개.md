# 1. 소개

이번 파트에서는 Claude Design이 어떤 도구인지, 어디에 활용하면 좋은지, 그리고 왜 이 도구가 실무에서 의미가 있는지 말씀드리겠습니다.

먼저 Claude Design은 Claude와 대화하면서 디자인 결과물, 프로토타입, 발표자료, 문서 초안 같은 시각적 결과물을 만들어가는 도구입니다.
그냥 답변만 주는 챗봇이 아니라, 왼쪽에서는 대화를 하고 오른쪽에서는 실제 결과물을 캔버스 형태로 보면서 계속 수정해나갈 수 있다는 점이 특징입니다.

쉽게 말하면,
아이디어는 있는데 화면으로 만들기 어렵거나,
기획 초안은 있는데 빠르게 시각화하고 싶거나,
처음부터 완성본을 만드는 대신 초안을 빨리 만들고 다듬고 싶을 때 굉장히 유용한 도구라고 보시면 됩니다.

예를 들어 이런 상황에서 잘 맞습니다.
- 서비스 소개용 랜딩페이지 초안을 빠르게 잡고 싶을 때
- 기능 화면 구성을 러프하게 보고 싶을 때
- 발표자료 초안을 짧은 시간 안에 만들고 싶을 때
- 디자이너, 기획자, 개발자가 같이 볼 수 있는 공통 초안을 만들고 싶을 때
- 아이디어를 말로 설명하는 대신 바로 형태로 확인하고 싶을 때

즉, Claude Design은 완성된 결과를 단번에 뽑아내는 도구라기보다,
초안을 빠르게 만들고 반복적으로 개선하는 데 강한 도구라고 이해하시면 좋습니다.

## 접속 방법
그럼 실제로 어떻게 들어가느냐를 보겠습니다.
기본 진입은 Claude 웹 서비스인 `https://claude.com`에서 시작한다고 이해하시면 됩니다.
그리고 Claude Design이 열려 있는 계정 또는 조직 계정으로 로그인해야 사용할 수 있습니다.

공식적으로 참고할 수 있는 문서는 아래와 같습니다.
- Claude Design 시작 가이드: `https://support.claude.com/en/articles/14604416-get-started-with-claude-design`
- Anthropic 공식 소개: `https://www.anthropic.com/news/claude-design-anthropic-labs`
- Claude 공식 사이트: `https://claude.com`

로그인 후에는 일반 Claude처럼 대화창이 보이지만,
Claude Design에서는 보통 왼쪽에 채팅 영역, 오른쪽에 캔버스 영역이 함께 보입니다.
왼쪽에서 원하는 결과를 설명하면, 오른쪽에서 그 결과가 실제 화면이나 문서 형태로 생성됩니다.

처음 접속했을 때 바로 시작하는 방법도 어렵지 않습니다.
순서는 이렇게 설명하시면 됩니다.

1. Claude에 로그인합니다.
2. Claude Design 사용 권한이 있는 계정인지 확인합니다.
3. 새 프로젝트를 시작합니다.
4. 만들고 싶은 결과물을 구체적으로 입력합니다.
5. 생성된 결과를 보고 다시 수정 요청을 합니다.

예를 들면 첫 입력은 이렇게 할 수 있습니다.
- "신규 서비스 소개용 랜딩페이지 초안을 만들어줘"
- "5장 분량의 사내 보고용 발표자료 구조를 만들어줘"
- "모바일 앱 홈 화면 와이어프레임을 만들어줘"

이때 중요한 것은 처음부터 완벽한 프롬프트를 쓰는 것이 아니라,
일단 시작하고 결과를 보면서 계속 고쳐나가는 방식이라는 점입니다.
공식 가이드에서도 참고 자료를 첨부해서 맥락을 추가할 수 있고,
채팅으로 큰 방향을 바꾸거나 인라인 코멘트로 특정 부분만 수정할 수 있다고 안내하고 있습니다.

## 사용료
다음은 사용료입니다.
이 부분은 강의에서 꼭 짚어주셔야 합니다.
왜냐하면 Claude Design은 그냥 Claude 안에 포함된 기능처럼 보일 수 있지만,
실제로는 사용량과 과금 구조가 별도로 관리되기 때문입니다.

공식 안내 기준으로 Claude Design은 Claude의 다른 기능과 독립적으로 사용량이 측정됩니다.
즉, 일반 채팅이나 Claude Code와 같은 한도를 그대로 쓰는 것이 아니라,
Claude Design만의 사용량 추적과 허용량이 따로 있습니다.

현재 확인된 내용으로는,
Claude Design은 연구 프리뷰 기준으로 아래 구독자에게 제공됩니다.
- Claude Pro
- Claude Max
- Claude Team
- Claude Enterprise

그래서 강의에서는 무료 사용자가 누구나 바로 쓰는 서비스라고 설명하기보다는,
현재는 유료 플랜이나 조직용 플랜 중심으로 제공된다고 안내하는 편이 정확합니다.

또 하나 중요한 점은 주간 허용량입니다.
Claude Design은 주간 allowance가 있고, 이 한도는 7일 단위로 리셋됩니다.
그리고 기본 제공량을 넘기면 extra usage를 추가 구매해서 더 사용할 수 있습니다.

일반 Claude 요금제 예시는 공식 가격 페이지 기준으로 대략 다음과 같습니다.
- Pro: 월 20달러, 연간 결제 시 월 환산 17달러
- Max: 월 100달러부터
- Team: 좌석당 월 25달러, 연간 결제 시 좌석당 월 20달러
- Enterprise: 별도 문의

하지만 여기서 중요한 것은,
이 가격표만 보고 Claude Design 비용을 다 이해했다고 보면 안 된다는 점입니다.
실제로는 아래 항목을 함께 봐야 합니다.
- 내가 사용하는 계정이 Pro인지, Max인지, Team인지
- 내 계정에서 Claude Design이 활성화되어 있는지
- 주간 허용량이 어느 정도인지
- 초과 사용 시 extra usage 구매가 가능한지
- 회사 계정이라면 관리자가 별도 정책을 걸어두었는지

조직 계정 관점에서도 차이가 있습니다.
공식 문서 기준으로 Team과 seat-based Enterprise는 사용자별 주간 허용량이 제공되고,
usage-based Enterprise는 기존 계약 기준의 표준 API 요율로 청구된다고 안내되어 있습니다.
또 Enterprise 일부 사용자에게는 시작 단계에서 약 20회 수준의 일반적인 프롬프트를 사용할 수 있는 1회성 크레딧이 제공된다는 내용도 있습니다.

그래서 이 부분은 이렇게 정리하시면 좋습니다.
Claude Design은 기능만 볼 것이 아니라,
접속 권한과 별도 사용량 정책까지 같이 확인해야 실제 운영 비용을 정확히 이해할 수 있습니다.

## 기존 방식 대비 장점
그럼 왜 이 도구를 써야 하느냐를 마지막으로 정리해보겠습니다.
기존에는 기획 초안을 만들거나 화면 아이디어를 정리할 때,
사람이 처음부터 목차를 잡고, 문장을 쓰고, 화면 구조를 그리고, 다시 수정하는 데 시간이 많이 들었습니다.

반면 Claude Design을 사용하면 시작 속도가 훨씬 빨라집니다.
아이디어를 바로 형태로 바꿔볼 수 있고,
마음에 안 들면 다시 설명하면서 다른 방향을 바로 시도할 수 있습니다.
또 여러 안을 짧은 시간 안에 비교해볼 수 있어서,
처음 한 장을 만드는 부담이 크게 줄어듭니다.

정리하면 Claude Design의 장점은 세 가지입니다.
- 시작이 빠릅니다.
- 수정이 쉽습니다.
- 여러 방향을 짧은 시간 안에 탐색할 수 있습니다.

그래서 특히 기획 초안, 디자인 방향 정리, 발표자료 구성, 아이디어 시각화처럼
처음 출발점이 중요한 업무에서 강점을 발휘합니다.

이 파트의 핵심 메시지는 이것입니다.
Claude Design은 완성본을 한 번에 뽑는 도구라기보다,
초안을 빠르게 만들고 대화하면서 개선해나가는 데 매우 강한 도구입니다.
그리고 실제로 쓰기 전에는 접속 권한과 사용료 구조까지 함께 이해하는 것이 중요합니다.
