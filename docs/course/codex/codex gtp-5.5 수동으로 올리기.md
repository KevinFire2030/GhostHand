# codex gtp-5.5 수동으로 올리기

## 개요
Codex CLI를 실행했을 때 기본 모델 선택 목록에는 `gpt-5.5`가 바로 보이지 않을 수 있습니다.
하지만 직접 모델명을 지정해서 실행하면 `gpt-5.5`로 세션을 시작할 수 있습니다.

즉,
- 모델 선택 UI에 안 보여도
- 명령어로 직접 지정하면
- `gpt-5.5` 사용이 가능한 경우가 있습니다.

## 실행 방법
PowerShell에서 아래처럼 실행합니다.

```powershell
codex -m gpt-5.5
```

실행 후 상단에 아래처럼 보이면 정상입니다.

```text
model: gpt-5.5
```

즉, 현재 세션이 `gpt-5.5` 모델로 시작된 것입니다.

## 예시
현재 위치가 `C:\WINDOWS\system32` 인 상태에서도 아래처럼 실행할 수 있습니다.

```powershell
codex -m gpt-5.5
```

실행 후 예시 화면:

```text
╭───────────────────────────────────────╮
│ >_ OpenAI Codex (v0.123.0-alpha.5)    │
│                                       │
│ model: gpt-5.5            change      │
│ directory: C:\WINDOWS\system32        │
╰───────────────────────────────────────╯
```

이 상태라면 `gpt-5.5` 적용이 완료된 것입니다.

## 특정 프로젝트 폴더에서 실행하기
실무에서는 보통 프로젝트 폴더로 이동한 뒤 실행하는 것이 좋습니다.

예시:

```powershell
cd E:\ax\상생\PoC2
codex -m gpt-5.5
```

이렇게 하면 해당 프로젝트 폴더를 기준으로 `gpt-5.5` 세션을 시작할 수 있습니다.

## 왜 필요한가?
일부 환경에서는 Codex 모델 선택 메뉴에 아래처럼 `gpt-5.5`가 보이지 않을 수 있습니다.

- `gpt-5.4`
- `gpt-5.4-mini`
- `gpt-5.3-codex`
- `gpt-5.2`

하지만 이 경우에도 모델명이 숨겨져 있거나 수동 지정만 허용되는 상태일 수 있으므로,
직접 `-m gpt-5.5` 옵션으로 시도해보는 것이 좋습니다.

## 한 줄 정리
`gpt-5.5`가 목록에 안 보여도 아래 명령으로 직접 올릴 수 있습니다.

```powershell
codex -m gpt-5.5
```
