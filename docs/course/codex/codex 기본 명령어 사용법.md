# codex 기본 명령어 사용법

## 개요
Codex를 실행한 뒤 `/` 를 누르면 기본 명령어 목록이 나타납니다.
이 문서는 자주 보게 되는 기본 명령어가 무엇인지,
그리고 각각을 어떤 상황에서 어떻게 쓰면 되는지 예시 중심으로 정리한 문서입니다.

실행 예시:

```powershell
codex -m gpt-5.5
```

실행 후 `/` 입력 시 보일 수 있는 기본 명령어 예시는 아래와 같습니다.

- `/model`
- `/permissions`
- `/sandbox-add-read-dir`
- `/experimental`
- `/memories`
- `/skills`
- `/review`

---

## 1. /model
### 역할
어떤 모델을 사용할지,
그리고 reasoning effort(추론 강도)를 어떻게 둘지 선택하는 명령어입니다.

### 언제 쓰나?
- 더 강한 모델로 바꾸고 싶을 때
- 더 빠른 모델로 바꾸고 싶을 때
- 작업 난이도에 따라 추론 강도를 조절하고 싶을 때

### 사용 예시
```text
/model
```

### 활용 예시
- 간단한 문서 수정만 할 때는 가벼운 모델 사용
- 복잡한 코드 분석이나 구조 변경은 더 강한 모델 사용
- 속도가 중요할 때는 Fast mode 고려

---

## 2. /permissions
### 역할
Codex가 현재 세션에서 어떤 권한으로 동작할지 선택하는 명령어입니다.

### 언제 쓰나?
- 파일 수정 권한 범위를 바꾸고 싶을 때
- 더 제한적으로 쓰고 싶을 때
- 반대로 더 넓은 권한이 필요한 작업을 할 때

### 사용 예시
```text
/permissions
```

### 활용 예시
- 문서 읽기만 시킬 때는 보수적으로
- 프로젝트 전체 수정 작업을 시킬 때는 넓게
- 초보자는 먼저 제한된 권한으로 시작하는 것이 안전

---

## 3. /sandbox-add-read-dir
### 역할
샌드박스 환경에서 특정 폴더를 읽을 수 있도록 추가하는 명령어입니다.

### 언제 쓰나?
- 현재 폴더 밖에 있는 자료를 Codex가 읽어야 할 때
- 다른 드라이브나 상위 폴더의 문서를 참고해야 할 때

### 사용 예시
```text
/sandbox-add-read-dir E:\ax\상생\PoC2
```

또는

```text
/sandbox-add-read-dir C:\Users\fireman_pro\Documents
```

### 활용 예시
- 보고서 샘플 폴더 읽기
- 참고 문서 폴더 읽기
- 다른 프로젝트의 README를 참고하게 하기

주의:
- 절대경로를 넣어야 합니다.
- 쓰기 권한이 아니라 읽기 권한 추가라고 이해하면 됩니다.

---

## 4. /experimental
### 역할
실험적 기능을 켜거나 끄는 명령어입니다.

### 언제 쓰나?
- 새로운 기능을 시험해보고 싶을 때
- 기본 기능 외 베타 성격 기능을 써보고 싶을 때

### 사용 예시
```text
/experimental
```

### 활용 예시
- 아직 정식이 아닌 기능 확인
- 새 UI/새 동작 방식 테스트

주의:
- 실험 기능은 안정성이 다를 수 있습니다.
- 중요한 작업에서는 기본 기능을 우선 사용하는 것이 안전합니다.

---

## 5. /memories
### 역할
메모리 사용 및 생성 방식을 설정하는 명령어입니다.

### 언제 쓰나?
- Codex가 어떤 정보를 기억해야 하는지 조정하고 싶을 때
- 메모리 기능이 켜져 있는지 확인하고 싶을 때

### 사용 예시
```text
/memories
```

### 활용 예시
- 반복적으로 쓰는 프로젝트 규칙 반영
- 자주 사용하는 스타일이나 컨벤션 기억시키기
- 장기 작업에서 맥락 유지 여부 확인

---

## 6. /skills
### 역할
특정 작업을 더 잘 수행하도록 도와주는 스킬을 사용하는 명령어입니다.

### 언제 쓰나?
- 특정 유형의 작업을 더 정확하게 시키고 싶을 때
- 사전에 정의된 작업 방식을 활용하고 싶을 때

### 사용 예시
```text
/skills
```

### 활용 예시
- 리뷰용 스킬
- 문서 생성용 스킬
- 특정 프로젝트 유형에 맞는 작업 방식 적용

초보자 관점에서는
"코덱스에게 일 잘하는 방법을 더 붙여주는 기능"
정도로 이해하면 쉽습니다.

---

## 7. /review
### 역할
현재 변경사항을 검토하고,
문제가 될 수 있는 부분을 찾아달라고 요청하는 명령어입니다.

### 언제 쓰나?
- 수정 작업 후 점검하고 싶을 때
- 커밋 전에 문제를 한번 훑고 싶을 때
- 코드/문서 품질을 빠르게 검토하고 싶을 때

### 사용 예시
```text
/review
```

### 활용 예시
- 현재 수정한 코드에서 오류 가능성 찾기
- 빠뜨린 예외 처리 확인
- 문서 오탈자/구조 점검

초보자에게 특히 좋은 이유는,
작업을 시킨 뒤 그냥 끝내지 않고
"한 번 더 봐줘"라고 할 수 있기 때문입니다.

---

## 8. 왕초보 추천 사용 순서
처음 Codex를 쓰는 사람에게는 아래 순서를 추천합니다.

1. `codex -m gpt-5.5` 실행
2. `/model` 로 현재 모델 확인
3. `/permissions` 로 권한 확인
4. 간단한 작업 요청
   - 예: 현재 폴더 구조 설명
   - 예: README 초안 작성
5. 필요하면 `/sandbox-add-read-dir` 로 참고 폴더 추가
6. 작업 후 `/review` 로 점검

즉,
처음에는 모든 명령을 다 외우기보다,
`/model`, `/permissions`, `/sandbox-add-read-dir`, `/review`
이 네 가지부터 익히면 충분합니다.

---

## 9. 한 줄 정리
Codex에서 `/` 를 눌렀을 때 나오는 기본 명령어는,
모델 선택, 권한 설정, 읽기 범위 추가, 메모리/스킬 활용, 결과 검토를 도와주는 작업 보조 메뉴라고 이해하면 됩니다.
