# 코덱스 1시간 강의안

## 1. 강의 개요
- 대상: 코덱스를 전혀 모르는 왕초보
- 시간: 1시간
- 방식: 설명 + 시연 + 간단 실습

## 2. 강의 목표
이번 강의가 끝나면 아래를 할 수 있도록 구성합니다.

- 처음으로 코덱스를 설치하고 실행할 수 있다.
- 코덱스와 ChatGPT의 차이를 설명할 수 있다.
- 코덱스와 OpenClaw / Claude Code의 차이를 설명할 수 있다.
- 코덱스를 업무에 활용할 수 있다.
- 초보가 할 수 있는 업무 활용 10가지를 이해할 수 있다.

## 3. 1시간 강의 진행안

### 0~5분, 오프닝
- 오늘 강의 목표 소개
- 코덱스가 무엇인지 아주 짧게 설명
- 오늘은 이론보다 "처음 설치하고 한번 써보는 것"에 초점이 있다고 안내

### 5~15분, 코덱스란 무엇인가
- 코덱스는 무엇인가
- 왜 필요한가
- 어떤 사람에게 유용한가
- ChatGPT와는 무엇이 다른가

### 15~25분, 설치 및 실행
- 설치 전 준비물 확인
- 설치 방법 안내
- 실행 방법 안내
- 첫 명령 실행 시연

### 25~35분, 비교 설명
- 코덱스 vs ChatGPT
- 코덱스 vs OpenClaw
- 코덱스 vs Claude Code
- 어떤 상황에서 무엇을 쓰면 좋은지

### 35~50분, 업무 활용
- 초보가 바로 써볼 수 있는 활용법 10가지
- 각 활용 예시 간단 설명
- 실제 시연 1~2개

### 50~60분, 실습 및 마무리
- 초간단 실습
- 자주 하는 실수 안내
- 오늘 내용 정리
- 질문 받기

## 4. 본 강의안 상세

## 4-1. 오프닝
안녕하세요.
오늘은 코덱스를 처음 접하는 분들을 대상으로,
설치부터 실행, 그리고 실제 업무에 어떻게 활용할 수 있는지까지 1시간 동안 아주 쉽게 살펴보겠습니다.

오늘 강의의 목표는 전문가 수준의 사용법을 익히는 것이 아닙니다.
처음 보는 분도
"아, 코덱스는 이런 거구나"
"이건 내 업무에도 써볼 수 있겠네"
라고 느끼실 수 있도록 만드는 것이 목적입니다.

특히 오늘은 왕초보 기준으로 진행하겠습니다.
즉,
- 설치해보고
- 실행해보고
- 어디에 쓰는지 이해하고
- 바로 써볼 수 있는 예시를 익히는 데 초점을 맞추겠습니다.

---

## 4-2. 코덱스란 무엇인가
코덱스는 쉽게 말하면,
코드 작업과 파일 작업, 개발 보조 작업에 특화된 AI 도구라고 이해하시면 됩니다.

일반적인 대화형 AI가 질문에 답하고 글을 써주는 데 강하다면,
코덱스는 실제 프로젝트 폴더를 보고,
코드나 문서를 읽고,
필요한 수정 작업을 도와주는 쪽에 더 강합니다.

즉,
그냥 아이디어를 물어보는 AI라기보다,
실제로 "작업하는 AI"에 더 가깝습니다.

예를 들면 이런 작업이 가능합니다.
- 프로젝트 구조 파악
- 코드 수정 제안
- 문서 생성
- 반복 작업 자동화
- 초안 코드 작성
- 기존 코드 설명

왕초보 분들은 이렇게 이해하시면 됩니다.
"ChatGPT가 똑똑한 대화 상대라면,
코덱스는 내 폴더 안에 들어와 같이 일하는 작업 도우미에 가깝다."

---

## 4-3. 코덱스와 ChatGPT의 차이
이 부분은 아주 중요합니다.
많은 분들이 "ChatGPT랑 뭐가 다른데요?"라고 묻습니다.

가장 쉬운 설명은 아래와 같습니다.

### ChatGPT
- 질문하고 답변받는 데 강함
- 아이디어 정리, 요약, 문서 작성에 강함
- 보통 대화 중심

### 코덱스
- 파일과 프로젝트를 직접 다루는 작업에 강함
- 코드 읽기, 수정, 생성에 강함
- 실제 작업 흐름에 더 가까움

즉,
ChatGPT는 "물어보는 AI"
코덱스는 "작업하는 AI"
라고 설명하면 이해가 빠릅니다.

물론 겹치는 부분도 있습니다.
둘 다 글을 쓰고 설명할 수 있습니다.
하지만 코덱스는 실제 파일과 프로젝트 맥락에서 더 강력합니다.

---

## 4-4. 코덱스와 OpenClaw / Claude Code 차이
이 부분도 초보자 입장에서 헷갈릴 수 있으니 쉽게 설명합니다.

### 코덱스
- 코드 및 프로젝트 작업에 특화된 AI 실행 도구
- 실제 파일을 읽고 수정하는 흐름에 강함
- 개발/문서 작업 자동화에 적합

### OpenClaw
- 여러 도구와 작업을 연결하고,
  비서처럼 전체 업무를 도와주는 운영 환경에 가까움
- 단순 코딩뿐 아니라 문서, 웹, 기억, 자동화, 메시징 등 범위가 넓음
- 쉽게 말해 "코덱스를 포함해 여러 능력을 묶어 쓰는 환경"

### Claude Code
- Claude 계열 모델을 활용하는 코드 작업 도구
- 코덱스와 비슷하게 코드 읽기/수정/생성에 강함
- 차이는 사용하는 모델과 도구 철학, 인터페이스, 결과 스타일에 있음

아주 쉽게 말하면,
- 코덱스: 코드 작업 특화 AI
- Claude Code: Claude 기반 코드 작업 AI
- OpenClaw: 이런 도구들을 포함해서 더 넓은 업무를 다루는 비서형 환경

즉,
코덱스와 Claude Code는 비슷한 종류,
OpenClaw는 그보다 더 넓은 운영 플랫폼이라고 보면 됩니다.

## 4-5. 설치 및 실행
이제 가장 중요한 설치와 실행입니다.

### 설치 전 준비물
- 노트북
- 인터넷 연결
- 터미널 또는 명령 프롬프트 사용 가능 환경
- 기본적인 명령 입력 가능 여부

### 설치
실제 설치 명령은 배포 방식에 따라 다를 수 있으므로,
강의에서는 현재 사용하는 환경 기준 명령을 보여주고 그대로 따라하게 하면 됩니다.

설치 설명 시 핵심은 아래입니다.
- 설치는 보통 명령 한 줄로 진행된다.
- 설치 후 버전 확인 명령으로 정상 설치 여부를 확인한다.
- 정상 설치되면 첫 실행을 해본다.

### 코덱스를 시작하는 3가지 방식
코덱스는 초보자 기준으로 아래 3가지 방식으로 시작한다고 이해하면 쉽습니다.

#### 1) ChatGPT 웹에서 Codex Cloud로 시작
가장 쉬운 입문 방식입니다.
웹 브라우저에서 아래 주소로 접속해서 시작할 수 있습니다.

- `https://chatgpt.com/codex/cloud`

이 방식의 장점은 설치 부담이 적고,
일단 코덱스가 어떤 느낌인지 빠르게 경험할 수 있다는 점입니다.
즉,
왕초보에게는 가장 진입장벽이 낮은 시작점입니다.

#### 2) 로컬 환경에 설치해서 시작
조금 더 본격적으로 쓰려면,
내 노트북이나 PC에 코덱스를 설치해서 시작할 수 있습니다.

이 방식은 실제 폴더, 문서, 프로젝트를 기준으로 작업하기 좋고,
반복 사용이나 업무 자동화 관점에서는 더 실용적입니다.
즉,
"내 컴퓨터 안에 있는 자료를 가지고 실제로 일하게 하고 싶다"면 이 방식이 좋습니다.

#### 3) 다른 도구/플랫폼과 함께 시작
코덱스는 단독으로만 쓰는 것이 아니라,
OpenClaw 같은 운영 환경이나 다른 AI 코딩 도구와 비교하거나 함께 쓰면서 시작할 수도 있습니다.

이 방식은 단순 체험보다,
"내 업무에는 어떤 도구 조합이 맞을까"를 보는 관점에 가깝습니다.
예를 들어,
- ChatGPT는 기획/설명
- 코덱스는 실제 작업
- OpenClaw는 전체 흐름 관리
같이 역할을 나눠서 생각할 수 있습니다.

강의에서는 이 3가지를 아래처럼 설명하면 좋습니다.
- 가장 쉬운 체험: Codex Cloud
- 가장 실무적인 시작: 로컬 설치
- 가장 확장된 활용: 다른 도구와 함께 사용

### 실행
처음 실행했을 때 중요한 것은 겁먹지 않는 것입니다.
실행 후 나오는 화면이나 프롬프트는
"이제 여기서 내가 시키는 일을 하는구나" 정도로 이해하면 충분합니다.

처음 시연은 아주 단순하게 가는 것이 좋습니다.
예를 들면:
- 현재 폴더 구조 설명해줘
- 이 문서 요약해줘
- 샘플 파일 하나 만들어줘

즉,
처음부터 복잡한 코딩보다
"말하면 작업을 한다"는 감각을 익히는 것이 중요합니다.

## 4-6. 초보가 할 수 있는 업무 활용 10가지
이 부분은 강의에서 가장 실용적으로 느껴질 수 있는 파트입니다.

### 1. 문서 초안 작성
회의 메모, 가이드, 설명 문서를 빠르게 초안으로 만들 수 있습니다.

### 2. 기존 문서 요약
긴 문서를 읽고 핵심만 정리할 수 있습니다.

### 3. 코드 설명 받기
기존 코드가 무슨 역할인지 쉽게 설명받을 수 있습니다.

### 4. 반복 수정 작업
여러 파일에서 비슷한 표현이나 구조를 한 번에 정리할 수 있습니다.

### 5. README 만들기
프로젝트 설명 문서를 자동으로 초안 작성할 수 있습니다.

### 6. 샘플 데이터 만들기
CSV, JSON, 테스트용 데이터 초안을 만들 수 있습니다.

### 7. 보고서 템플릿 만들기
반복적으로 쓰는 보고서 형식을 미리 만들어둘 수 있습니다.

### 8. 폴더 구조 분석
프로젝트 폴더가 어떻게 구성되어 있는지 빠르게 파악할 수 있습니다.

### 9. 초안 코드 작성
간단한 스크립트나 자동화 코드를 빠르게 만들 수 있습니다.

### 10. 기존 작업 개선 아이디어 얻기
현재 업무를 더 단순하게 만들 수 있는 방법을 제안받을 수 있습니다.

이 10가지를 보면,
코덱스는 개발자만 쓰는 도구가 아니라,
비개발자도 "작업 보조 AI"로 충분히 활용할 수 있습니다.

## 4-7. 실습 예시
실습은 아주 짧고 쉽게 가는 것이 좋습니다.

### 실습 1
- 현재 폴더를 설명해줘

### 실습 2
- 샘플 README 파일 초안 만들어줘

### 실습 3
- 이 문서를 5줄로 요약해줘

실습 목표는 어려운 작업을 해내는 것이 아니라,
"내가 말하면 코덱스가 실제로 일한다"는 경험을 만드는 것입니다.

---

## 4-8. 초보자가 자주 하는 실수
- 너무 복잡한 요청부터 시작함
- 무엇을 원하는지 अस्पष्ट하게 말함
- 한 번에 너무 많은 일을 시킴
- 결과를 검토하지 않고 그대로 믿음

그래서 초보자에게는 아래 원칙을 권합니다.
- 짧게 시키기
- 한 번에 하나씩 시키기
- 결과 확인하기
- 필요한 경우 다시 수정 요청하기

---

## 5. 강의 마무리
오늘은 코덱스를 처음 접하는 분들을 기준으로,
설치하고, 실행하고, ChatGPT와의 차이를 이해하고,
업무 활용 예시 10가지까지 같이 봤습니다.

오늘 꼭 기억하시면 좋은 것은 아래 3가지입니다.
- 코덱스는 작업형 AI다.
- ChatGPT와 비슷해 보여도 실제 프로젝트 작업에 더 강하다.
- 처음에는 작은 업무부터 붙여보는 것이 가장 좋다.

한 줄로 정리하면,
**코덱스는 왕초보도 설치해서 바로 써볼 수 있는 작업형 AI 도구이며,
특히 문서, 코드, 반복 작업, 초안 작성 업무에서 빠르게 효과를 볼 수 있습니다.**

감사합니다.
