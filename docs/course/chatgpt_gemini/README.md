# ChatGPT Enterprise / GPTs / Codex 핸즈온 실습

## 강의 안내

이 문서는 **해외지원팀 AI PoC 인력 대상 핸즈온 실습**을 위해 준비한 **원페이지 강의 자료**입니다.

오늘 실습은 여러 페이지를 왔다 갔다 하지 않고, **이 페이지 하나를 띄워놓고 순서대로 따라오는 방식**으로 진행합니다.

강사는 이 문서를 기준으로 설명하고, 교육생은 각자 PC에서 이 페이지를 열어둔 상태로 **예시 프롬프트를 복사/붙여넣기** 하며 실습을 진행합니다.

---

## 오늘 강의 목표

오늘 강의의 목표는 아래 4가지입니다.

- ChatGPT Enterprise의 핵심 기능을 업무 관점에서 직접 써본다.
- GPTs를 활용해 간단한 에이전트형 PoC를 만드는 흐름을 이해한다.
- Codex가 어떤 작업에 강한지 감을 잡는다.
- 해외지원팀 업무에 연결할 수 있는 실습 아이디어를 얻는다.

---

## 오늘 전체 구성

오늘 강의는 아래 **3개 큰 섹션**으로 진행합니다.

1. **ChatGPT Enterprise, 바로 쓰는 업무 활용**
2. **GPTs, 업무형 에이전트 PoC 만들기**
3. **Codex, 실행형 자동화와 코딩 에이전트**

## 1-3. 주요 기능 실습

### A. 새 채팅

#### 실습 포인트

새 채팅은 가장 기본 기능이지만, 실제 업무에서는 가장 자주 쓰는 출발점입니다.

활용 예시:
- 영어 메일 초안 작성
- 회의 메모 정리
- 보고서 문장 다듬기
- 정보 비교 정리
- 업무 설명 문서 초안 만들기

#### 프롬프트 예시 1

```text
아래 상황을 바탕으로 해외 파트너에게 보내는 정중한 영어 메일 초안을 작성해줘.

상황:
- 원래 전달 예정일은 이번 주 금요일
- 내부 검토가 지연되어 다음 주 수요일로 일정 변경 필요
- 최대한 협조에 감사하는 톤
- 너무 길지 않게 작성
```

#### 프롬프트 예시 2

```text
아래 회의 메모를 바탕으로 핵심 내용만 정리해줘.

형식:
1. 결정사항
2. 담당자
3. 후속 액션
4. 리스크

회의 메모:
- 고객사 요청 사항은 이번 주 내 1차 회신 필요
- 시스템 로그는 김대리가 추가 확인
- 견적안은 다음 주 월요일까지 초안 작성
- 일정 지연 가능성이 있어 고객 커뮤니케이션 필요
- 기술팀과 지원팀 간 역할 구분을 문서로 남기기로 함
```

#### 프롬프트 예시 3

```text
아래 문장을 보고서에 들어갈 수 있게 더 자연스럽고 전문적인 표현으로 다듬어줘.

문장:
현재 고객사 요청사항이 많아서 대응이 늦어지고 있고, 관련 부서랑 같이 보고 있는데 아직 정확한 일정은 확인이 안 된 상태입니다.

조건:
- 너무 과장되지 않게
- 회사 보고서 문체로
- 2가지 버전 제시
```

#### 프롬프트 예시 4

```text
아래 영어 문장을 해외 비즈니스 커뮤니케이션 관점에서 검토해줘.

1. 어색한 부분이 있으면 수정해주고
2. 왜 수정했는지도 간단히 설명해줘
3. 더 자연스러운 대안 표현이 있으면 함께 제시해줘

문장:
We will share the result after internal checking is finished.
```

#### 프롬프트 예시 5

```text
아래 3가지 업무 요청 방식의 장단점을 비교해서 표로 정리해줘.

항목:
1. 이메일 요청
2. 메신저 요청
3. 회의에서 구두 요청

조건:
- 비교 항목은 속도, 정확성, 추적 용이성, 협업 편의성
- 마지막에 어떤 상황에서 어떤 방식이 적합한지도 정리해줘
```

### B. 이미지

#### 실습 포인트

이미지 기능은 발표자료, 보고서, 교육자료, 설명용 시각화 자료를 빠르게 만드는 데 쓸 수 있습니다.

#### 프롬프트 예시 1

```text
해외지원팀의 AI 업무 활용 교육 자료 표지에 어울리는 이미지를 만들어줘.

조건:
- 전체 분위기는 깔끔하고 전문적
- AI, 글로벌 협업, 업무 생산성 향상 느낌이 함께 보이면 좋음
- 발표 자료 첫 장에 들어갈 수 있는 스타일
- 텍스트는 이미지 안에 넣지 말 것
- 너무 화려하지 않고 회사 교육자료에 어울리게
```

#### 프롬프트 예시 2

```text
아래 업무 흐름을 한눈에 이해할 수 있는 일러스트 스타일 이미지로 만들어줘.

업무 흐름:
- 해외 법인에서 문의 접수
- 본사 지원팀에서 내용 검토
- 관련 부서 협의
- 결과 회신

조건:
- 발표 자료에 넣기 좋게 단순하고 명확한 구성
- 아이콘이나 흐름 느낌이 잘 보이게
- 복잡한 텍스트는 넣지 말 것
- 비즈니스 문서에 어울리는 색감
```

#### 프롬프트 예시 3

```text
RAG 기반 챗봇의 개념을 비전문가도 직관적으로 이해할 수 있게 설명하는 이미지를 만들어줘.

조건:
- 사용자의 질문이 들어오고
- 사내 문서 또는 지식 저장소를 참고한 뒤
- 답변이 생성되는 흐름이 느껴지게
- 너무 기술 다이어그램처럼 어렵지 않게
- 교육 자료에 넣기 좋은 깔끔한 스타일
- 텍스트는 최소화하거나 넣지 말 것
```

#### 프롬프트 예시 4

```text
기업의 실무자가 AI를 활용해 업무를 처리하는 장면을 보고서 삽입용 이미지로 만들어줘.

조건:
- 과장된 미래도시 느낌 말고 현실적인 사무 환경
- 노트북, 대시보드, 문서, 협업 느낌이 자연스럽게 보일 것
- 전체적으로 신뢰감 있고 차분한 분위기
- 회사 보고서에 넣어도 어색하지 않게
- 텍스트는 넣지 말 것
```

#### 프롬프트 예시 5

```text
ChatGPT와 Codex를 활용한 실무형 핸즈온 교육을 안내하는 이미지를 만들어줘.

조건:
- 교육, 실습, 디지털 업무도구, 생산성 향상 느낌
- 초보자도 부담 없이 참여할 수 있는 친근한 분위기
- 동시에 전문성도 느껴지게
- 사내 교육 포스터나 안내 페이지에 활용 가능한 스타일
- 텍스트는 이미지 안에 넣지 말 것
```

# 2. GPTs, 업무형 에이전트 PoC 만들기

## 2-1. GPTs란

GPTs는 단순 대화형 챗봇이 아니라, **지침 + 지식 + 출력 형식**을 설계해 특정 업무를 수행하도록 만든 맞춤형 에이전트입니다.

핵심은 아래 3가지입니다.
- 어떤 역할을 맡길 것인가
- 어떤 문서를 참고하게 할 것인가
- 어떤 형식으로 결과를 내게 할 것인가

---

## 2-2. PoC1. RAG 기반 챗봇

### 실습 목표

해외지원팀에서 반복적으로 받는 질문에 대해, 내부 가이드 문서를 기반으로 1차 답변을 제공하는 GPTs를 만든다고 가정합니다.

### 실습

- RAG 실습 파일 다운로드
- GPTs 생성 화면 진입
- 시나리오 확인
- 프롬프트 복사/붙여넣기
- GPTs 항목 채우기
- 테스트
- 공유

### 시나리오

예시 질문:
- 경비 정산 시 먼저 확인해야 하는 항목은 무엇인가?
- 특정 업무 요청은 어느 부서로 문의해야 하는가?
- 해외지원 프로세스는 어떻게 진행되는가?
- 자주 발생하는 요청 유형별 기본 대응은 무엇인가?

### 프롬프트 예시

```text
이 GPT의 목적은 해외지원팀 관련 반복 질문에 대해, 업로드된 가이드 문서를 근거로 1차 답변을 제공하는 것이다.

이 GPT는 아래와 같이 동작한다.
1. 사용자가 해외지원 업무 관련 질문을 입력한다.
2. GPT는 업로드된 문서에서 관련 내용을 먼저 확인한다.
3. 문서 근거를 바탕으로 간결하고 실무적으로 답변한다.
4. 문서에 없는 내용은 추측하지 않는다.
5. 문서에 없는 경우에는 담당 부서 또는 추가 확인 필요 사항을 안내한다.
6. 필요 시 관련 질문 예시를 함께 제안한다.

이 GPT는 아래 원칙을 반드시 지켜야 한다.
- 업로드된 문서를 근거로 답변할 것
- 근거 없는 추측을 하지 말 것
- 짧고 명확하게 답변할 것
- 사용자가 바로 다음 행동을 할 수 있게 안내할 것
- 문서에 없는 내용을 아는 척하지 말 것
```

### GPTs 들어갈 항목 생성

- 이름
- 설명
- 지침
- 대화 스타터
- Knowledge 파일

### 테스트 질문 예시

```text
해외지원 요청 접수 후 기본 처리 절차를 간단히 설명해줘.
```

```text
이 질문이 문서에 없으면 누구에게 확인해야 하는지 안내해줘.
```

```text
반복적으로 들어오는 문의를 FAQ 형태로 정리해줘.
```

## 2-3. PoC2. 보고서 자동 생성 봇

### 실습 목표

기사, 메모, 링크 등을 입력하면 기존 보고서 형식에 맞춘 초안을 자동으로 생성하는 GPTs를 만든다고 가정합니다.

### 실습

- 기존 보고서 파일 샘플 다운로드
- 시나리오 확인
- 프롬프트 복사/붙여넣기
- GPTs 항목 채우기
- 결과 확인
- 공유

### 시나리오

- 기사 링크 입력
- 메모 입력
- 제목/핵심 이슈 입력
- 기존 보고서 형식 참고
- 팀 공유용 브리핑 초안 생성

### 프롬프트 예시

```text
이 GPT의 목적은 기사, 메모, 링크를 입력받아 기존 보고서 형식과 작성 규칙을 참고해 팀 공유용 보고서 초안을 생성하는 것이다.

이 GPT는 아래와 같이 동작한다.
1. 사용자가 기사, 링크, 메모 또는 핵심 이슈를 입력한다.
2. GPT는 제공된 정보와 업로드된 참고 보고서를 바탕으로 적절한 구조를 만든다.
3. 핵심 내용을 요약하고, 시사점과 추가 확인 항목을 구분해서 정리한다.
4. 기존 보고서의 형식은 참고하되 문장을 그대로 복사하지 않는다.
5. 짧고 명확한 bullet 중심으로 결과를 작성한다.
```

### GPTs 들어갈 항목 생성

- 이름
- 설명
- 지침
- 대화 스타터
- 참고 보고서 파일

# 3. Codex, 실행형 자동화와 코딩 에이전트

## 3-1. Codex란

Codex는 단순히 답변을 주는 챗봇이 아니라, **파일 생성, 코드 작성, 수정 방향 제안, 반복 작업 자동화**까지 도와주는 코딩 에이전트형 도구입니다.

## 3-2. ChatGPT vs Codex

설명 포인트:
- ChatGPT는 질문하고 답을 받는 데 강함
- Codex는 실제 작업 결과물을 만드는 데 더 강함
- 둘은 경쟁 관계라기보다 역할이 다름

## 3-3. 네비게이션 vs 자율주행

강의용 비유:
- ChatGPT는 네비게이션
- Codex는 자율주행

즉, ChatGPT는 방향을 잘 잡아주고,
Codex는 실제로 손발이 많이 가는 작업을 더 적극적으로 수행하는 느낌으로 설명할 수 있습니다.

## 3-4. Codex 실행하는 3가지 방법

- CLI
- App
- IDE

### 설명 포인트

- 초보자는 App 방식이 상대적으로 진입장벽이 낮음
- 익숙한 사람은 CLI나 IDE 연동이 더 강력할 수 있음

## 3-5. ChatGPT는 어렵고 Codex는 강한 작업 예시

예시:
- 파일 생성
- 다중 파일 수정
- 간단한 프로그램 만들기
- 자동화 스크립트 작성
- 브라우저 자동화 초안 만들기
- 메일 자동화 코드 만들기
- 반복 작업 정리용 스크립트 만들기
- 엑셀 병합 스크립트 작성
- 입력/처리/출력 구조가 있는 업무 자동화
- 나중에 재사용 가능한 스킬 형태로 정리

# 마무리

오늘 실습에서 가장 중요한 메시지는 아래와 같습니다.

- ChatGPT Enterprise는 바로 쓰는 업무 생산성 도구다.
- GPTs는 지식, 지침, 출력 형식을 설계해서 업무형 에이전트를 빠르게 만드는 도구다.
- Codex는 실제 파일 생성, 코드 작성, 자동화 초안 작성에 강한 도구다.
- PoC는 크게 시작하는 것이 아니라, 작은 실습에서 바로 검증해보는 것이 중요하다.

한 줄로 정리하면 이렇습니다.

**ChatGPT Enterprise, GPTs, Codex는 반복 업무를 줄이고, 문서와 지식을 활용한 실무형 PoC를 빠르게 검증하는 데 매우 유용한 도구입니다.**
