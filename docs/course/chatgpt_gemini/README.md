# ChatGPT Enterprise / GPTs / Codex 핸즈온 실습

## 강의 안내

이 문서는 **해외지원팀 AI PoC 인력 대상 핸즈온 실습**을 위해 준비한 **원페이지 강의 자료**입니다.

오늘 실습은 가능하면 여러 페이지를 왔다 갔다 하지 않고, **이 페이지 하나를 띄워놓고 순서대로 따라오는 방식**으로 진행합니다.

강사는 이 문서를 기준으로 설명하고, 교육생은 각자 PC에서 이 페이지를 열어둔 상태로 **예시 프롬프트를 복사/붙여넣기** 하며 실습을 진행합니다.

---

## 오늘 강의 목표

오늘 강의의 목표는 아래 3가지입니다.

- ChatGPT Enterprise의 핵심 기능을 업무 관점에서 직접 써본다.
- GPTs를 활용해 간단한 에이전트형 PoC를 만드는 흐름을 이해한다.
- 해외지원팀 업무에 연결할 수 있는 실습 아이디어를 얻는다.

---

## 진행 방식

- 강사는 PC와 TV를 연결해 이 페이지를 띄워놓고 설명합니다.
- 교육생은 각자 웹브라우저에서 이 페이지를 띄워놓습니다.
- 실습은 아래 순서대로 진행합니다.
- 예시 프롬프트는 모두 **코드블록 형태**로 제공하므로 그대로 복사/붙여넣기 하면 됩니다.
- 결과가 마음에 들지 않으면 추가 요청을 통해 더 다듬습니다.

---

## 오늘 실습 순서

1. 새 채팅 실습
2. 이미지 실습
3. 심층 리서치 실습
4. GPTs / RAG 기반 챗봇 실습
5. GPTs 활용 PoC 아이디어 소개
6. 마무리

---

# 1. 새 채팅 실습

## 실습 포인트

새 채팅은 가장 기본 기능이지만, 실제 업무에서는 가장 자주 쓰는 출발점입니다.

예를 들면 아래 같은 작업에 바로 활용할 수 있습니다.

- 영어 메일 초안 작성
- 회의 메모 정리
- 보고서 문장 다듬기
- 정보 비교 정리
- 업무 설명 문서 초안 만들기

## 실습 프롬프트 1

```text
아래 상황을 바탕으로 해외 파트너에게 보내는 정중한 영어 메일 초안을 작성해줘.

상황:
- 원래 전달 예정일은 이번 주 금요일
- 내부 검토가 지연되어 다음 주 수요일로 일정 변경 필요
- 최대한 협조에 감사하는 톤
- 너무 길지 않게 작성
```

## 실습 프롬프트 2

```text
아래 회의 메모를 바탕으로 핵심 내용만 정리해줘.

형식:
1. 결정사항
2. 담당자
3. 후속 액션
4. 리스크

회의 메모:
- 고객사 요청 사항은 이번 주 내 1차 회신 필요
- 시스템 로그는 김대리가 추가 확인
- 견적안은 다음 주 월요일까지 초안 작성
- 일정 지연 가능성이 있어 고객 커뮤니케이션 필요
- 기술팀과 지원팀 간 역할 구분을 문서로 남기기로 함
```

## 실습 프롬프트 3

```text
아래 문장을 보고서에 들어갈 수 있게 더 자연스럽고 전문적인 표현으로 다듬어줘.

문장:
현재 고객사 요청사항이 많아서 대응이 늦어지고 있고, 관련 부서랑 같이 보고 있는데 아직 정확한 일정은 확인이 안 된 상태입니다.

조건:
- 너무 과장되지 않게
- 회사 보고서 문체로
- 2가지 버전 제시
```

## 추가 요청 예시

```text
조금 더 간결하게 다시 작성해줘.
```

```text
해외지원팀에서 바로 쓸 수 있게 바꿔줘.
```

```text
영어와 한국어 버전을 같이 보여줘.
```

# 2. 이미지 실습

## 실습 포인트

이미지 기능은 단순 재미 요소가 아니라, 발표자료, 보고서, 교육자료, 설명용 시각화 자료를 빠르게 만드는 데 쓸 수 있습니다.

## 실습 프롬프트 1

```text
해외지원팀의 AI 업무 활용 교육 자료 표지에 어울리는 이미지를 만들어줘.

조건:
- 전체 분위기는 깔끔하고 전문적
- AI, 글로벌 협업, 업무 생산성 향상 느낌이 함께 보이면 좋음
- 발표 자료 첫 장에 들어갈 수 있는 스타일
- 텍스트는 이미지 안에 넣지 말 것
- 너무 화려하지 않고 회사 교육자료에 어울리게
```

## 실습 프롬프트 2

```text
아래 업무 흐름을 한눈에 이해할 수 있는 일러스트 스타일 이미지로 만들어줘.

업무 흐름:
- 해외 법인에서 문의 접수
- 본사 지원팀에서 내용 검토
- 관련 부서 협의
- 결과 회신

조건:
- 발표 자료에 넣기 좋게 단순하고 명확한 구성
- 아이콘이나 흐름 느낌이 잘 보이게
- 복잡한 텍스트는 넣지 말 것
- 비즈니스 문서에 어울리는 색감
```

## 실습 프롬프트 3

```text
RAG 기반 챗봇의 개념을 비전문가도 직관적으로 이해할 수 있게 설명하는 이미지를 만들어줘.

조건:
- 사용자의 질문이 들어오고
- 사내 문서 또는 지식 저장소를 참고한 뒤
- 답변이 생성되는 흐름이 느껴지게
- 너무 기술 다이어그램처럼 어렵지 않게
- 교육 자료에 넣기 좋은 깔끔한 스타일
- 텍스트는 최소화하거나 넣지 말 것
```

## 추가 요청 예시

```text
좀 더 미니멀한 스타일로 다시 만들어줘.
```

```text
발표 자료에 넣을 거라서 더 단정하게 정리해줘.
```

```text
비즈니스 문서용 느낌으로 바꿔줘.
```

# 3. 심층 리서치 실습

## 실습 포인트

심층 리서치는 단순 검색이 아니라, 업무용 조사 메모나 브리핑 초안을 빠르게 얻는 데 강점이 있습니다.

## 실습 프롬프트 1

```text
국내외 기업들이 생성형 AI를 실제 업무에 도입한 사례를 조사해줘.

조건:
- 고객지원, 사내지식검색, 문서작성, 보고서 자동화 중심
- 국내 사례와 해외 사례를 구분해서 정리
- 각 사례별로 무엇을 자동화했는지, 기대효과는 무엇인지 요약
- 마지막에는 우리 회사 해외지원팀 관점에서 참고할 만한 시사점 5가지를 정리
```

## 실습 프롬프트 2

```text
ChatGPT Enterprise, Microsoft Copilot, Google Gemini for Workspace를 업무 활용 관점에서 비교 조사해줘.

조건:
- 비교 항목: 문서작성 지원, 데이터 활용, 협업성, 보안/관리, 사용자 편의성
- 표로 먼저 정리
- 그 다음 각 서비스의 강점과 약점을 요약
- 마지막에는 해외지원팀의 실무 적용 관점에서 어떤 상황에 어떤 도구가 적합할지 의견 제시
```

## 실습 프롬프트 3

```text
아래 주제로 10분 안에 읽을 수 있는 회의 전 브리핑 문서를 만들어줘.

주제:
해외지원 조직에서 생성형 AI를 활용해 반복 업무를 줄이는 방안

조건:
- 배경
- 주요 활용 영역
- 기대효과
- 리스크
- 먼저 작게 시작할 수 있는 PoC 아이디어
- 경영진에게 설명할 때 사용할 수 있는 핵심 메시지 3개 포함
```

## 추가 요청 예시

```text
핵심만 남기고 경영진 보고용 1페이지 요약으로 바꿔줘.
```

```text
표, bullet, 결론 형식으로 다시 정리해줘.
```

```text
당장 실행 가능한 아이디어만 추려줘.
```

# 4. GPTs / RAG 기반 챗봇 실습

## 실습 포인트

이 파트에서는 GPTs를 활용해 **문서 기반 질의응답 챗봇**을 만드는 흐름을 이해합니다.

핵심은 아래와 같습니다.

- 사용자가 질문한다.
- GPT가 업로드된 문서 또는 지식 파일을 참고한다.
- 문서 근거 기반으로 답변한다.
- 모르는 내용은 추측하지 않고 추가 확인을 안내한다.

즉, “그냥 대답을 잘하는 챗봇”이 아니라, **업무 문서를 참고하는 에이전트형 챗봇**을 만드는 개념입니다.

## 예시 시나리오

해외지원팀에서 반복적으로 받는 질문에 대해, 내부 가이드 문서를 기반으로 1차 답변을 제공하는 GPTs를 만든다고 가정합니다.

예시 질문:
- 경비 정산 시 먼저 확인해야 하는 항목은 무엇인가?
- 특정 업무 요청은 어느 부서로 문의해야 하는가?
- 해외지원 프로세스는 어떻게 진행되는가?
- 자주 발생하는 요청 유형별 기본 대응은 무엇인가?

## GPTs 지침 예시

```text
이 GPT의 목적은 해외지원팀 관련 반복 질문에 대해, 업로드된 가이드 문서를 근거로 1차 답변을 제공하는 것이다.

이 GPT는 아래와 같이 동작한다.
1. 사용자가 해외지원 업무 관련 질문을 입력한다.
2. GPT는 업로드된 문서에서 관련 내용을 먼저 확인한다.
3. 문서 근거를 바탕으로 간결하고 실무적으로 답변한다.
4. 문서에 없는 내용은 추측하지 않는다.
5. 문서에 없는 경우에는 담당 부서 또는 추가 확인 필요 사항을 안내한다.
6. 필요 시 관련 질문 예시를 함께 제안한다.

이 GPT는 아래 원칙을 반드시 지켜야 한다.
- 업로드된 문서를 근거로 답변할 것
- 근거 없는 추측을 하지 말 것
- 짧고 명확하게 답변할 것
- 사용자가 바로 다음 행동을 할 수 있게 안내할 것
- 문서에 없는 내용을 아는 척하지 말 것
```

## GPTs 테스트용 질문 예시

```text
해외지원 요청 접수 후 기본 처리 절차를 간단히 설명해줘.
```

```text
이 질문이 문서에 없으면 누구에게 확인해야 하는지 안내해줘.
```

```text
반복적으로 들어오는 문의를 FAQ 형태로 정리해줘.
```

## 실습 진행 멘트 예시

- 여기서는 GPTs를 단순 챗봇이 아니라, 문서를 참고하는 업무용 에이전트라고 생각하시면 됩니다.
- 중요한 것은 모델 자체보다도, 어떤 문서를 붙이고 어떤 지침을 주느냐입니다.
- 실무에서는 FAQ, 운영가이드, 정책문서, 프로세스 문서를 붙이면 바로 PoC가 가능합니다.

# 5. GPTs 활용 PoC 아이디어

오늘 바로 다 실습하지 않더라도, 아래와 같은 방향으로 빠르게 PoC를 만들 수 있습니다.

## PoC 아이디어 1. RAG 기반 챗봇
- 내부 가이드 문서 기반 Q&A
- 반복 문의 1차 대응
- FAQ 자동화

## PoC 아이디어 2. 보고서 자동 생성 봇
- 기사, 메모, 링크 입력
- 기존 보고서 형식에 맞춘 초안 생성
- 팀 공유용 브리핑 자동화

## PoC 아이디어 3. 정형 데이터 분석 보조 GPT
- CSV 또는 엑셀 기반 요약
- 차이 분석
- 보고서 문장 초안 생성

## PoC 아이디어 4. 액션 연동형 GPT
- 외부 API 호출
- 특정 시스템과 연계
- 결과물 자동 반환

핵심은 **완벽한 시스템을 한 번에 만드는 것보다, 작게 시작해 빠르게 검증하는 것**입니다.

# 6. 마무리

오늘 실습에서 가장 중요한 메시지는 아래와 같습니다.

- 새 채팅은 가장 빠른 업무 활용 출발점이다.
- 이미지와 심층 리서치는 결과물의 폭을 넓혀준다.
- GPTs는 지식, 지침, 출력 형식을 설계해서 업무형 에이전트를 빠르게 만드는 도구다.
- PoC는 크게 시작하는 것이 아니라, 작은 실습에서 바로 검증해보는 것이 중요하다.

한 줄로 정리하면 이렇습니다.

**ChatGPT Enterprise와 GPTs는 반복 업무를 줄이고, 문서와 지식을 활용한 실무형 PoC를 빠르게 검증하는 데 매우 유용한 도구입니다.**

감사합니다.
