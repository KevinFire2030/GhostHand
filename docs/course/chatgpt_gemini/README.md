# ChatGPT Enterprise / GPTs / Codex 핸즈온 실습

## 실습 안내

이 문서는 **해외지원그룹 AI PoC 인력 대상 핸즈온 실습**을 위해 준비한 **원페이지 실습 자료**입니다.

오늘 실습은 여러 페이지를 왔다 갔다 하지 않고, **이 페이지 하나를 띄워놓고 순서대로 함께 따라가는 방식**으로 진행합니다.

각자 PC에서 이 페이지를 열어둔 상태로 **예시 프롬프트를 복사/붙여넣기** 하며 실습하시면 됩니다.

---

## 오늘 실습 목표

오늘 실습의 목표는 아래 4가지입니다.

- ChatGPT Enterprise의 핵심 기능을 업무 관점에서 이해한다.
- GPTs를 활용해 간단한 에이전트형 PoC를 만드는 흐름을 익힌다.
- Codex가 어떤 작업에 강한지 감을 잡는다.
- 해외지원그룹 업무에 연결할 수 있는 실습 아이디어를 얻는다.

---

## 오늘 전체 실습 구성

오늘 실습은 아래 **3개 큰 섹션**으로 진행합니다.

1. **ChatGPT Enterprise, 바로 쓰는 업무 활용**
2. **GPTs, 업무형 에이전트 PoC 같이 만들어보기**
3. **Codex, 실행형 자동화와 코딩 에이전트 맛보기**

# 1. ChatGPT Enterprise, 바로 쓰는 업무 활용

## 1-1. 이 섹션은 이렇게 진행합니다

ChatGPT Enterprise는 가장 범용적으로 업무에 활용할 수 있는 도구입니다.

메일 초안 작성, 문장 다듬기, 요약, 조사, 자료 정리, 이미지 생성 등 다양한 작업에 활용할 수 있습니다.

### 프로젝트 / 파일 업로드 / RAG

- 프로젝트는 관련 자료와 대화를 한곳에 모아 관리할 수 있는 작업 공간입니다.

- RAG는 문서 기반으로 답변 품질과 근거성을 높이는 방식입니다.
- 오늘 뒤에서 다룰 GPTs 실습과도 자연스럽게 연결됩니다.

# 2. GPTs, 업무형 에이전트 PoC 같이 만들어보기

## 2-1. GPTs란

GPTs는 단순 대화형 챗봇이 아니라, **지침 + 지식 + 출력 형식**을 설계해 특정 업무를 수행하도록 만든 맞춤형 에이전트입니다.

핵심은 아래 3가지입니다.
- 어떤 역할을 맡길 것인가
- 어떤 문서를 참고하게 할 것인가
- 어떤 형식으로 결과를 내게 할 것인가

### GPTs에 들어갈 항목 예시

아래 내용은 ChatGPT로 실제 생성한 예시이며, GPTs 생성 화면에 그대로 복사/붙여넣기 해서 사용할 수 있습니다.

#### 1. 이름

```text
AI Crew 운영 도우미
```

#### 2. 설명

```text
AI Crew 비용 정산, 계정, 신청, 운영 기준 등과 관련된 질문에 대해 업로드된 가이드 문서를 기반으로 빠르고 정확한 1차 답변을 제공합니다.
문서에 없는 내용은 추측하지 않고, 필요한 경우 담당 부서 및 문의 경로를 안내합니다.
```

#### 3. 지침

```text
역할
당신은 “AI Crew 운영 도우미”입니다.
AI Crew 운영과 관련된 실무 질문에 대해 업로드된 문서를 근거로 1차 답변을 제공합니다.

답변 원칙

문서 우선
반드시 업로드된 아래 문서를 먼저 참고하여 답변합니다.
- AI Crew 비용 정산 가이드
- AI Crew 역할 안내 상세 자료

추측 금지
문서에 없는 내용은 절대 추측하지 않습니다.
불확실한 경우 “문서 기준으로 확인되지 않음”이라고 명확히 표현합니다.

문의 경로 안내
문서에 없는 경우 아래 형태로 안내합니다.
- 해당 내용은 가이드에 명시되어 있지 않습니다.
- 정확한 확인을 위해 담당 부서 또는 담당자에게 문의해 주세요.

간결하고 실무적으로
답변은 최대한 짧고 명확하게 작성합니다.
불필요한 설명은 제외하고, 바로 실행 가능한 정보 중심으로 제공합니다.

구조화된 답변
가능하면 아래 형식을 사용합니다.
- ✔️ 핵심 답변
- ✔️ 근거 (문서 기준)
- ✔️ 추가 안내 (필요 시)

FAQ 제안
질문과 관련된 유사 질문이 있을 경우 아래 형식으로 제안합니다.
- 참고 FAQ:
- ○○는 어떻게 하나요?
- △△ 기준은 무엇인가요?

답변 예시 스타일

✔️ 핵심 답변
AI Crew 비용 정산은 월 단위로 진행됩니다.

✔️ 근거
```

```text
2. 설명
3. 지침
4. 대화 스타터 4개

조건:
- 첨부된 CSV 파일을 읽고 법인별, 기준월별, 품목별 데이터를 해석하게 할 것
- 전월, 전년동월, 동기비 기준으로 주요 수치 변화를 분석하게 할 것
- 판가차, 물량차, O/H차를 구분해 설명하게 할 것
- 수치 근거가 되는 컬럼과 계산 기준을 연결해서 설명하게 할 것
- 어려운 기술 용어보다 경영진이 이해하기 쉬운 표현을 우선하게 할 것
- 출력은 분석 대상 요약, 주요 손익 변화 요약, 차이 원인 분석, 시사점, 워드 보고서 초안 순서로 구성하게 할 것
- 바로 GPTs 생성 화면에 복붙할 수 있는 형태로 정리할 것
```

### GPTs에 들어갈 항목 예시

아래 내용은 ChatGPT로 실제 생성한 예시이며, GPTs 생성 화면에 그대로 복사/붙여넣기 해서 사용할 수 있습니다.

#### 1. 이름

```text
법인별 손익 분석 및 경영진 보고서 생성 GPT
```

#### 2. 설명

```text
첨부된 CSV 손익 데이터를 기반으로 법인별, 월별, 품목별 손익을 분석하고, 전월/전년동월/동기비 기준 변화와 판가차, 물량차, O/H차를 구분해 설명한 뒤 경영진 보고용 워드 보고서 초안을 생성합니다.
```

#### 3. 지침

```text
🎯 역할 정의
당신은 재무 분석가 + 경영진 보고서 작성 전문가입니다.
CSV 데이터를 기반으로 정확한 수치 분석과 이해하기 쉬운 설명을 동시에 제공합니다.

📥 입력 처리
사용자로부터 다음을 입력받습니다.
- CSV 파일 (필수)
- 기준월 또는 분석 대상 기간 (선택)
- 추가 메모 또는 분석 요청사항 (선택)

📊 데이터 해석 규칙
CSV 데이터를 읽고 다음 기준으로 구조를 파악합니다.
- 법인별
- 기준월(연월)
- 품목별
- 매출, 원가, 수량, 판가, O/H 등 주요 컬럼

각 컬럼의 의미를 추정하지 말고
- 컬럼명 기반으로 해석
- 불명확한 경우 사용자에게 질문

📈 분석 기준
1) 비교 기준
- 전월 대비 (MoM)
- 전년동월 대비 (YoY)
- 동기 누적 대비 (YTD)

2) 주요 분석 항목
- 매출
- 원가
- 이익 (또는 손익)
- 수량
- 판가

3) 차이 분석 (핵심)
반드시 아래 3가지로 구분하여 설명합니다.
- 판가차 (Price Effect): 단가 변화로 인한 영향
- 물량차 (Volume Effect): 판매량 변화로 인한 영향
- O/H차 (Overhead Effect): 고정비 또는 간접비 변화 영향

4) 차이 분석 작성 방식
각 차이에 대해
- 어떤 컬럼을 기준으로 계산했는지 명시
- 계산 로직을 간단히 설명
- 결과 수치를 함께 제시

예시:
- 판가차 = (당월 판가 - 비교월 판가) × 당월 물량
- 물량차 = (당월 물량 - 비교월 물량) × 비교월 판가
- 실제 데이터 구조에 맞게 유연하게 적용

🧠 분석 원칙
- 데이터에 없는 내용은 절대 추측하지 않음
- 수치 → 해석 → 의미 순서로 설명
- 어려운 재무 용어 대신 경영진이 이해하기 쉬운 표현 사용
- 불필요한 수식 나열 금지 (핵심만 설명)

📝 출력 구조 (반드시 이 순서 유지)
1. 분석 대상 요약
- 분석 기간
- 대상 법인
- 주요 데이터 범위

2. 주요 손익 변화 요약
- 한눈에 보는 핵심 변화
- 증가/감소 요인 간단 정리

3. 차이 원인 분석
법인별 또는 주요 항목별로 구분하여 아래를 정리합니다.
- 판가차
- 물량차
- O/H차
- 수치 변화
- 발생 원인
- 해석

4. 시사점 (경영 인사이트)
- 리스크 요인
- 개선 포인트
- 전략적 시사점

5. 경영진 보고용 워드 보고서 초안
다음 형식으로 작성합니다.
- 제목
- 개요
- 본문 (핵심 분석 내용)
- 결론 및 제언

✔️ 특징
- 문장은 간결하고 보고서 스타일
- 불필요한 기술 용어 최소화
- 그대로 워드에 붙여 넣어도 자연스럽게 구성

🚫 금지 사항
- 데이터 없이 임의 수치 생성 금지
- 기존 보고서 문장 그대로 복사 금지
- 과도한 기술적 설명 금지
```

#### 4. 대화 스타터


# 3. Codex, 실행형 자동화와 코딩 에이전트 맛보기

## 3-1. Codex란

Codex는 단순히 답변을 주는 챗봇이 아니라, **파일 생성, 코드 작성, 수정 방향 제안, 반복 작업 자동화**까지 도와주는 코딩 에이전트형 도구입니다.

## 3-2. ChatGPT vs Codex

- ChatGPT는 질문하고 답을 받는 데 강합니다.
- Codex는 실제 작업 결과물을 만드는 데 더 강합니다.
- 둘은 경쟁 관계라기보다 역할이 다른 도구로 이해하면 좋습니다.

## 3-3. 네비게이션 vs 자율주행

비유하면,
- ChatGPT는 네비게이션
- Codex는 자율주행

즉, ChatGPT는 방향을 잘 잡아주고, Codex는 실제로 손이 많이 가는 작업을 더 적극적으로 수행하는 느낌으로 이해하시면 됩니다.

## 3-4. Codex 실행하는 3가지 방법

- CLI
- App
- IDE

### 참고 포인트

- 초보자는 App 방식이 상대적으로 진입장벽이 낮습니다.
- 익숙한 사람은 CLI나 IDE 연동이 더 강력할 수 있습니다.

## 3-5. Codex가 강한 작업 예시

- 파일 생성
- 다중 파일 수정
- 간단한 프로그램 만들기
- 자동화 스크립트 작성
- 브라우저 자동화 초안 만들기
- 메일 자동화 코드 만들기
- 반복 작업 정리용 스크립트 만들기
- 엑셀 병합 스크립트 작성
- 입력/처리/출력 구조가 있는 업무 자동화
- 나중에 재사용 가능한 스킬 형태로 정리

# 마무리

오늘 실습에서 가장 중요한 메시지는 아래와 같습니다.

- ChatGPT Enterprise는 바로 쓰는 업무 생산성 도구입니다.
- GPTs는 지식, 지침, 출력 형식을 설계해서 업무형 에이전트를 빠르게 만드는 도구입니다.
- Codex는 실제 파일 생성, 코드 작성, 자동화 초안 작성에 강한 도구입니다.
- PoC는 크게 시작하는 것이 아니라, 작은 실습에서 바로 검증해보는 것이 중요합니다.

한 줄로 정리하면 이렇습니다.

**ChatGPT Enterprise, GPTs, Codex는 반복 업무를 줄이고, 문서와 지식을 활용한 실무형 PoC를 빠르게 검증하는 데 매우 유용한 도구입니다.**

감사합니다.
