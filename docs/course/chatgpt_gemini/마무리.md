# 마무리

오늘은 GPTs를 활용해서 AX PM 그룹 관점의 AX 과제 PoC를 어떻게 빠르게 만들 수 있는지 실습 중심으로 살펴봤습니다.

지금까지 본 내용을 유형으로 정리하면 다음과 같습니다.

- PoC1: 문서 기반 RAG 챗봇
- PoC2: 외부 뉴스 기반 보고서 초안 생성
- PoC3: 정형 데이터 기반 분석 및 보고서 생성
- PoC4: 외부 API 연동형 자동화 GPT

이 네 가지를 보면,
GPTs는 단순한 대화형 챗봇이 아니라
업무에 필요한 입력을 받고,
지식을 참고하고,
정해진 규칙대로 결과물을 만들어내는 빠른 PoC 도구로 활용할 수 있다는 점을 보실 수 있습니다.

중요한 것은 모델 자체보다,
- 어떤 입력을 받을지
- 어떤 지식을 붙일지
- 어떤 지침을 줄지
- 어떤 출력 형식을 요구할지
를 얼마나 명확하게 설계하느냐입니다.

특히 AX PM 관점에서는 완성형 시스템을 처음부터 만드는 것보다,
우선 GPTs로 가능성을 빠르게 검증하고,
그 다음에 실제 서비스나 업무 프로세스로 확장하는 접근이 훨씬 현실적입니다.

오늘 실습의 핵심을 한 줄로 정리하면 이렇습니다.

**GPTs는 AX 과제 아이디어를 빠르게 PoC로 만들어보고, 업무 적용 가능성을 검증하는 데 매우 유용한 도구입니다.**

감사합니다.
