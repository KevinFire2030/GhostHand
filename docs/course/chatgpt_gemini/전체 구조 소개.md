# 전체 구조 소개

지금 보고 계신 `docs/course/chatgpt_gemini` 폴더에는 GPTs 기반 AX 과제 PoC 예시가 유형별로 정리되어 있습니다.
오늘은 이걸 순서대로 보면서, GPTs가 어떤 업무에 적용될 수 있는지 같이 살펴보겠습니다.

## 1. GPTs_PoC1_생성가이드
첫 번째는 **RAG 기반 운영 도우미**입니다.
예를 들어 AI Crew 운영 관련 FAQ나 Q&A에 대해,
미리 올려둔 문서를 바탕으로 1차 답변을 해주는 챗봇입니다.

이 유형은 사내 가이드 문서, FAQ, 제도 안내, 운영 규정처럼
문서 기반으로 반복 질의응답이 많은 업무에 잘 맞습니다.

## 2. GPTs_PoC2_생성가이드
두 번째는 **외부 뉴스 기반 내부 보고서 초안 생성**입니다.
외부 기사 링크나 제목을 입력하면,
기존 보고서 형식과 작성 Rule을 참고해서 내부 공유용 보고서 초안을 만드는 방식입니다.

이 유형은 AX PM 조직에서 특히 활용도가 높습니다.
외부 동향을 빠르게 정리해서 보고 자료로 바꿔야 할 때,
GPTs가 초안 작성 시간을 크게 줄여줄 수 있습니다.

## 3. GPTs_PoC3_생성가이드
세 번째는 **정형 데이터 기반 분석형 GPT**입니다.
샘플 CSV 같은 데이터를 넣고,
손익 분석 결과나 보고서 초안을 생성하는 형태입니다.

이 유형은 단순 문서 질의응답보다 한 단계 더 나아가,
데이터 구조 정의와 산출식 설계가 중요해지는 케이스입니다.
즉, GPT를 잘 만드는 것보다 입력 데이터와 계산 기준을 잘 설계하는 것이 더 중요합니다.

## 4. GPTs_PoC4_생성가이드
네 번째는 **외부 Action/API 연동형 GPT**입니다.
예시로는 HeyGen API를 연결해서,
사용자가 입력한 텍스트를 아바타 영상으로 생성하는 방식입니다.

이 유형은 GPT가 단순히 텍스트를 생성하는 수준을 넘어서,
외부 시스템을 호출하고 결과물을 받아오는 자동화 형태로 확장되는 사례입니다.

## 5. 오늘 실습에서 보면 좋은 포인트
오늘은 각 PoC를 보실 때 아래를 중심으로 보시면 좋습니다.

- 이 GPT의 입력은 무엇인가?
- 이 GPT가 참고하는 지식은 무엇인가?
- 지침은 얼마나 구체적으로 줘야 하는가?
- 출력 결과는 실무에서 바로 쓸 수 있는 수준인가?
- 향후 실제 서비스로 확장하려면 무엇이 더 필요할까?

즉, 오늘은 GPTs를 예쁘게 만드는 시간이 아니라,
업무에 맞는 입력, 지식, 지침, 출력 구조를 어떻게 설계하는지 보는 시간이 되면 좋겠습니다.

그럼 이제 첫 번째 PoC부터 하나씩 보겠습니다.
