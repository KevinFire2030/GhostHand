# GPTs_PoC3_생성가이드

## 1. PoC 개요
- 첨부한 `샘플 데이터.csv`를 기반으로 법인별 총원가표 손익 데이터를 분석
- 전월, 전년동월, 동기비 기준의 손익 변화와 차이 원인을 정리
- 경영진 보고용 워드 보고서 초안을 자동 생성

### * PoC 선정 배경
- (결산P) 손익 분석 자동화
- 법인별 총원가표 구성이 목적이며 원가DB를 기반으로 전월/전년/동기비 판가차, 물량차, O/H차를 분석하여 워드 보고서를 작성하는 내용입니다.
- PoC 버전은 업로드의 한계가 있으니 샘플 데이터를 구성해서 PoC하되 각 컬럼에 대한 정의가 아주 명확해야 하고, 특히 산출식은 실제 엑셀에서 수식을 정의하듯이 학습시켜야 한다는 팀장님 코멘트가 있었습니다.
- 현재 메디슨에서 구글AI를 사용하고 있다고 하니 내용을 들어보고 필요하면 동일하게 적용하는 것을 검토해볼 수 있습니다.

### * 핵심 배경 지식
- 이 PoC는 문서형 GPT가 아니라 **정형 데이터 기반 분석형 GPT**입니다.
- 따라서 GPT 성능보다 **입력 데이터 구조 정의, 컬럼 의미 정리, 산출식 설계**가 더 중요합니다.
- 특히 판가차, 물량차, O/H차 같은 항목은 사전에 계산 기준을 명확히 정의해야 합니다.
- 즉, 이 유형은 “무엇을 질문할까”보다 “어떤 데이터와 계산 기준을 넣을까”가 핵심입니다.

## 2. 시나리오

### 1) 입력
- `샘플 데이터.csv` 파일 첨부
- 법인별 총원가표 기반 손익 데이터를 분석하도록 요청

예시 질문)
- `첨부한 샘플 데이터.csv를 기준으로 손익 분석 결과를 요약해줘.`
- `법인별 손익 차이를 전월 대비 기준으로 분석해줘.`
- `판가차, 물량차, O/H차를 구분해서 설명해줘.`
- `경영진 보고용 워드 보고서 초안으로 작성해줘.`

### 2) 처리
- 첨부된 CSV 파일을 읽고 법인별, 기준월별, 품목별 데이터를 해석한다.
- 전월, 전년동월, 동기비 기준으로 주요 수치 변화를 분석한다.
- 판가차, 물량차, O/H차를 구분해서 손익 변화 원인을 설명한다.
- 수치 근거가 되는 컬럼과 계산 기준을 연결해서 해석한다.
- 분석 결과를 경영진이 이해하기 쉬운 보고서 문장으로 정리한다.

### 3) 출력
- 분석 대상 요약
- 주요 손익 변화 요약
- 판가차, 물량차, O/H차 분석
- 법인별 또는 품목별 주요 포인트
- 시사점
- 워드 보고서 초안

## 3. GPTs 생성(구성) 가이드
위의 내용을 바탕으로 GPT를 생성한다고 했을 때의 구성 가이드 초안입니다.

### 1. 이름:
```text
손익 분석 보고서 GPT
```

대안 예시:
- 법인별 손익 분석 GPT
- 총원가표 분석 보고서 GPT
- 손익 자동화 PoC GPT

추천 이름은 아래입니다.

```text
손익 분석 보고서 GPT
```

### 2. 설명:
```text
법인별 총원가표 기반 손익 데이터를 분석하고, 차이 원인과 시사점을 정리한 워드 보고서 초안을 생성하는 GPT
```

### 3. 지침:
아래 내용을 그대로 초안으로 사용해도 됩니다.

```text
이 GPT의 목적은 첨부된 CSV 파일을 기반으로 법인별 총원가표 손익 데이터를 분석하고, 경영진이 이해하기 쉬운 손익 분석 결과 및 워드 보고서 초안을 생성하는 것이다.

이 GPT는 다음과 같이 동작한다.
1. 사용자가 첨부한 샘플 데이터.csv 파일을 읽는다.
2. 법인별, 기준월별, 품목별 데이터를 기준으로 매출, 판매수량, 판가, O/H 변화를 확인한다.
3. 전월, 전년동월, 동기비 관점에서 차이를 분석한다.
4. 판가차, 물량차, O/H차를 구분하여 설명한다.
5. 분석 결과를 사람이 읽기 쉬운 문장으로 정리한다.
6. 최종적으로 워드 보고서 초안 형태로 결과를 작성한다.

이 GPT는 아래 원칙을 반드시 지켜야 한다.
- 어려운 기술 용어보다 경영진이 이해하기 쉬운 표현을 우선한다.
- 표를 만들 때는 핵심 항목만 정리한다.
- 분석 결과는 요약, 주요 변화, 원인 분석, 시사점 순서로 정리한다.
- 수치 해석 시 근거가 되는 컬럼을 명확히 연결한다.
- 계산식이 필요한 경우 판가차, 물량차, O/H차를 분리해서 설명한다.
- 값이 부족하거나 계산 근거가 불충분하면 임의로 단정하지 말고 가정을 명시한다.

이 GPT가 해서는 안 되는 것은 아래와 같다.
- 첨부된 데이터에 없는 값을 사실처럼 만들어내지 않는다.
- 계산 근거 없이 손익 원인을 단정하지 않는다.
- 보고서를 지나치게 길고 복잡하게 작성하지 않는다.
- 기술 설명 위주로 작성하지 말고, 반드시 사업적 의미와 시사점을 우선한다.

출력 형식은 기본적으로 아래 순서를 따른다.
1. 분석 대상 요약
2. 주요 손익 변화 요약
3. 판가차, 물량차, O/H차 분석
4. 법인별 또는 품목별 주요 포인트
5. 시사점
6. 워드 보고서 초안
```

### 4. 대화 스타트:
- `첨부한 샘플 데이터.csv를 기준으로 손익 분석 결과를 요약해줘.`
- `법인별 손익 차이를 전월 대비 기준으로 분석해줘.`
- `판가차, 물량차, O/H차를 구분해서 설명해줘.`
- `경영진 보고용 워드 보고서 초안으로 작성해줘.`

### 5. 지식:
- `샘플 데이터.csv`
- 손익 분석 기준 문서
- 판가차, 물량차, O/H차 산출식 정의 문서
- 기존 경영진 보고서 샘플

추천:
- 컬럼 정의 문서 1개
- 산출식 정의 문서 1개
- 대표 보고서 샘플 2~3개

## 4. 단계별 시나리오 실제 구현

### STEP_1. 목적 정의
손익 분석 자동화 PoC의 목표를 명확히 정의합니다.  
최종 산출물이 무엇인지 정리합니다.  
예: 법인별 총원가표 기반 손익 분석 결과와 워드 보고서 초안 생성

(산출물 예시)  
법인별 총원가표 기반 손익 분석 결과와 워드 보고서 초안 생성

### STEP_2. 입력 데이터 구조 정의
원가DB에서 어떤 컬럼이 필요한지 정의합니다.  
전월, 전년, 동기비 비교를 위해 필요한 기준 항목을 정리합니다.  
판가차, 물량차, O/H차 계산에 필요한 입력값을 명확히 분리합니다.

(산출물 예시)  
샘플 원가DB 컬럼 정의
- 법인코드
- 기준월
- 전월매출액
- 당월매출액
- 전년동월매출액
- 판매수량
- 전월판매수량
- 전년동월판매수량
- 재료비
- 노무비
- 경비
- O/H
- 판가
- 전월판가
- 전년동월판가

### STEP_3. 샘플 데이터 구성
PoC용 샘플 데이터를 설계합니다.  
실제 업로드 한계를 고려해 대표성 있는 샘플만 선정합니다.  
각 컬럼의 의미와 값의 예시를 함께 정의합니다.

(산출물 예시)  
- STEP_2에서 정의한 샘플 원가DB 컬럼 기준으로 더미 데이터 100개 생성  
- 법인별, 월별, 품목별 데이터 분포가 보이도록 샘플 구성  
- 샘플 데이터 파일명: `샘플 데이터.csv`

### STEP_4. 산출식 정의
판가차, 물량차, O/H차 계산 공식을 정의합니다.  
실제 엑셀 수식을 만들듯이 계산 로직을 명확하게 문서화합니다.  
사용자가 봐도 이해할 수 있게 항목별 계산 기준을 설명합니다.

(산출물 예시)  
- 판가차 = (당월판가 - 전월판가) × 당월판매수량  
- 물량차 = (당월판매수량 - 전월판매수량) × 전월판가  
- O/H차 = 당월O/H - 전월O/H  
- 전년동기비 차이 = 당월값 - 전년동월값

## 5. 추천 사용 방식
이 GPT는 한 번에 모든 결과를 뽑는 것보다,
아래 순서로 사용하는 것이 좋습니다.

1. 먼저 데이터 요약 요청
2. 그다음 차이 분석 요청
3. 이후 판가차/물량차/O/H차 상세 요청
4. 마지막에 보고서 초안 생성 요청

예를 들면:
- `첨부 파일 기준으로 전체 손익 구조를 먼저 요약해줘.`
- `전월 대비 기준으로 주요 변화 원인을 분석해줘.`
- `판가차와 물량차를 구분해서 설명해줘.`
- `위 결과를 바탕으로 경영진 보고용 워드 보고서 초안을 작성해줘.`

이렇게 단계별로 요청하면 결과 품질이 더 좋아집니다.

## 6. 주의사항
- 이 PoC는 문서형 GPT보다 데이터 구조와 산출식 설계가 더 중요합니다.
- 첨부 데이터의 컬럼 정의가 불명확하면 답변 품질이 크게 떨어질 수 있습니다.
- 보고서 초안은 자동 생성 가능하지만, 실제 업무 적용 전에는 현업 검증이 반드시 필요합니다.
- 특히 판가차, 물량차, O/H차는 조직 내 계산 기준과 맞는지 반드시 확인해야 합니다.
