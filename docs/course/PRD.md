# PRD - 메타버스 기반 에이전트 교육 플랫폼

## 1. 제품 개요
이 제품은 사용자가 메타버스 공간 안에서 놀이공원을 탐험하듯 학습하고, 레고블럭 조립하듯 에이전트를 만들며, 미션을 완료할 때마다 라이센스와 보상을 받는 온라인 교육 플랫폼이다.

핵심 컨셉은 다음 세 가지의 결합이다.
- 메타버스형 학습 공간
- 레고블럭 조립식 에이전트 빌더
- 게이미피케이션 기반 학습 경험

이 제품은 기존의 지루한 온라인 강의를 넘어, 체험형이고 몰입감 있는 AI/에이전트 교육 경험을 제공하는 것을 목표로 한다.

---

## 2. 문제 정의
기존 온라인 교육은 다음과 같은 한계를 가진다.
- 강의 영상과 문서 중심이라 몰입감이 낮다.
- 학습자가 수동적으로 소비하는 구조가 많다.
- 실제로 무언가를 “만들어본 경험” 없이 끝나는 경우가 많다.
- 기술 교육은 어렵고 추상적으로 느껴진다.
- 완주 동기가 약해 중도 이탈이 많다.

특히 AI 에이전트 교육은 개념, 구조, 툴, 스킬, 자동화, 활용 사례 등이 복합적으로 얽혀 있어 초보자가 접근하기 어렵다.

---

## 3. 제품 목표
### 주요 목표
- 사용자가 재미있게 AI 에이전트를 학습하도록 만든다.
- 학습자가 직접 툴, 스킬, 에이전트를 만들어보는 경험을 제공한다.
- 학습 몰입도를 높여 완주율을 높인다.
- 비개발자도 접근 가능한 에이전트 교육 경험을 만든다.

### 성공 기준
- 사용자가 최소 1개의 에이전트를 완성한다.
- 학습자가 과정을 재미있다고 느낀다.
- 단순 시청이 아니라 실제 조립/실습 경험이 남는다.
- 라이센스, 배지, 레벨업 요소를 통해 재방문 동기가 생긴다.

---

## 4. 타겟 사용자
### 1차 타겟
- AI와 자동화에 관심 있는 일반 사용자
- OpenClaw, 모바일 에이전트, AX에 관심 있는 실무자
- 비개발자이지만 생성형 AI를 활용해보고 싶은 사용자

### 2차 타겟
- 교육 기관
- 기업 내부 AX 교육 담당자
- 청소년/메이커형 체험 교육 시장

---

## 5. 핵심 사용자 시나리오
### 시나리오 1. 메타버스 학습 공간 탐험
사용자는 플랫폼에 접속한 뒤, 놀이공원처럼 구성된 메타버스 학습 공간을 이동한다. 각 구역은 서로 다른 학습 주제를 가진다.

예시 구역:
- 왜 모바일 에이전트인가
- 전체 아키텍처
- 활용 사례
- 에이전트 스쿨

### 시나리오 2. 에이전트 스쿨 체험
사용자는 에이전트 스쿨에 प्रवेश하여 툴을 만들고, 툴을 조합해 스킬을 만들고, 스킬을 조합해 에이전트를 완성한다.

### 시나리오 3. 보상 및 성취
사용자가 미션을 완료하거나 에이전트를 완성하면 라이센스, 배지, 레벨업 등의 보상을 받는다.

---

## 6. 핵심 기능
### 6.1 메타버스형 학습 공간
- 가상 맵 기반 탐색 UI
- 존별 학습 콘텐츠 배치
- 이동형 사용자 경험
- 공간 기반 내비게이션

### 6.2 에이전트 스쿨
- 단계형 실습 흐름
- 툴 → 스킬 → 에이전트 구조 학습
- 조립형 제작 경험
- 실습 결과 저장

### 6.3 레고블럭 조립식 에이전트 빌더
- 자연어 기반 툴 생성
- 툴 노드 생성
- 노드 조합으로 스킬 생성
- 스킬 조합으로 에이전트 생성
- GUI 드래그앤드롭 편집

### 6.4 게이미피케이션
- 퀘스트
- 배지
- 라이센스
- 레벨 시스템
- 진행률 표시
- 챌린지 모드

### 6.5 학습 지원 기능
- 튜토리얼
- 추천 학습 동선
- 예제 에이전트 제공
- 진행 상태 저장
- 복습/재도전 기능

---

## 7. 정보 구조
### 상위 구조
- 메인 로비
- 주제별 학습 존
- 에이전트 스쿨
- 마이 라이센스 / 배지
- 내가 만든 에이전트
- 챌린지 존

### 학습 흐름
1. 입장
2. 첫 미션 안내
3. 개념 존 탐험
4. 실습 존 진입
5. 툴 생성
6. 스킬 생성
7. 에이전트 완성
8. 라이센스 발급
9. 다음 챌린지 제안

---

## 8. UX 원칙
- 학습보다 탐험처럼 느껴져야 한다.
- 설명보다 체험이 먼저 와야 한다.
- 작은 성공 경험을 빠르게 제공해야 한다.
- 사용자는 “내가 만들었다”는 감각을 가져야 한다.
- 복잡한 기술 용어는 최대한 시각적 구조로 치환한다.

---

## 9. 기술 스택 방향
### 메타버스 영역
- 3D 또는 2.5D 공간 UI
- 웹 기반 접속 가능 구조
- 공간 이동 및 인터랙션 지원

### 에이전트 빌더 영역
- GUI 기반 노드 편집기
- 자연어 입력으로 툴 생성
- 스킬/에이전트 구조 저장
- 실행 결과 피드백 UI

### 교육/운영 영역
- 사용자 계정 및 진행 상태 관리
- 배지/라이센스 발급 시스템
- 콘텐츠 관리 시스템
- 예제/미션 데이터 관리

---

## 10. MVP 범위
### MVP에 포함할 것
- 간단한 메인 로비
- 3~4개의 학습 존
- 기본 에이전트 스쿨
- 툴/스킬/에이전트 생성 흐름
- 1개의 라이센스 발급 기능
- 진행 상태 저장

### MVP에서 제외할 것
- 완전한 자유형 메타버스 월드
- 고도화된 멀티플레이 협업 기능
- 복잡한 소셜 기능
- 과도한 3D 그래픽 완성도
