# 레고블락 방식과 n8n식 에이전트 빌더 비교, 슬라이드 요약

## 슬라이드 1. 에이전트는 어떻게 만들어지는가
### 핵심 메시지
에이전트는 처음부터 거대한 하나의 시스템으로 만들어지는 것이 아니라, 작은 기능 단위들을 조합해 점진적으로 구성된다.

### 요약 포인트
- 툴은 가장 작은 기능 단위다.
- 스킬은 여러 툴을 묶은 재사용 가능한 기능 묶음이다.
- 에이전트는 여러 스킬을 연결한 전체 구조다.
- 즉, 에이전트 구축은 조합의 과정이다.

### 한 줄 문장
**에이전트는 한 번에 만드는 것이 아니라, 작은 기능을 조합해 키워가는 구조다.**

---

## 슬라이드 2. 레고블락 방식 vs n8n식 빌더
### 레고블락 방식
- 레고블락 = 툴
- 레고블락 소품 = 스킬
- 작품 = 에이전트
- 장점: 직관적이고 입문자 설명에 강함

### n8n식 에이전트 빌더
- 자연어로 툴 생성
- 툴은 노드가 됨
- 노드 조합으로 스킬 생성
- 스킬 조합으로 에이전트 생성
- 장점: GUI 기반 제품 경험 설명에 강함

### 한 줄 문장
**레고블락 방식은 구조를 이해시키고, n8n식 빌더는 구축 경험을 상상하게 만든다.**

---

## 슬라이드 3. 왜 이 비유가 중요한가
### 핵심 메시지
에이전트를 어렵고 추상적인 기술이 아니라, 누구나 조립 가능한 구조로 이해시키는 것이 중요하다.

### 요약 포인트
- 작은 툴부터 시작할 수 있다.
- 툴을 묶어 스킬로 재사용할 수 있다.
- 스킬을 연결해 목적 있는 에이전트를 만들 수 있다.
- 비개발자도 접근 가능한 빌더 경험으로 확장할 수 있다.

### 결론 문장
**에이전트 구축의 본질은 거대한 AI를 만드는 것이 아니라, 작은 기능을 조합해 목적 있는 구조를 만드는 것이다.**
