# 레고블락만들기식 에이전트 구축

에이전트를 어렵게 생각할 필요는 없다. 복잡한 시스템처럼 보이지만, 실제로는 작은 기능 단위들을 조합해 하나의 목적 있는 구조를 만드는 과정에 가깝다. 쉽게 말하면, 에이전트는 레고블락을 조립해 작품을 만드는 방식으로 이해할 수 있다.

## 1. 레고블락 = 각각의 툴
툴은 가장 작은 기능 단위다. 하나의 툴은 하나의 동작을 수행한다.

예를 들면 다음과 같다.
- 지메일 읽기
- 지메일 쓰기
- 주간보고 초안 작성
- 노션에 저장하기
- 웹 검색하기
- 캘린더 일정 조회하기

이 각각은 혼자서도 쓸 수 있지만, 그 자체만으로는 큰 목적을 이루기 어렵다. 레고블락 한 개만으로는 집이 아니라 작은 부품 하나에 가까운 것과 같다.

## 2. 레고블락 소품 = 스킬 = 툴 조합
스킬은 여러 툴을 특정 목적에 맞게 묶어놓은 조합이다. 즉, 레고블락 몇 개를 조립해 자동차 바퀴나 창문 같은 소품을 만드는 것과 비슷하다.

예를 들면 다음과 같은 스킬을 생각할 수 있다.
- 메일 요약 스킬
  - 지메일 읽기
  - 중요 메일 분류
  - 요약 작성

- 주간보고 작성 스킬
  - 최근 업무 내역 읽기
  - 주요 성과 정리
  - 보고서 초안 작성

- 일정 브리핑 스킬
  - 캘린더 조회
  - 일정 우선순위 정리
  - 오늘의 일정 요약 작성

즉, 스킬은 툴보다 한 단계 높은 재사용 가능한 기능 묶음이다.

## 3. 작품 = 에이전트 = 스킬 조합
에이전트는 여러 스킬이 연결되어 하나의 결과물을 만들어내는 전체 구조다. 레고블락과 소품을 조합해 최종적으로 자동차, 집, 우주선 같은 작품을 만드는 것과 같다.

예를 들어,
- 업무 비서 에이전트
  - 메일 요약 스킬
  - 일정 브리핑 스킬
  - 주간보고 작성 스킬

- 개인 생활관리 에이전트
  - 일정 확인 스킬
  - 메모 정리 스킬
  - 알림 발송 스킬

이처럼 에이전트는 단일 툴이 아니라, 여러 스킬의 조합으로 목적 있는 행동을 수행하는 구조다.

## 4. 왜 이 비유가 중요한가
많은 사람들이 에이전트를 들으면 처음부터 거대한 인공지능 시스템을 떠올린다. 하지만 실제 구축은 그렇게 시작하지 않는다.

보통의 순서는 다음과 같다.
1. 작은 툴 하나를 만든다.
2. 툴 몇 개를 묶어 스킬을 만든다.
3. 스킬 몇 개를 묶어 에이전트를 만든다.

이 접근이 중요한 이유는 다음과 같다.
- 작게 시작할 수 있다.
- 실패해도 수정이 쉽다.
- 재사용이 가능하다.
- 점진적으로 확장할 수 있다.

즉, 에이전트 구축은 처음부터 완성형 작품을 만드는 일이 아니라, 블락을 하나씩 쌓아 올리는 과정이다.

## 5. 쉽게 에이전트를 만들 수 있다는 뜻
결국 에이전트는 신비한 존재가 아니라, 기능 블락을 조립한 구조다.

- 툴은 블락이다.
- 스킬은 블락을 조합한 부품이다.
- 에이전트는 부품을 조합한 작품이다.

이렇게 생각하면 에이전트 구축은 훨씬 쉬워진다. 처음부터 거대한 시스템을 설계하려고 하기보다, 작은 툴 하나부터 만들고, 그것을 조합해 스킬을 만들고, 다시 그것들을 엮어 목적 있는 에이전트로 확장하면 된다.

그래서 에이전트 구축은 결국 “레고블락 만들기”에 가깝다. 작은 블락을 만들 수 있다면, 누구나 자신만의 에이전트를 조립해볼 수 있다.
