# OpenClaw 텔레그램 그룹챗을 토픽별 세션으로 나누어 사용하는 방법

> 목표: 텔레그램 그룹 안에 여러 개의 **토픽**을 만들고, OpenClaw 봇을 불러서 주제별로 대화를 분리해 사용하는 방법을 처음부터 끝까지 따라 할 수 있게 정리합니다.

예를 들어 이런 식으로 나누어 쓸 수 있습니다.

- `일상의 재설계` 토픽: 생활 루틴, 습관, 일정 관리 이야기
- `개발` 토픽: 코딩, Git, 서버, 배포 이야기
- `아이디어` 토픽: 사업 아이디어, 글감, 기획 이야기
- `테스트` 토픽: 봇 호출, 기능 확인, 실험용 대화

이렇게 나누면 OpenClaw가 토픽마다 다른 대화 흐름으로 이해하기 쉬워집니다.

---

## 1. 먼저 알아야 할 개념

### 1-1. 텔레그램 그룹이란?

텔레그램에서 여러 사람이 함께 대화하는 방입니다.

카카오톡 단체방처럼 생각하면 됩니다.

### 1-2. 토픽이란?

토픽은 그룹 안에 만드는 작은 대화방입니다.

하나의 큰 그룹 안에 여러 주제별 방을 만드는 기능입니다.

예를 들면 `openclaw`라는 그룹 안에 다음과 같은 토픽을 만들 수 있습니다.

```text
openclaw 그룹
├─ 일반
├─ 일상의 재설계
├─ 개발
├─ 블로그 글쓰기
└─ 테스트
```

각 토픽은 서로 다른 주제의 대화를 담습니다.

### 1-3. OpenClaw 세션이란?

OpenClaw가 대화를 기억하고 이어 가는 단위입니다.

쉽게 말하면 OpenClaw 입장에서의 “대화방 기억장치”입니다.

텔레그램 그룹이 토픽을 지원하면, OpenClaw는 보통 다음처럼 토픽 정보를 함께 받습니다.

```text
telegram:-1003844548929 topic:1
telegram:-1003844548929 topic:15
```

여기서 `topic:1`, `topic:15`처럼 토픽 번호가 다르면 OpenClaw가 다른 대화 흐름으로 구분할 수 있습니다.

즉, 토픽을 나누면 주제별로 대화 컨텍스트를 분리해서 쓰기 좋아집니다.

## 2. 준비물

따라 하기 전에 아래가 필요합니다.

### 준비물 1: 텔레그램 계정

스마트폰이나 PC에 텔레그램이 설치되어 있어야 합니다.

- 스마트폰: Telegram 앱
- PC: Telegram Desktop 또는 웹 텔레그램

### 준비물 2: OpenClaw 봇

텔레그램에서 OpenClaw와 연결된 봇이 필요합니다.

예시:

```text
@openclaw_260205_bot
```

각자의 환경에서는 봇 이름이 다를 수 있습니다.

### 준비물 3: 토픽을 지원하는 텔레그램 그룹

텔레그램의 토픽 기능은 일반 1:1 채팅이 아니라 **그룹**에서 사용합니다.

그리고 그룹에서 토픽 기능이 켜져 있어야 합니다.

### 준비물 4: 봇 권한

OpenClaw 봇이 그룹 안에서 제대로 동작하려면 권한이 필요합니다.

특히 봇이 직접 토픽을 만들게 하려면 다음 권한이 필요할 수 있습니다.

- 관리자 권한
- 토픽 관리 권한
- 메시지 읽기/쓰기 권한

권한이 없으면 봇이 메시지는 읽어도 토픽 생성은 못 할 수 있습니다.

## 3. 전체 흐름 한눈에 보기

처음부터 끝까지의 흐름은 이렇습니다.

```text
1. 텔레그램 그룹 만들기
2. 그룹에서 토픽 기능 켜기
3. OpenClaw 봇을 그룹에 초대하기
4. 봇에게 필요한 권한 주기
5. 토픽 만들기
6. 토픽 안에서 OpenClaw 봇 부르기
7. 토픽별로 주제를 정해 대화하기
```

이제 하나씩 해 보겠습니다.

---

## 4. 텔레그램 그룹 만들기

이미 그룹이 있다면 이 단계는 건너뛰어도 됩니다.

### 스마트폰에서 만들기

1. 텔레그램 앱을 엽니다.
2. 오른쪽 아래의 연필 모양 버튼을 누릅니다.
3. `새 그룹`을 누릅니다.
4. 그룹에 넣을 사람을 선택합니다.
5. 그룹 이름을 입력합니다.

예시:

```text
openclaw
```

6. 만들기 버튼을 누릅니다.

### PC에서 만들기

1. Telegram Desktop을 엽니다.
2. 왼쪽 위 메뉴 버튼을 누릅니다.
3. `New Group` 또는 `새 그룹`을 누릅니다.
4. 사람을 선택합니다.
5. 그룹 이름을 입력합니다.
6. 생성합니다.

---

## 5. 그룹에서 토픽 기능 켜기

토픽 기능은 그룹 설정에서 켭니다.

### 스마트폰 기준

1. 그룹방에 들어갑니다.
2. 화면 위쪽의 그룹 이름을 누릅니다.
3. 연필 모양의 편집 버튼을 누릅니다.
4. `토픽` 또는 `Topics` 항목을 찾습니다.
5. 토픽 기능을 켭니다.
6. 저장합니다.

### PC 기준

1. 그룹방에 들어갑니다.
2. 오른쪽 위 또는 상단의 그룹 이름을 누릅니다.
3. `Edit` 또는 `편집`을 누릅니다.
4. `Topics` 또는 `토픽`을 켭니다.
5. 저장합니다.

토픽 기능을 켜면 그룹 안에 주제별 공간을 만들 수 있습니다.

---

## 6. OpenClaw 봇을 그룹에 초대하기

### 방법

1. 그룹방에 들어갑니다.
2. 그룹 이름을 누릅니다.
3. `멤버 추가` 또는 `Add Members`를 누릅니다.
4. OpenClaw 봇 이름을 검색합니다.

예시:

```text
@openclaw_260205_bot
```

5. 봇을 선택하고 추가합니다.

봇이 그룹에 들어오면 이제 그룹 안에서 OpenClaw를 부를 수 있습니다.

---

## 7. OpenClaw 봇에게 권한 주기

OpenClaw 봇이 단순히 답변만 하면 되는 경우에는 기본 권한으로도 충분할 수 있습니다.

하지만 다음 일을 시키려면 관리자 권한이 필요할 수 있습니다.

- 새 토픽 만들기
- 토픽 이름 바꾸기
- 토픽 관리하기
- 메시지 삭제하기
- 고정 메시지 관리하기

### 권한 주는 방법

1. 그룹 이름을 누릅니다.
2. `관리자` 또는 `Administrators`로 들어갑니다.
3. `관리자 추가`를 누릅니다.
4. OpenClaw 봇을 선택합니다.
5. 필요한 권한을 켭니다.

권장 권한:

```text
메시지 보내기: 켜기
토픽 관리: 켜기
메시지 고정: 필요하면 켜기
메시지 삭제: 필요하면 켜기
```

처음에는 `토픽 관리` 권한이 중요합니다.

---

## 8. 사람이 직접 토픽 만들기

이제 토픽을 만들어 보겠습니다.

### 예시: `일상의 재설계` 토픽 만들기

1. 텔레그램 그룹에 들어갑니다.
2. 그룹 상단 이름을 누릅니다.
3. `토픽` 또는 `Topics` 화면으로 들어갑니다.
4. `새 토픽 만들기`를 누릅니다.
   - 어떤 화면에서는 말풍선에 `+`가 붙은 아이콘으로 보일 수 있습니다.
5. 토픽 이름을 입력합니다.

```text
일상의 재설계
```

6. 만들기 버튼을 누릅니다.

그러면 그룹 안에 `일상의 재설계`라는 토픽이 생깁니다.

---

## 9. OpenClaw에게 토픽을 만들라고 시키기

봇에게 토픽 관리 권한이 있다면, 직접 만들라고 시킬 수도 있습니다.

예시:

```text
지니야, 일상의 재설계 토픽 만들어줘
```

그러면 OpenClaw가 텔레그램에 토픽 생성 요청을 보냅니다.

실제로는 이런 일이 일어납니다.

```text
1. 사용자가 “일상의 재설계 토픽 만들어줘”라고 말함
2. OpenClaw가 텔레그램 그룹에서 토픽 생성 기능을 호출함
3. 텔레그램이 새 토픽을 만듦
4. 새 토픽 ID가 생성됨
5. OpenClaw가 완료 메시지를 보냄
```

완료되면 이런 식으로 확인할 수 있습니다.

```text
분부대로 ‘일상의 재설계’ 토픽을 만들었사옵니다.
```

봇이 토픽을 만들지 못한다면 대부분 권한 문제입니다.

그럴 때는 그룹 관리자 설정에서 봇에게 `토픽 관리` 권한이 있는지 확인하세요.

---

## 10. 토픽 안에서 OpenClaw 부르기

토픽을 만들었으면 그 안으로 들어갑니다.

예를 들어 `일상의 재설계` 토픽으로 들어갑니다.

그 안에서 봇을 부릅니다.

예시:

```text
@openclaw_260205_bot 지니야, 이제 여기서 일상 루틴 이야기를 하자
```

또는 봇이 이미 대화를 잘 듣고 있다면 이렇게 말해도 됩니다.

```text
지니야, 이제 여기서 일상 루틴 이야기를 하자
```

그러면 OpenClaw는 이 메시지가 어느 토픽에서 왔는지 함께 받습니다.

예시:

```text
group: openclaw
 topic: 일상의 재설계
 topic_id: 15
```

이렇게 토픽 ID가 붙기 때문에, OpenClaw는 다른 토픽과 구분해서 대화를 이어 갈 수 있습니다.

---

## 11. 토픽별 세션이 왜 좋은가?

토픽을 나누면 대화가 섞이지 않습니다.

### 토픽을 안 나누면

한 그룹에서 모든 이야기를 하게 됩니다.

```text
오늘 운동 계획 이야기
서버 배포 이야기
블로그 글쓰기 이야기
가족 일정 이야기
Git 오류 이야기
```

이렇게 섞이면 OpenClaw도 앞뒤 맥락을 헷갈릴 수 있습니다.

### 토픽을 나누면

주제별로 깔끔하게 정리됩니다.

```text
일상의 재설계 토픽
→ 수면, 운동, 식사, 루틴 이야기

개발 토픽
→ 코드, Git, 배포, 서버 이야기

블로그 토픽
→ 글감, 초안, 발행 계획 이야기
```

OpenClaw에게도 훨씬 좋은 구조입니다.

## 12. 실제 사용 예시

### 예시 1: 일상 관리 토픽

토픽 이름:

```text
일상의 재설계
```

처음 메시지:

```text
지니야, 이 토픽에서는 내 하루 루틴을 같이 설계하자.
수면, 운동, 식사, 집중 시간만 다루자.
```

이렇게 시작하면 이 토픽의 역할이 분명해집니다.

그다음부터는 이렇게 말하면 됩니다.

```text
오늘은 7시에 일어났고 운동은 못 했어. 내일 루틴 다시 짜줘.
```

OpenClaw는 이 토픽이 일상 루틴용이라는 흐름 안에서 답변할 수 있습니다.

### 예시 2: 개발 작업 토픽

토픽 이름:

```text
개발
```

처음 메시지:

```text
지니야, 이 토픽에서는 GhostHand 프로젝트 개발 이야기만 하자.
작업, 버그, Git, 배포를 여기서 관리하자.
```

그다음 요청:

```text
E:\ax\PRJs\GhostHand\docs 폴더에 blog 폴더를 만들고 푸시해줘.
```

OpenClaw가 작업을 수행하고 결과를 이 토픽에 보고할 수 있습니다.

### 예시 3: 블로그 글쓰기 토픽

토픽 이름:

```text
블로그
```

처음 메시지:

```text
지니야, 이 토픽에서는 블로그 글감과 초안을 관리하자.
초등학생도 이해할 수 있게 쉽게 쓰는 걸 원칙으로 하자.
```

그다음 요청:

```text
OpenClaw 텔레그램 토픽 세션 사용법을 블로그 글로 정리해줘.
```

이렇게 하면 글쓰기 관련 맥락이 따로 쌓입니다.

## 13. 추천 토픽 구성

처음에는 너무 많이 만들지 않는 것이 좋습니다.

아래 정도면 충분합니다.

```text
일반
일상의 재설계
개발
블로그
아이디어
테스트
```

### 각 토픽의 역할

#### 일반

가볍게 묻고 답하는 공간입니다.

#### 일상의 재설계

생활 루틴, 습관, 일정, 건강 관리를 다룹니다.

#### 개발

프로젝트, 코드, Git, 배포, 서버 문제를 다룹니다.

#### 블로그

글감, 초안, 수정, 발행 계획을 다룹니다.

#### 아이디어

사업, 콘텐츠, 제품, 자동화 아이디어를 모읍니다.

#### 테스트

봇 기능 확인, 실험, 권한 테스트를 합니다.

---

## 14. 처음 시작할 때 추천하는 첫 메시지

토픽을 만들었다면 첫 메시지로 역할을 정해 주는 것이 좋습니다.

아래 문장을 복사해서 써도 됩니다.

```text
지니야, 이 토픽은 [주제 이름] 전용으로 쓰자.
앞으로 이 토픽에서는 [다룰 내용]만 이야기하고,
다른 주제는 다른 토픽에서 이야기하자.
```

예시:

```text
지니야, 이 토픽은 일상의 재설계 전용으로 쓰자.
앞으로 이 토픽에서는 수면, 운동, 식사, 집중 루틴만 이야기하고,
개발이나 블로그 이야기는 다른 토픽에서 이야기하자.
```

이렇게 해 두면 나중에 OpenClaw가 토픽의 목적을 이해하기 쉬워집니다.

## 15. 주의할 점

### 15-1. 토픽별 대화는 분리되지만, 모든 것이 완전히 따로인 것은 아닙니다

토픽별로 대화 흐름은 분리해서 관리하기 좋습니다.

하지만 다음은 OpenClaw 전체에서 공유될 수 있습니다.

```text
장기 기억
작업공간 파일
프로젝트 파일
사용자 선호 설정
봇 설정
```

예를 들어 사용자를 `전하`라고 부르는 설정은 모든 토픽에서 이어질 수 있습니다.

반대로 `일상의 재설계` 토픽에서만 하던 세부 대화는 다른 토픽에서 바로 보이지 않을 수 있습니다.

### 15-2. 중요한 정보는 명확히 말해 주세요

토픽이 나뉘어 있으면 OpenClaw가 다른 토픽의 대화까지 항상 알고 있다고 생각하면 안 됩니다.

필요하면 이렇게 말해 주세요.

```text
지니야, 참고로 개발 토픽에서 GhostHand 프로젝트를 작업 중이야.
여기서는 그 내용을 블로그 글로 정리하자.
```

### 15-3. 봇이 답하지 않으면 멘션해 보세요

그룹에서는 봇이 모든 메시지에 답하지 않도록 설정되어 있을 수 있습니다.

그럴 때는 봇 이름을 붙여 부르면 됩니다.

```text
@openclaw_260205_bot 지니야, 이 토픽에서 이야기하자
```

## 16. 문제가 생겼을 때 확인할 것

### 문제 1: 새 토픽 버튼이 안 보여요

확인할 것:

```text
그룹이 맞는가?
토픽 기능이 켜져 있는가?
내 계정이 관리자 권한을 가지고 있는가?
텔레그램 앱이 최신 버전인가?
```

### 문제 2: 봇이 토픽을 만들지 못해요

확인할 것:

```text
봇이 그룹에 들어와 있는가?
봇이 관리자 권한을 가지고 있는가?
봇에게 토픽 관리 권한이 있는가?
```

### 문제 3: 봇이 답장을 안 해요

확인할 것:

```text
봇을 @이름으로 멘션했는가?
OpenClaw Gateway가 실행 중인가?
텔레그램 채널 설정이 되어 있는가?
봇이 메시지를 읽을 수 있는 권한이 있는가?
```

### 문제 4: 다른 토픽의 내용을 모르는 것 같아요

정상일 수 있습니다.

토픽별 세션은 주제별로 분리해서 쓰는 것이 목적입니다.

다른 토픽의 내용을 이어서 써야 한다면 간단히 요약해서 알려 주세요.

예시:

```text
지니야, 개발 토픽에서 OpenClaw 텔레그램 그룹 설정을 마쳤어.
이제 여기서는 그 내용을 블로그로 정리하자.
```

## 17. 권장 사용 습관

### 17-1. 토픽 이름은 짧고 분명하게

좋은 예:

```text
일상의 재설계
개발
블로그
아이디어
```

나쁜 예:

```text
이것저것
나중에 볼 것
아무거나
```

토픽 이름이 분명해야 사람도, OpenClaw도 이해하기 쉽습니다.

### 17-2. 첫 메시지로 토픽의 목적을 알려 주기

토픽을 만들자마자 이렇게 말해 주세요.

```text
지니야, 이 토픽에서는 블로그 글쓰기만 다루자.
```

이 한 줄이 나중에 큰 도움이 됩니다.

### 17-3. 주제가 바뀌면 다른 토픽으로 이동하기

`일상의 재설계` 토픽에서 갑자기 서버 배포 이야기를 오래 하면 토픽을 나눈 의미가 줄어듭니다.

그럴 때는 `개발` 토픽으로 가서 이야기하는 것이 좋습니다.

### 17-4. 큰 작업은 전용 토픽을 만들기

예를 들어 책을 쓰거나, 앱을 만들거나, 긴 프로젝트를 진행한다면 전용 토픽을 만드는 것이 좋습니다.

예시:

```text
GhostHand 개발
OpenClaw 실험실
블로그 운영
개인 루틴 개선
```


## 19. 가장 짧은 실전 예시

아래처럼만 해도 시작할 수 있습니다.

```text
1. 텔레그램 그룹 만들기
2. 그룹 설정에서 토픽 켜기

4. 봇에게 토픽 관리 권한 주기
5. “일상의 재설계” 토픽 만들기
6. 그 토픽에 들어가기

@openclaw_260205_bot 지니야, 이 토픽은 일상의 재설계 전용으로 쓰자.
앞으로 여기서는 수면, 운동, 식사, 집중 루틴을 같이 관리하자.
```

이제부터 그 토픽은 일상 관리용 OpenClaw 대화 공간처럼 사용할 수 있습니다.

## 20. 마무리

텔레그램 그룹의 토픽 기능을 OpenClaw와 함께 쓰면, 하나의 그룹 안에서도 여러 개의 독립된 작업 공간처럼 대화를 나눌 수 있습니다.

핵심은 간단합니다.

```text
그룹 = 큰 집
토픽 = 방
OpenClaw 세션 = 각 방에서 이어지는 대화 기억
```

주제가 다르면 방을 나누듯이 토픽을 나누면 됩니다.

처음에는 `일상의 재설계`, `개발`, `블로그`, `테스트` 정도만 만들어도 충분합니다.

그리고 각 토픽의 첫 메시지에서 “이 방은 무엇을 위한 곳인지”만 분명히 알려 주세요.

그러면 OpenClaw를 훨씬 더 깔끔하고 편하게 사용할 수 있습니다.
