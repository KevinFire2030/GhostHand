# Hermes Agent 윈도우에 설치하는 방법

> 기준: Hermes Agent 공식 저장소 `NousResearch/hermes-agent`  
> 핵심: **Windows 네이티브 설치는 공식 미지원**이며, **WSL2 Ubuntu 환경에서 설치**하는 방식이 권장된다.

## 0. 결론

Hermes Agent는 Windows에 바로 설치하는 방식이 아니라, Windows 안의 Linux 환경인 **WSL2**에 설치한다.

공식 안내 기준:

- Linux 지원
- macOS 지원
- WSL2 지원
- Android Termux 지원
- Windows 네이티브 미지원

따라서 Windows 사용자는 다음 흐름으로 진행한다.

```text
Windows
└─ WSL2
   └─ Ubuntu
      └─ Hermes Agent 설치
```

## 1. 사전 준비

PowerShell 또는 Windows Terminal을 연다.

먼저 WSL 상태를 확인한다.

```powershell
wsl --status
wsl --list --verbose
```

이미 Ubuntu가 보이면 다음 단계로 넘어간다.

예시:

```text
NAME              STATE           VERSION
Ubuntu            Running         2
```

`docker-desktop`만 있고 Ubuntu가 없다면 Ubuntu 배포판을 설치해야 한다.

## 2. Ubuntu WSL 설치

PowerShell 또는 Windows Terminal에서 실행한다.

```powershell
wsl --install -d Ubuntu
```

설치 후 Ubuntu를 처음 실행한다.

```powershell
wsl -d Ubuntu
```

처음 실행 시 사용자명과 비밀번호를 만들라는 안내가 나올 수 있다.

예시:

```text
Enter new UNIX username:
New password:
Retype new password:
```

이 초기 설정을 완료해야 이후 명령을 정상 실행할 수 있다.

## 3. Ubuntu 패키지 업데이트

Ubuntu 셸 안에서 실행한다.

```bash
sudo apt update
sudo apt upgrade -y
```

기본 도구도 설치한다.

```bash
sudo apt install -y curl git ca-certificates build-essential
```

## 4. Hermes Agent 설치

Ubuntu 셸 안에서 공식 설치 명령을 실행한다.

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

설치가 끝나면 셸 설정을 다시 불러온다.

```bash
source ~/.bashrc
```

또는 zsh를 쓰는 경우:

```bash
source ~/.zshrc
```

## 5. 설치 확인

```bash
hermes --help
hermes doctor
```

문제가 없다면 Hermes CLI를 실행한다.

```bash
hermes
```

## 6. 기본 설정

전체 설정 마법사를 실행한다.

```bash
hermes setup
```

모델을 별도로 설정하려면 다음 명령을 사용한다.

```bash
hermes model
```

도구 설정은 다음 명령으로 확인한다.

```bash
hermes tools
```

## 7. OpenClaw에서 마이그레이션

기존 OpenClaw 사용자는 Hermes에서 OpenClaw 설정, 메모리, 스킬, API 키 등을 가져올 수 있다.

먼저 미리보기:

```bash
hermes claw migrate --dry-run
```

실제 마이그레이션:

```bash
hermes claw migrate
```

사용자 데이터만 옮기고 비밀키는 제외하려면:

```bash
hermes claw migrate --preset user-data
```

충돌 시 덮어쓰려면:

```bash
hermes claw migrate --overwrite
```

마이그레이션 대상 예시:

- `SOUL.md`
- `MEMORY.md`, `USER.md`
- 사용자 생성 Skill
- command allowlist
- messaging 설정
- API keys
- TTS assets
- workspace instructions

## 8. Telegram 등 메시징 Gateway 설정

Hermes는 CLI뿐 아니라 Telegram, Discord, Slack, WhatsApp, Signal 등 메시징 채널에서도 사용할 수 있다.

Gateway 설정:

```bash
hermes gateway setup
```

Gateway 시작:

```bash
hermes gateway start
```

전체 설정 마법사에서 함께 설정하려면:

```bash
hermes setup
```

## 9. 자주 막히는 지점

### 9-1. `docker-desktop`만 보이고 Ubuntu가 없음

`wsl --list --verbose` 결과가 다음처럼 보이면 Ubuntu가 아직 없는 상태다.

```text
NAME              STATE           VERSION
* docker-desktop  Running         2
```

이 경우 Ubuntu를 설치한다.

```powershell
wsl --install -d Ubuntu
```

### 9-2. Ubuntu가 Running인데 명령이 멈춤

Ubuntu 첫 실행 초기화가 끝나지 않은 경우가 많다.

Windows Terminal 또는 PowerShell에서 직접 실행한다.

```powershell
wsl -d Ubuntu
```

사용자명과 비밀번호를 만들고 초기 설정을 완료한다.

### 9-3. `hermes` 명령을 찾을 수 없음

설치 후 PATH가 아직 적용되지 않은 상태일 수 있다.

```bash
source ~/.bashrc
```

그래도 안 되면 새 터미널을 열고 다시 확인한다.

```bash
which hermes
hermes --help
```

### 9-4. Windows 파일에 접근하고 싶음

WSL 안에서 Windows 드라이브는 `/mnt` 아래에 있다.

예시:

```bash
cd /mnt/e/ax/PRJs/GhostHand
```

Windows 경로:

```text
E:\ax\PRJs\GhostHand
```

WSL 경로:

```text
/mnt/e/ax/PRJs/GhostHand
```

## 10. 추천 설치 순서 요약

```powershell
# Windows PowerShell
wsl --status
wsl --list --verbose
wsl --install -d Ubuntu
wsl -d Ubuntu
```

```bash
# Ubuntu WSL
sudo apt update
sudo apt upgrade -y
sudo apt install -y curl git ca-certificates build-essential
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
source ~/.bashrc
hermes doctor
hermes setup
```

## 11. 운영 팁

- Windows에서는 Hermes를 직접 설치하려 하지 말고 WSL2 Ubuntu를 기준으로 생각한다.
- 장기 운영은 WSL 안에서 Hermes Gateway를 실행하는 방식으로 구성한다.
- 기존 OpenClaw 사용자는 설치 직후 `hermes claw migrate --dry-run`으로 이전 범위를 먼저 확인한다.
- Telegram 연동까지 할 경우 BotFather 토큰과 허용 사용자 설정을 미리 준비하면 빠르다.
- 프로젝트 파일은 `/mnt/e/...`처럼 WSL에서 Windows 드라이브를 마운트해 접근할 수 있다.
