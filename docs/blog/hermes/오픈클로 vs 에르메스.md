# 오픈클로 vs 에르메스: 개인 AI 에이전트 역할 분담안

![오픈클로 vs 에르메스 인포그래픽](./오픈클로%20vs%20에르메스%20인포그래픽.svg)

## 1. 한 줄 요약

오픈클로(OpenClaw)는 **가볍게 붙여 쓰는 실전형 AI 에이전트**에 가깝고, 에르메스(Hermes)는 **메모리·스킬·지식 그래프·스케줄링을 축적해 개인 AI팀처럼 운영하는 학습형 에이전트**에 가깝다.

둘은 경쟁 관계로만 보기보다, 사용 목적에 따라 역할을 나누면 가장 효율적이다.

- **오픈클로**: 빠른 실행, 익숙한 워크플로, 단순 자동화
- **에르메스**: 장기 기억, 스킬 재사용, 멀티 플랫폼, 반복 업무 자동화, 개인화

---

## 2. 오픈클로와 에르메스 비교 분석

### 2.1 기본 방향성

#### 오픈클로

오픈클로는 사용자가 요청한 작업을 빠르게 처리하는 데 초점이 있다. 메신저나 CLI에서 AI를 호출하고, 필요한 명령을 수행하고, 결과를 돌려받는 형태의 실용적인 에이전트 경험을 제공한다.

특히 기존 사용자가 이미 프롬프트, SOUL, 작업 습관, 도구 호출 방식에 익숙하다면 진입 장벽이 낮다.

#### 에르메스

에르메스는 단순 요청 처리보다 **지속적으로 축적되는 개인 AI 운영체제**에 가깝다.

메모리, 스킬, 크론 작업, 메시징 게이트웨이, 파일·터미널·브라우저·GitHub·생산성 도구 등을 묶어 장기적으로 사용자에게 맞춰지는 구조를 지향한다.

즉, 한 번 대답하고 끝나는 챗봇이 아니라 시간이 갈수록 다음 항목이 쌓인다.

- 사용자의 선호와 작업 방식
- 반복 업무 절차
- 프로젝트별 규칙
- 자주 쓰는 도구와 명령
- 주기적으로 실행할 자동화 작업
- 외부 채널로 전달할 알림과 리포트

---

## 3. 핵심 비교

### 3.1 사용성

- **오픈클로**
  - 빠르게 시작하기 좋다.
  - 기존 사용자가 익숙한 방식으로 바로 업무를 시킬 수 있다.
  - 단순 질의응답, 파일 작업, 간단한 자동화에 적합하다.

- **에르메스**
  - 처음에는 설정할 수 있는 옵션이 많아 다소 복잡하게 느껴질 수 있다.
  - 대신 한 번 세팅하면 텔레그램, 크론, 스킬, 메모리, 외부 도구 연동을 통해 장기 운영성이 좋다.
  - 개인 비서라기보다 개인 AI팀을 구성하는 느낌에 가깝다.

### 3.2 기억과 개인화

- **오픈클로**
  - 세션 중심의 작업에는 충분하다.
  - 장기 기억이나 절차 축적은 상대적으로 제한적이다.

- **에르메스**
  - 사용자 프로필, 환경 정보, 프로젝트 규칙을 메모리로 저장할 수 있다.
  - 반복되는 절차는 스킬로 만들어 다음 작업에서 재사용할 수 있다.
  - 과거 대화 검색을 통해 이전 작업 맥락을 다시 불러올 수 있다.

### 3.3 자동화

- **오픈클로**
  - 사용자가 호출했을 때 빠르게 반응하는 온디맨드 작업에 강하다.
  - 간단한 자동화나 즉시 실행 작업에 적합하다.

- **에르메스**
  - 크론 작업으로 정기 리포트, 모니터링, 알림, 반복 업무를 예약할 수 있다.
  - 텔레그램 같은 메시징 플랫폼으로 결과를 자동 전달할 수 있다.
  - 장기적으로 비동기 업무를 맡기는 구조에 강하다.

### 3.4 도구 확장성

- **오픈클로**
  - 필요한 도구를 붙여 빠르게 사용하는 실용성이 좋다.
  - 구조가 단순할수록 운영 부담이 적다.

- **에르메스**
  - 파일, 터미널, 브라우저, GitHub, 이메일, 노션, 구글 워크스페이스, 스포티파이, 스마트홈 등 다양한 도구 확장을 전제로 한다.
  - 스킬 시스템을 통해 특정 도메인의 작업 절차를 문서화하고 반복 실행할 수 있다.

## 4. 장단점

## 4.1 오픈클로 장점

- **가볍다**: 빠르게 붙여 쓰기 좋다.
- **직관적이다**: 사용자가 바로 명령하고 바로 결과를 받는 흐름이 단순하다.
- **기존 사용자에게 익숙하다**: 이미 오픈클로 방식으로 작업하던 사람은 전환 비용이 낮다.
- **단기 작업에 좋다**: 일회성 요청, 간단한 파일 작업, 빠른 확인 업무에 적합하다.
- **운영 부담이 적다**: 많은 기능을 세팅하지 않아도 기본 사용성이 나온다.

## 4.2 오픈클로 단점

- **장기 기억과 절차 축적이 약하다**: 반복 업무를 체계적으로 자산화하기 어렵다.
- **스킬 기반 운영이 제한적이다**: 프로젝트별 표준 절차를 쌓아두고 재사용하는 구조는 상대적으로 약하다.
- **비동기 자동화가 약하다**: 정기 리포트, 모니터링, 예약 실행 같은 운영형 업무에는 한계가 있다.
- **멀티 채널 운영이 제한될 수 있다**: 여러 플랫폼으로 알림·결과를 보내는 구조는 에르메스가 더 유리하다.

## 4.3 에르메스 장점

- **메모리 기반 개인화**: 사용자의 선호, 프로젝트 규칙, 환경 정보를 장기적으로 기억한다.
- **스킬 기반 재사용**: 반복 작업 절차를 스킬로 저장해 다음 작업에서 바로 활용한다.
- **크론 기반 자동화**: 정기 실행, 모니터링, 알림, 리포트에 강하다.
- **메신저 연동**: 텔레그램 등 메시징 플랫폼을 통해 AI와 자연스럽게 협업할 수 있다.
- **멀티 도구 확장성**: GitHub, 파일, 터미널, 브라우저, 문서 도구 등 다양한 업무 도구를 연결할 수 있다.
- **AI팀 운영에 적합**: 단일 챗봇이 아니라 여러 도구와 작업자를 조합해 운영하는 방식에 가깝다.

## 4.4 에르메스 단점

- **초기 설정이 복잡할 수 있다**: 모델, 프로바이더, 게이트웨이, 도구, 스킬 등 이해해야 할 요소가 많다.
- **기능이 많아 학습 비용이 있다**: 단순 질의응답만 원하는 사용자에게는 과하게 느껴질 수 있다.
- **운영 철학이 필요하다**: 메모리와 스킬을 어떻게 관리할지 정하지 않으면 복잡도만 늘어날 수 있다.
- **환경 의존성이 있다**: 로컬 파일, 터미널, 메시징, API 키 등 주변 환경 설정이 결과 품질에 영향을 준다.

## 5. 에르메스가 오픈클로 대비 잘하는 것

## 5.1 장기 기억 관리

에르메스의 가장 큰 강점은 기억을 단순 로그가 아니라 업무 자산으로 다룬다는 점이다.

예를 들어 다음과 같은 정보를 저장해 다음 세션에서도 활용할 수 있다.

- 사용자가 선호하는 답변 스타일
- 특정 프로젝트의 빌드·테스트 명령
- 자주 사용하는 디렉터리 구조
- 반복되는 업무 절차
- 이전에 해결했던 문제의 접근 방식

오픈클로가 빠른 실행에 강하다면, 에르메스는 시간이 갈수록 사용자에게 맞춰지는 누적형 구조에 강하다.

## 5.2 스킬 기반 업무 표준화

에르메스는 반복 작업을 스킬로 만들 수 있다.

예를 들어 다음과 같은 업무를 스킬화할 수 있다.

- GitHub PR 작성 절차
- 코드 리뷰 체크리스트
- 유튜브 자막 요약 방식
- 문서 생성 템플릿
- 특정 프로젝트의 배포 절차
- 장애 분석 루틴

이렇게 하면 매번 같은 설명을 반복하지 않아도 된다. 에르메스가 스킬을 읽고 정해진 절차에 따라 작업한다.

## 5.3 예약 실행과 비동기 업무

에르메스는 크론 작업을 통해 사람 없이도 반복 실행할 수 있다.

활용 예시는 다음과 같다.

- 매일 아침 뉴스 요약 보내기
- 매주 프로젝트 진행 상황 리포트 생성
- 특정 API 상태 모니터링
- 서버 디스크 용량 경고
- 블로그 RSS 변경 감지
- 정해진 시간에 텔레그램으로 알림 발송

이 영역은 오픈클로보다 에르메스가 확실히 강하다.

## 5.4 메신저 중심 운영

에르메스는 텔레그램 같은 메시징 플랫폼과 잘 맞는다.

PC 앞에 있지 않아도 모바일에서 다음과 같은 일을 시킬 수 있다.

- 파일 수정 요청
- Git 커밋·푸시 요청
- 서버 상태 확인
- 정기 작업 결과 확인
- 이미지·문서 생성 요청
- 장기 작업 완료 알림 받기

즉, 에르메스는 AI를 터미널 안에 가두지 않고 일상 대화 채널로 끌어오는 데 강하다.

## 5.5 개인 AI팀 구성

에르메스는 단일 모델 하나를 쓰는 방식보다 여러 도구, 스킬, 하위 에이전트, 예약 작업을 엮는 방향에 가깝다.

따라서 다음처럼 역할을 나눌 수 있다.

- 리서치 담당
- 문서 작성 담당
- 코드 수정 담당
- 리뷰 담당
- 배포 담당
- 알림 담당

이런 구조는 장기 프로젝트나 반복 업무가 많은 사용자에게 유리하다.

---

## 6. 에르메스가 오픈클로 대비 잘 못하는 것

## 6.1 단순함

에르메스는 강력하지만 단순하지는 않다.

오픈클로처럼 “켜고 바로 시킨다”는 감각을 기대하면, 에르메스의 설정 항목과 도구 체계가 복잡하게 느껴질 수 있다.

## 6.2 초기 진입 속도

가볍게 한두 가지 작업만 하고 싶다면 오픈클로가 더 빠르게 느껴질 수 있다.

에르메스는 다음 요소를 이해할수록 가치가 커진다.

- 메모리
- 스킬
- 크론
- 게이트웨이
- 모델 프로바이더
- 외부 도구 연동
- 프로젝트별 컨텍스트

반대로 말하면, 이 구조를 이해하기 전까지는 장점이 바로 체감되지 않을 수 있다.

## 6.3 관리 비용

에르메스는 잘 쓰면 업무 자산이 쌓이지만, 방치하면 다음 문제가 생길 수 있다.

- 오래된 스킬이 현재 작업 방식과 맞지 않음
- 메모리에 불필요한 정보가 쌓임
- 크론 작업이 많아져 관리가 어려워짐
- 도구 설정이 꼬이면 원인 추적이 필요함

따라서 에르메스는 “설치형 도구”라기보다 “운영하는 AI 시스템”으로 보는 편이 맞다.

## 6.4 과한 기능감

단순히 질문하고 답변받는 용도라면 에르메스는 과할 수 있다.

오픈클로가 손에 잡히는 멀티툴이라면, 에르메스는 작업실 전체를 만드는 느낌이다. 작은 나사 하나 조이려는 사람에게는 작업실이 부담스러울 수 있다.

---

## 7. 역할 분담안

## 7.1 오픈클로가 맡으면 좋은 일

오픈클로는 다음 업무에 적합하다.

- 즉시 실행형 단기 작업
- 가벼운 질의응답
- 간단한 파일 확인·수정
- 빠른 실험
- 복잡한 설정 없이 바로 처리할 업무
- 기존 오픈클로 워크플로가 이미 안정화된 작업

### 추천 포지션

**“빠른 실행 담당 AI”**

오픈클로는 생각났을 때 바로 부르는 도구로 두면 좋다. 복잡한 장기 기억이나 자동화가 필요 없는 일은 오픈클로가 더 편할 수 있다.

## 7.2 에르메스가 맡으면 좋은 일

에르메스는 다음 업무에 적합하다.

- 반복되는 업무 자동화
- 프로젝트별 작업 규칙 관리
- 장기 기억이 필요한 개인화 작업
- 스킬로 표준화할 수 있는 절차
- 정기 리포트·모니터링·알림
- 텔레그램 기반 모바일 AI 비서
- GitHub, 문서, 파일, 브라우저, 터미널이 함께 필요한 복합 작업

### 추천 포지션

**“개인 AI팀 운영 플랫폼”**

에르메스는 단기 도구보다 장기 운영 플랫폼으로 쓰는 것이 좋다. 한 번 세팅해두면 반복 업무를 계속 줄여주는 방향으로 가치가 커진다.

---

## 8. 실전 역할 분담 예시

### 8.1 콘텐츠 제작

- **오픈클로**
  - 아이디어 초안 생성
  - 짧은 문장 수정
  - 제목 후보 뽑기

- **에르메스**
  - 유튜브 자막 추출 후 블로그 글로 변환
  - 기존 문서 스타일을 참고해 새 글 작성
  - 작성 파일을 Git에 커밋하고 푸시
  - 정기적으로 콘텐츠 아이디어 리포트 생성

### 8.2 개발 업무

- **오픈클로**
  - 간단한 코드 질문
  - 특정 함수 수정
  - 빠른 오류 확인

- **에르메스**
  - 레포 구조 분석
  - 브랜치 생성, 커밋, 푸시, PR 작성
  - 테스트 실행 및 실패 로그 분석
  - 코드 리뷰 체크리스트 기반 검토
  - 반복되는 개발 절차를 스킬로 저장

### 8.3 개인 업무 자동화

- **오픈클로**
  - 지금 필요한 정보 검색
  - 간단한 메모 정리
  - 즉석 계산이나 변환

- **에르메스**
  - 매일 아침 텔레그램으로 일정 요약
  - 주간 회고 자동 생성
  - 특정 폴더 변경 감지
  - 블로그·뉴스·RSS 모니터링
  - 장기 메모리 기반 개인화 응답

## 9. 결론

오픈클로와 에르메스는 둘 중 하나만 선택해야 하는 관계가 아니다.

- **오픈클로는 빠른 실행에 강하다.**
- **에르메스는 장기 운영과 자동화에 강하다.**

따라서 추천하는 분담은 다음과 같다.

- 당장 처리할 가벼운 일은 **오픈클로**
- 반복되고 축적되어야 하는 일은 **에르메스**
- 모바일 알림, 예약 실행, 스킬화, 장기 기억이 필요한 일은 **에르메스**
- 실험적이고 짧은 작업은 **오픈클로**

최종적으로 에르메스는 오픈클로의 대체재라기보다, 오픈클로에서 하던 작업을 더 오래 기억하고 더 체계적으로 자동화하는 상위 운영 레이어로 볼 수 있다.

즉, 오픈클로가 **손에 들고 쓰는 AI 도구**라면 에르메스는 **내 일을 기억하고 반복 업무를 맡아주는 개인 AI팀**이다.
