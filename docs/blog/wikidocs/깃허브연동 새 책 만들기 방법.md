# 위키독스에서 깃허브 연동 새 책 만들기 방법

> 참고 문서: https://wikidocs.net/321336

이 문서는 위키독스에서 **깃허브와 연동되는 새 책**을 만드는 기본 절차를 정리한 글입니다.

위키독스의 일반 책 만들기와 달리, 깃허브 연동 책은 원고 파일을 GitHub 저장소에서 관리하고 위키독스가 그 저장소 내용을 책으로 보여주는 방식입니다.

---

## 1. 준비물

깃허브 연동 책을 만들기 전에 아래 계정이 필요합니다.

- 위키독스 계정
- GitHub 계정
- GitHub 저장소를 만들 수 있는 권한

가능하면 위키독스에 사용할 이메일과 GitHub 계정 정보를 미리 확인해 둡니다.

---

## 2. 위키독스 로그인

먼저 위키독스에 로그인합니다.

```text
https://wikidocs.net
```

로그인 후 우측 상단의 계정 메뉴로 이동합니다.

---

## 3. 나의 책 메뉴로 이동

계정 메뉴에서 **계정설정**으로 들어간 뒤, **나의 책** 메뉴를 엽니다.

흐름은 다음과 같습니다.

```text
위키독스 로그인
→ 계정설정
→ 나의 책
```

이 화면에서 내가 만든 책 목록을 확인하거나 새 책을 만들 수 있습니다.

---

## 4. 깃허브 책 만들기 선택

나의 책 화면에서 **깃허브 책 만들기** 버튼을 클릭합니다.

여기서 중요한 점은 일반 책 만들기가 아니라, 반드시 **깃허브 책 만들기**를 선택해야 한다는 것입니다.

```text
나의 책
→ 깃허브 책 만들기
```

---

## 5. 책 정보 입력

깃허브 책 만들기 화면에서 책 정보를 입력합니다.

대표적으로 다음 정보를 입력합니다.

- 책 제목
- 책 설명
- 카테고리
- 공개 여부
- GitHub 연동 정보

책 제목은 위키독스에 표시되는 이름입니다.

예를 들어 다음처럼 입력할 수 있습니다.

```text
책 제목: 나의 AI 자동화 노트
책 설명: AI 에이전트와 자동화 실습을 정리한 책입니다.
```

---

## 6. GitHub 저장소 연동

깃허브 연동 책은 GitHub 저장소와 연결되어야 합니다.

위키독스 화면에서 안내하는 절차에 따라 GitHub 계정 인증 또는 저장소 생성/연동을 진행합니다.

연동 후에는 보통 다음과 같은 흐름으로 작업합니다.

```text
1. GitHub 저장소에 원고 파일 작성
2. 변경 내용을 commit
3. GitHub에 push
4. 위키독스에서 책 반영 확인
```

---

## 7. 원고 작성 방식

GitHub 저장소에는 마크다운 파일을 작성해 책 내용을 구성합니다.

예시 구조는 다음과 같습니다.

```text
my-wikidocs-book/
├─ README.md
├─ 01_시작하기.md
├─ 02_환경설정.md
└─ 03_실습하기.md
```

파일명과 목차 구성 방식은 위키독스의 GitHub 연동 규칙에 맞춰 정리합니다.

처음에는 짧은 문서 하나를 만들어 push한 뒤, 위키독스에서 정상 반영되는지 확인하는 것이 좋습니다.

---

## 8. Git으로 수정 내용 올리기

로컬에서 원고를 수정했다면 다음 순서로 GitHub에 올립니다.

```bash
git status
git add .
git commit -m "docs: add wikidocs book draft"
git push
```

push가 완료되면 GitHub 저장소에 변경 내용이 반영됩니다.

---

## 9. 위키독스에서 반영 확인

GitHub에 push한 뒤 위키독스의 해당 책 페이지로 돌아가 반영 여부를 확인합니다.

반영이 바로 되지 않는다면 다음을 확인합니다.

- GitHub 저장소가 올바르게 연결되어 있는지
- 원고 파일이 저장소에 정상적으로 push되었는지
- 위키독스에서 요구하는 파일명/목차 규칙을 지켰는지
- 저장소 접근 권한이 맞는지

---

## 10. 전체 흐름 요약

전체 과정을 한눈에 보면 다음과 같습니다.

```text
위키독스 로그인
→ 계정설정
→ 나의 책
→ 깃허브 책 만들기
→ 책 정보 입력
→ GitHub 저장소 연동
→ 원고 작성
→ commit & push
→ 위키독스에서 반영 확인
```

깃허브 연동 방식의 장점은 원고를 코드처럼 버전 관리할 수 있다는 점입니다.

책을 계속 고쳐 나가야 한다면, 위키독스 편집기만 사용하는 것보다 GitHub 연동 방식이 훨씬 안정적입니다.
